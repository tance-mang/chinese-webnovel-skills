# 爽点设计（shuangdian）

"爽"是网文的核心商品。本技能负责设计和强化爽点桥段，尤其是打脸。

## 何时用
- 设计打脸/装13/扮猪吃老虎/逆袭/复仇桥段
- 剧情太平、读者说"不爽"
- 一个爽点要写得更解气

## 三段式爽感闭环
任何爽点都遵循：
```
压抑 → 蓄势 → 释放
```
- **压抑**：把不公/嘲讽/危机写具体（越具体，释放越爽）
- **蓄势**：主角隐忍/布局/觉醒（番茄类 1–2 章内必释放，起点类可铺 3–5 章）
- **释放**：打脸/翻盘/碾压，把爽感外显写足

## 打脸四拍（最常用）
```
1. 嘲讽：反派/众人看不起主角，越嚣张越好
2. 沉默：主角不辩解，淡淡出手（蓄势的张力）
3. 碾压：实力/真相/反转一击致命
4. 围观：旁观者的震惊反应——这一拍最容易被新手漏掉，但最爽
```

## 爽点升级链（防疲劳）
同一种爽点不能原地重复，必须升级对象/规模/方式：
```
打脸同窗 → 打脸长辈 → 打脸宗门 → 打脸天才 → 打脸大势力 → 打脸天道
```
设计前先问：上一次同类爽点的"量级"是多少？这次要更高。

## 工作流
1. 确认爽点类型（调 `references/trope-library.md` 的 23 个元素选型）
2. 确认频道：男频重实力碾压，女频重情绪解气/体面打脸（调 `references/channel-guide.md`）
3. 按三段式 + 打脸四拍设计桥段
4. 检查是否比同类前作升级了
5. 输出桥段大纲或直接写片段

## 频道差异
- **男频打脸**：实力碾压、越级挑战、围观惊呼
- **女频打脸**：让瞧不起我的人后悔、体面解气、打白莲不掉价（作者素材库的宅斗场景）

## 输出
```
【爽点类型】
【压抑】具体的不公/嘲讽
【蓄势】主角怎么忍/怎么布局
【释放】打脸四拍逐拍
【升级】比上次同类爽在哪
```
