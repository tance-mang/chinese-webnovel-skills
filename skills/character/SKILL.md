# 人物设定（character）

网文人物不必复杂，但必须**有记忆点、服务爽感、前后一致**。

## 何时用
- 设计主角/反派/女主男主/配角
- 做人设卡、人物小传、关系网
- 人物脸谱化、立不住、OOC（前后不一致）

## 工作流

### 主角设计
- **名字**：避开烂大街网文姓名（叶辰/苏念/林婉儿），起贴时代、有记忆点的名 → 调 `references/name-library.md`，或用 `name` 技能专门起一批
- **记忆点**：一个让读者记住的标签（口头禅/习惯/独特价值观）
- **代入感**：男频要能让读者"想成为他"，女频要能让读者"想成为她/被这样爱"（调 `references/channel-guide.md`）
- **主观能动性**：主角要自己做选择，不能被金手指牵着走
- **成长弧光**：性格/心境从开篇到结局怎么变（不只是变强）
- **金手指适配**：性格与金手指的化学反应（调 `references/goldfinger-types.md`）

### 反派设计
- **强度匹配**：反派要够强够聪明，主角赢才有含金量
- **动机合理**：反派也有自己的逻辑，不是为坏而坏
- **打脸价值**：越嚣张/越位高，被打脸越爽（调 `references/trope-library.md`）
- **层级递进**：小反派→中反派→大BOSS，逐级升级

### 配角 / CP
- 配角要有功能（推进/衬托/调节），别工具人到没记忆点
- CP 要有"关系张力"：误会/身份差/双向奔赴的节奏

## 输出：人设卡
```
【姓名】+ 一句话记忆点
【身份】开局身份 → 真实身份/最终身份
【性格】3 个关键词 + 1 个反差点
【核心欲望】他/她到底要什么
【金手指/能力】
【成长弧光】开篇状态 → 结局状态
【与主线关系】
【关系网】与主要角色的关系和张力
【经典台词】1–2 句（体现性格）
```

## 一致性
- 设定后建议存为人物档案，写正文时对照，避免 OOC。
- 大长篇用 `review` 定期检查人物状态是否前后矛盾。
