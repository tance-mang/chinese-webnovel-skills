# 平台趋势调研（trends）

网文是追风口的生意。本技能拉取**当下**各平台流行什么，帮作者选对题材、贴对标签。

## ⚠️ 数据时效说明
平台喜好会变，本技能**优先使用联网搜索（WebSearch）拉取最新数据**；若环境无联网工具，则基于 `references/platform-profiles.md` 给出常青判断并明确提示"非实时，建议联网核对"。

## 何时用
- 开新书前的市场调研（"现在番茄流行什么"）
- 查某平台当下热门标签/题材/榜单
- 追风口、找蓝海题材
- 想了解某平台读者最近的偏好变化

## 工作流

### 第 1 步：明确范围
确认：哪个平台？男频/女频？要题材趋势还是热门标签还是榜单？
（知乎盐选偏短篇付费；UC 偏中老年下沉；Webnovel 是出海，读者结构不同——调 `references/platform-profiles.md`）

### 第 2 步：联网调研
用 WebSearch 检索（示例查询）：
- "<平台> 男频/女频 热门题材 <当前年月>"
- "<平台> 新书榜 / 飙升榜 <当前月>"
- "<平台> 签约 热门标签 <当前年>"
- "网文 风口 题材 <当前季度>"
聚合多条结果，去重，标注信息时间。

### 第 3 步：输出趋势报告
```
【平台 + 频道 + 调研时间】
【当下热门题材】Top 5（+ 简评：还能不能进 / 是否已红海）
【热门标签】可直接用的标签清单（按热度）
【上榜套路】当下榜单常见的金手指/爽点套路
【读者偏好变化】最近在追什么、厌倦了什么
【蓝海建议】竞争小但有需求的细分方向
【选题建议】结合趋势的 2-3 个可写方向
【数据来源 + 时效】（列出处，标注非官方/可能滞后）
```

### 第 4 步：衔接选题
把趋势结论喂给 `idea` 生成具体选题，喂给 `submission` 选对标签。

## 让它"日常更新"（自动化）
静态插件本身不会自己更新数据，但可以让它每天自动跑：
- 用 Claude Code 的定时/循环能力，例如每天早上自动运行本技能查某平台趋势（参考 `/schedule` 或 `/loop`）。
- 或作者手动每周跑一次，存进自己的 book-bible 追踪。

## 原则
- **务必标注数据时效和来源**，网文趋势过期很快。
- 区分"还能进的风口"和"已经红海的题材"，别让作者追到尾部。
- 出海平台（Webnovel 等）的偏好和国内不同，分开判断。

## 下一步
趋势定了 → 用 `idea` 落选题、`submission` 配标签。
