# 投稿与过稿经验库

> 写得好 ≠ 签得了约。过稿是另一套游戏：编辑在 3 章内、读者在 3 分钟内做判断。
> 本库供 `submission` / `idea` / `review` 调用。各平台具体政策会变，**实时数据用 `trends` 联网核对**。

---

## 一、编辑/系统在看什么（签约判断点）

网文签约本质看**两个数据预期**：
1. **留存**：黄金三章能不能让读者点到第 4 章（收藏率/追读率）
2. **商业性**：题材是不是当下平台要的、爽点能不能持续供给

编辑快速过稿时盯的点（按顺序）：
- **前 3 句**：有没有立刻抓人（钩子）
- **第 1 章**：金手指/核心卖点上没上、主角立没立住
- **前 3 章**：有没有第一个小高潮、主线悬念清不清楚
- **简介/标签**：卖点一句话说得清吗、题材标签对不对平台
- **更新稳定性**：能不能日更稳定（签约后考核）

## 二、黄金三章过稿标准（自检清单）

- [ ] 第 1 句立危机或反差
- [ ] 第 1 章内金手指/核心设定上线（番茄）或最晚第 3 章（起点）
- [ ] 第 1 章主角有一个让人记住的点
- [ ] 第 3 章前有第一个明确爽点/小高潮
- [ ] 三章内主线悬念清晰（读者知道"接下来要看什么"）
- [ ] 每章章尾有钩
- [ ] 没有大段世界观说明文/背景交代（最大劝退点）

## 三、各平台投稿要点

| 平台 | 投稿要点 | 易拒原因 |
|---|---|---|
| **起点** | 重设定深度与文笔，男频玄幻/都市为主；签约看三章+大纲 | 节奏太慢、开篇铺垫太长、同质化 |
| **番茄** | 极致快节奏，第1章上爽点上钩；数据流（完读/追读）说话 | 开篇拖、爽点慢、单章太长 |
| **七猫** | 下沉爽文，套路直给，悬念前置 | 太文艺、不够直白 |
| **晋江** | 女频精品，重情感与人物，文笔在意 | 三观不正、人设崩、感情线弱 |
| **UC** | 落魄主角身怀绝技开局，狗血强冲突，中老年下沉 | 太文艺、冲突不够狗血 |
| **知乎盐选** | 第一人称短篇，开篇第一句埋悬念，段落≤3行 | 开篇不抓人、不够沉浸 |
| **Webnovel（出海）** | 男频系统/都市最吃香，爽点直给，适配翻译读者 | 文化梗太重、节奏慢 |

## 四、简介与标签（过稿的"门面"）

### 简介公式
```
[身份/处境钩] + [金手指/转折] + [核心爽点预期] + [悬念尾钩]
```
- 例（男频）：被未婚妻退婚当天，他觉醒了吞噬万物的系统。三年后归来，当年看不起他的人，跪着求他收下门票。
- 例（女频）：穿成恶毒女配的第一天，她就把剧本烧了。男主追妻火葬场？不好意思，姐忙着升级打脸。

### 标签
- 选 **3–5 个平台热门标签**，第一个标签=核心卖点。
- 标签要"对平台胃口"，用 `trends` 查当下热标。

## 五、签约后的更新考核
- **日更字数**：多数平台要求日更 3000–6000 字稳定。
- **断更红线**：签约期断更易解约，开书前先存稿 3–5 万字。
- **数据维护**：追读掉了要会救（用 `review` 诊断）。
