# 大纲设计（outline）

把选题展开成可执行的多层大纲，保证百万字长线不崩设定、不断爽点。

## 何时用
- 用户要整本书大纲 / 分卷大纲 / 单章细纲
- 规划主线、支线、伏笔
- 已有设定，需要排剧情节奏

> 可带"平台 + 频道 + 题材"参数，如 `/webnovel:outline 番茄 男频 都市战神`，背后自动调 `references/platform-profiles.md`（节奏）+ `genre-library.md`（题材套路）+ `channel-guide.md`（频道逻辑）。

## 大纲三层结构

### 层一：全书主线（故事骨架）
- **篇幅规划**：先定档位（短/中/长篇）和目标总字数 → 算卷数/章数（调 `references/length-standards.md`：总字数 ÷ 单章字数 ≈ 章数，每卷 25万–35万切卷；长篇先冲 ≥8万过分发坎）
- **核心矛盾**：贯穿全书的主线冲突（主角 vs 什么）
- **成长弧线**：主角从 A 到 Z 的实力/心境跃迁（分阶段）；同步定**力量体系/战力刻度**（存档案，防越级，见 `project-memory.md`）
- **大爆点规划**：每 10–20 万字一个卷级高潮（身份揭晓/势力翻盘/打脸大佬）
- **终局**：结局画面 + 主线大谜团的答案
- **伏笔总表**：编号 + 埋点章节 → 回收章节 + 重要度（这是防烂尾的关键）

### 层二：卷纲（每卷 = 一个完整的爽点闭环）
每卷给出：
- 卷主题 + 本卷主要敌人/目标
- 入卷钩子 + 出卷爆点
- 本卷爽点链（参考 `references/trope-library.md`）
- 本卷金手指成长台阶（参考 `references/goldfinger-types.md`）
- 埋下/回收的伏笔

### 层三：章纲（单章蓝图）
每章给出：
- 一句话剧情
- 本章爽点/钩子（章尾钩调 `references/hook-library.md`）
- 出场人物 + 关键道具
- 与主线/伏笔的关联
- 目标字数（按平台，调 `references/platform-profiles.md`）

## 工作流
1. 确认要做哪一层（全书 / 某一卷 / 某几章）。缺前置信息就先问或调 `idea` 的产出。
2. 按平台定节奏：番茄高密度爽点、起点可铺长线（调 `platform-profiles.md`）。
3. 先列骨架，再填细节，**显式标注伏笔的埋点和回收点**。
4. 输出结构化大纲（markdown 表格/列表），方便用户直接用 `expand` 写正文。

## 原则
- **黄金三章**：前 3 章必须有完整的小钩-冲突-小高潮，决定收藏。
- **爽点不断档**：检查每 3 章是否有情绪释放，番茄类每章都要有。
- **伏笔必回收**：埋的每个钩都要在总表里有回收计划。
- **节奏呼吸感**：紧张（战斗/打脸）后给短暂舒缓（日常/感情），再拉紧。

## 下一步
章纲好了 → 用 `expand` 写正文；想强化某段爽点 → 用 `shuangdian`；查读者留存 → `review`；长篇查逻辑/时间线/战力一致 → `continuity`。
> 长篇建议大纲一出就用 `memory` 建档（设定/人物/伏笔/时间线/战力），边写边更，防百万字失忆。
