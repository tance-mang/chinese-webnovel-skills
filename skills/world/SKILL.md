# 世界观先行搭建（world）

强设定题材（玄幻/仙侠/历史/战争/末世/电竞）大多**世界观先行**：先搭世界规则，人物再在里面活动，才不逻辑崩。本技能帮你把世界搭成**可存档、随剧情变化、能约束人物**的体系，而不是写完就忘的背景板。

## 何时用
- 开一本强设定的书，先立世界（力量/种族/势力/政治/经济/地理/历史）
- 怕"写到后面世界对不上、主角无视自己定的规则、去的地方上一章已经没了"
- 要让世界**随剧情变状态**（战争推进、城市损毁、势力更替）
- 和相邻技能分工：`world`=搭世界规则；`goldfinger`=主角的外挂；`outline`=剧情主线；`memory`=存档；`continuity`=核对一致

## 工作流（详见 `references/worldbuilding.md`）

### 1. 搭世界骨架
按题材覆盖需要的模块，每条给**唯一标签**便于跨章引用：
- 力量/武学/魔法体系（境界阶梯 + 规则 + 上限与代价）
- 种族/势力（立场/强弱/与主角关系）、政治、经济（货币/稀缺资源）
- 地理地图（关键地点/距离/势力范围）、历史大事记（开局前发生过什么）
> 不是题材都要全建：都市换成"圈层/职场规则"，短篇一句话世界即可。存进 `00_设定.md`（大长篇可拆 `08_世界观.md`）。

### 2. 让世界活起来（世界状态）
按章记关键状态挂到时间线（`07_时间线.md`）：战争=爆发、北城损毁=20%→60%→沦陷、粮道=断。**人物行动依赖世界状态**，防"去北城但北城已被夷平"。

### 3. 人物与世界耦合
能力符合体系（火系下雨减伤）、行动受地理/政治限制、可触发事件随世界状态变。先立规则，后守规则。

### 4. 写章前"世界四问"（背景驱动剧情）
写第 N 章前先答：① 当前世界状态？② 主角此刻能做/不能做什么？③ 冲突是否从世界规则长出来？④ 环境/地形对本章情绪或战斗的影响？把答案注入写作提示。

## 输出
- **世界设定卡**（分模块 + 标签 + 规则 + 关联时间线），可直接存档
- **世界状态表**（随章更新的关键变量）
- **写章前核对清单**（世界四问）

## 原则
- **档案=单一事实来源**：定了就对照，要改先改档案。
- **世界要活不要展览**：随章更新状态；但世界观是地基不是设定集，始终服务爽点和追读。
- 老实说：没有真正的"自动注入/世界推演引擎"，靠的是**档案 + 每章人工核对**达到同样效果。

## 配合
- 存世界设定/状态 → `memory`（`references/project-memory.md`）
- 世界状态/规则一致性核对 → `continuity`
- 主角外挂规则与代价 → `goldfinger`；把世界排进剧情 → `outline`
