# 投稿过稿诊断（submission）

写得好不一定签得了约。本技能从**编辑/签约视角**体检稿子，帮作者过稿。

## 何时用
- 准备投稿/签约，想先自检
- 写/优化书名、简介、标签
- 黄金三章过稿体检
- "为什么我的书没签约 / 怎么提高过稿率"

## 工作流

### A. 过稿体检（最常用）
1. 确认目标平台（番茄/晋江/起点/UC/知乎盐选/Webnovel）→ 调 `references/submission-guide.md` + `references/platform-profiles.md`
2. **核对篇幅/字数门槛**（调 `references/length-standards.md`）：番茄短故事 ≥6000 才能签、长篇首签 ≥2万、全平台分发 ≥8万；字数没到门槛先提示作者补够再投。
3. 按"黄金三章过稿标准"逐条核对前三章（调 submission-guide 的自检清单）
4. 输出**签约可能性评估 + 致命劝退点 + 逐条改法**：
```
【目标平台】
【签约可能性】高/中/低 + 一句话结论
【致命劝退点】最可能被拒的 1-3 点（按编辑视角排序）
  - 开篇是否 3 句抓人
  - 金手指/卖点上线时机
  - 三章内有无小高潮
  - 有无大段背景说明文（头号劝退）
  - 单章字数/节奏是否合平台
【逐条改法】每个问题给具体改法
【优先级】先改哪个最能提过稿率
```

### B. 书名/简介/标签
- **书名**：3 个候选，突出核心卖点和爽点
- **简介**：用公式 `身份钩 + 金手指转折 + 爽点预期 + 悬念尾钩`（见 submission-guide）
- **标签**：建议 3–5 个，第一个=核心卖点；当下热门标签用 `trends` 联网查

### C. 投稿策略
- 一稿多投的平台改造建议（番茄砍铺垫、起点补设定、晋江加情感线）
- 存稿与日更考核提醒

### D. 投稿记录联动（记录"为什么没过"）
和创作档案 `06_投稿记录.md` 联动（格式见 `references/project-memory.md`、模板 `templates/submission-log-template.md`）：
- **投稿前**：先读 `06_投稿记录.md`，把历史**高频拒因标签**（如 #开篇慢 #爽点晚 #同质化）拎出来，本次体检重点查这些，避免重犯。
- **投稿后**：用户告诉你结果，**追加一条记录**：平台/日期/结果/编辑反馈/你的判断/改动/复盘标签。
- **归纳**：累计 3 次以上的拒因标签 = 系统性问题，明确指出并给针对性改法。
> 没有创作档案就提示用 `memory` 建档。

## 原则
- 站在**编辑和系统数据**的角度看，不是站在作者自嗨的角度。
- 最常见的拒稿原因是"开篇铺垫太长/爽点太慢"——优先查这个。
- 平台政策会变，涉及"当下平台要什么题材"时，提示用 `trends` 联网核对。

## 下一步
体检出问题 → 用 `hook` 改开篇、`idea` 调卖点、`expand` 重写三章、`review` 复检。
> 投稿前再用 `proofread` 过一遍**错别字 + 敏感/违禁词**（像番茄后台标红+一键替换），免得因错字/触审被打回。
