# 世界观先行库（World Engine）

> 玄幻/仙侠/历史/电竞/战争/末世这类**强设定**题材，大作者多是"**世界观先行**"——先把世界规则搭好，人物再在里面活动，写出来才不逻辑崩。
> 供 `world` / `outline` / `memory` / `continuity` 调用。与 `project-memory.md` 配套：这里定**怎么搭世界**，那里定**存哪**。

> ⚠️ 老实说一句：提示词/技能工具**没有真正的"自动注入 + 世界推演引擎"**（那是代码层的事）。本库给的是**可落地的工作流 + 档案结构 + 写章前核对清单**——靠"档案当单一事实来源 + 每章核对"达到同样效果，不靠假装的 simulation。

---

## 一、世界设定模块（搭骨架，存进 00_设定.md 或 大长篇拆 08_世界观.md）

强设定题材建议覆盖这几块；每块给**唯一标签**便于跨章引用（如 `#力量-炼气`、`#势力-天剑宗`、`#地理-北城`）：

| 模块 | 要点 | 标签例 |
|---|---|---|
| 力量/武学/魔法体系 | 境界阶梯、能力规则、上限与代价 | `#力量-金丹` |
| 种族/势力 | 各方立场、强弱、与主角关系 | `#势力-魔门` |
| 政治体系 | 谁掌权、规则、禁忌 | `#政治-皇权` |
| 经济体系 | 货币、资源、稀缺品（灵石/筹码/粮草）| `#经济-灵石` |
| 地理地图 | 关键地点、距离、势力范围 | `#地理-北城` |
| 历史大事记 | 开局前发生过什么、留下的伏笔 | `#史-灭门案` |

每个条目写：**标签 + 描述 + 规则(可触发/可引用) + 关联时间线**。

> 不是题材都要全建：都市/言情把"力量体系"换成"圈层规则/职场规则"即可；短篇只需一句话世界设定。

## 二、世界状态随章变化（World State，存进 07_时间线.md）

静态世界会崩。世界要随剧情**变状态**，并约束人物能做什么。按章记关键状态：

```
# 世界状态（随章更新，挂在时间线上）
- 第1章  战争爆发=是；北城损毁=20%；主角位置=进入北城
- 第8章  北城损毁=60%；粮道断=是；天剑宗闭山=是
- 第12章 北城沦陷；幸存者南迁
```

好处：人物行动**依赖世界状态**——防"主角去北城，可北城上一章已被夷平/根本没提过它存在"。

## 三、人物与世界耦合（能力受规则约束）

人物不孤立，要和世界规则绑定，胜负才可信、金手指才不飞天：
- **能力符合体系**：火焰剑伤害有区间，下雨/水属性场地里要打折——别无视自己定的规则。
- **行动受地理/政治限制**：去某地要路程、要通行、要避开某势力，不能瞬移无视。
- **可触发事件随世界状态变**：北城沦陷后，"在北城买装备"这条路就没了。

> 这条和 `goldfinger`（金手指要有门槛/代价）、`continuity`（战力一致/资源变化）是一伙的：**先立规则，再守规则**。

---

## 四、背景驱动剧情（写每章前，先问世界四问）

写第 N 章前，先核对（这就是"写前注入"的人工版）：
1. **当前世界状态是什么？**（看 07_时间线 的世界状态最新行）
2. **主角此刻在世界里能做什么 / 不能做什么？**（受体系/地理/政治/资源限制）
3. **本章冲突点从世界规则里长出来吗？**（而不是凭空塞）
4. **环境/天气/地形对这章的情绪或战斗有什么影响？**（让环境推进剧情，别只写景，见 `ai-detector.md` 环境审计）

把这四问的答案塞进写作提示，生成的内容就不会脱离世界逻辑。

---

## 五、世界先行 vs 人物先行（怎么选）
- **世界先行**：玄幻/仙侠/历史/战争/末世/电竞——规则即看点，先搭世界。
- **人物先行**：言情/都市情感/短篇——情绪即看点，先立人物，世界够用就行（一句话设定）。
- 长篇即便人物先行，也要有**最小世界一致性**（地理/时间/规则别穿帮）。

## 原则
- **档案 = 单一事实来源**：世界设定一旦定，写正文对照它；要改先改档案再写。
- **先立规则，后守规则**：定了"下雨火系减伤"，全书就认。
- **世界要活**：随章更新世界状态，别留个一动不动的背景板。
- 别过度：强设定也要服务爽点和追读，世界观是地基不是展览馆——读者要故事不要设定集。
