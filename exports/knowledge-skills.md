## 技能：aidetect
**触发场景**：中文网文 AI 味量化检测与评分。当用户要给正文"打个 AI 味分/量化看看有多像 AI"、要知道"问题具体在第几句"、要查比喻密度/连续同句式/句长波动/段落分布/高频情绪词重复/极端词堆砌，或要在改稿前先定位 AI 味重灾区时使用。产出 AI 味指数(0–10)+八项指标实测+扣分定位清单。

# AI 味量化检测器（aidetect）

`deslop` 告诉你"怎么改"，本技能告诉你"**有多重、扣分在哪几项、问题在第几句**"。把"我觉得有点 AI 味"变成可量化的诊断：**AI 味指数 7.2/10，扣分在比喻密度和连续同句式，位置见清单。**

## 何时用
- 想给一段/一章正文打个"AI 味分"
- 改稿前先定位 AI 味重灾区，按优先级下刀
- 量化对比"改前 vs 改后"降了多少
- 和相邻技能的分工：
  - `aidetect`：**量化检测+评分+定位**（出诊断）
  - `deslop`：表面改（修辞/标点/重复/极端词）
  - `human`：深层改（失控感/非理性/不闭环）
  - `annotate`：单章节奏可视化（钩子/爽点/修饰词）
  - `review`：整卷读者留存五维诊断

## 八项量化指标（详见 `references/ai-detector.md`）
1. **比喻密度** ≤3/千字且不连用（"像…似…宛如…"三连=污染）
2. **排比/三段式** ≤1处/章（"不是A不是B而是C"）
3. **连续同句式** ≤2 句（连续3+句"主语+动作"报警）
4. **句长波动** 一段内长/短 ≥3 倍，要有碎句也要有长句
5. **段落分布** 长短段交错，忌全长/全短/雷同对仗
6. **高频情绪词重复** 震惊/瞳孔一缩/心头一震… 同章各≤1
7. **极端词** 非常/极其/瞬间/顿时… 删九成，≤2–3/千字
8. **转折词/空泛升华/标点误用** 然而/与此同时…、"生活总是充满挑战"、`...`/句号当顿号

> ⚠️ 这是**估算式启发规则**，不是逐字计数：扫一遍估每千字次数、找连续段、挑最密的位置。重在**定位 + 趋势**。

## AI 味指数（0–10，越高越浓 = 越该改）
连续同句式、句长波动权重最高（最毁人味感）。分档：
- **0–3 像真人**：偶有小瑕，可发
- **4–6 半成品**：有机器痕迹，按定位改重点项
- **7–10 浓 AI 味**：一眼假，整章过 `deslop` + `human`

## 输出：量化扫描报告
```
【AI 味量化扫描 · 第 N 章 / 选段】
字数：__  AI 味指数：__/10  一句话结论：……

指标(实测 vs 阈值)：比喻密度 / 连续同句式 / 句长波动 / 段落分布 /
高频情绪词 / 极端词 / 排比三段式 / 转折升华标点  —— 逐项 ✓/⚠/✗

【扣分定位】（给到句/段）
① 第_段："原句" → 连续4句主语+动作 → 改法
② 第_段："像…似…宛如…" → 比喻三连 → 删两个
【优先改】先动 AI 味分最高的 1–2 项
```

## 原则
- **先定位后评分**：分数服务于"改哪里"，不是为打分。
- **结合平台**：番茄本就短句快节奏——短句密集≠AI，关键看**有没有波动、有没有碎句和长句穿插**。
- **只诊断不改稿**：出报告后交给 `deslop`/`human`/`emotion` 动手。

## 配合
- 表面改 → `deslop`（`references/anti-ai-checklist.md`）
- 深层人味 → `human`（`references/human-texture.md`）
- 句式节奏 → `references/sentence-rhythm.md`
- 情绪演出来 → `emotion`（`references/show-emotion.md`）

## 技能：annotate
**触发场景**：中文网文正文标注与节奏可视化。当用户要在正文里标出钩子/爽点/高潮/打脸/伏笔的位置、看一章的节奏分布、检查爽点密度、或标记和建议修饰词（短平快爽文爱加的强化词）时使用。产出带标签的正文 + 节奏数据表，方便作者直观看数据。

# 正文标注与节奏可视化（annotate）

把一章正文"透视"出来：钩子在哪、爽点几个、高潮在第几段、修饰词够不够。让作者**一眼看见自己的节奏数据**。

## 何时用
- 想看"这章爽点够不够、钩子在哪、节奏对不对"
- 标记/检查修饰词（短平快爽文的"味儿"词）
- 给一章做发布前的可视化体检
- 和 `review` 的区别：`review` 是整卷/整书的宏观五维诊断；`annotate` 是**单章逐段的微观标注**

## 工作流

### 第 1 步：逐段标注（输出带标签的正文）
把用户的正文按段落标注，行内插入标签：
- `【钩】` 钩子（开篇钩/章尾钩/段落悬念）
- `【爽】` 爽点（逆袭/打脸/碾压等，调 `references/trope-library.md`）
- `【高潮】` 本章情绪顶点
- `【脸】` 打脸桥段（标出四拍：嘲讽/沉默/碾压/围观）
- `【伏】` 伏笔埋点
- `【修】` 修饰词（标出已用的强化词/围观反应词，调 `references/xiushi-ci.md`）
- `【缓】` 节奏舒缓段

### 第 2 步：输出节奏数据表
```
【本章体检表】
字数：____
钩子：开篇钩 ✓/✗　章尾钩 ✓/✗　段落悬念 __ 个
爽点：__ 个（类型：打脸/逆袭/…）｜密度评价：达标/偏少（按平台）
高潮：在第 __ 段｜情绪强度：__/5
打脸四拍：嘲讽__ 沉默__ 碾压__ 围观__（缺哪拍标出）
修饰词：强化词__ 围观反应__ ｜密度：合适/偏少/过载(油腻)
伏笔：埋__ 个　回收__ 个
节奏曲线：紧—缓—紧 是否有呼吸感
AI 味（粗判）：比喻密度/连续同句式/句长波动 有无异常 → 要量化评分用 `aidetect`
```

### 第 3 步：给改进建议
- 爽点断档的段 → 建议补什么（指到段）
- 打脸缺了"围观"拍 → 建议加围观反应词（给 2-3 个候选，调 `xiushi-ci.md`）
- 修饰词偏少的爽点句 → 建议在哪句加什么词
- 修饰词过载的段 → 建议删到 2-3 个（避免油腻/AI 味）
- 章尾收尾平淡 → 建议换哪种章尾钩（调 `hook-library.md`）

## 原则
- 标注要**落到段/句**，不空泛。
- 修饰词建议遵循"稀疏使用、峰值集中、频道有别"（见 `references/xiushi-ci.md`）。
- 按平台判断密度：番茄要密，起点可疏（调 `references/platform-profiles.md`）。
- 男频重"猛/爽/燃"的强化词与围观反应，女频重神态/情绪词。

## 下一步
按标注 → 用 `expand` 重写弱段、`shuangdian` 补打脸、`deslop` 清理过载修饰词；要 AI 味量化评分+定位 → `aidetect`。

## 技能：character
**触发场景**：中文网文人物设定。当用户要设计主角、反派、配角、CP，做人设卡、人物小传、角色弧光、人物关系网，或人物"立不住/脸谱化/前后不一"时使用。产出有记忆点、有成长弧、服务爽感的人设卡。

# 人物设定（character）

网文人物不必复杂，但必须**有记忆点、服务爽感、前后一致**。

## 何时用
- 设计主角/反派/女主男主/配角
- 做人设卡、人物小传、关系网
- 人物脸谱化、立不住、OOC（前后不一致）

## 工作流

### 主角设计
- **名字**：避开烂大街网文姓名（叶辰/苏念/林婉儿），起贴时代、有记忆点的名 → 调 `references/name-library.md`，或用 `name` 技能专门起一批
- **记忆点**：一个让读者记住的标签（口头禅/习惯/独特价值观）
- **代入感**：男频要能让读者"想成为他"，女频要能让读者"想成为她/被这样爱"（调 `references/channel-guide.md`）
- **主观能动性**：主角要自己做选择，不能被金手指牵着走
- **一个不理性的点（人味）**：给主角一个偏执/执念/会赌会自毁的点——"明知错还做"才像活人，太正常=太 AI（调 `human` 技能 / `references/human-texture.md`）
- **成长弧光**：性格/心境从开篇到结局怎么变（不只是变强）
- **金手指适配**：性格与金手指的化学反应（调 `references/goldfinger-types.md`）

### 反派设计
- **强度匹配**：反派要够强够聪明，主角赢才有含金量
- **动机合理**：反派也有自己的逻辑，不是为坏而坏
- **打脸价值**：越嚣张/越位高，被打脸越爽（调 `references/trope-library.md`）
- **层级递进**：小反派→中反派→大BOSS，逐级升级

### 配角 / CP
- 配角要有功能（推进/衬托/调节），别工具人到没记忆点
- CP 要有"关系张力"：误会/身份差/双向奔赴的节奏

## 输出：人设卡
```
【姓名】+ 一句话记忆点
【身份】开局身份 → 真实身份/最终身份
【性格】3 个关键词 + 1 个反差点
【核心欲望】他/她到底要什么
【金手指/能力】
【成长弧光】开篇状态 → 结局状态
【与主线关系】
【关系网】与主要角色的关系和张力
【语言指纹】风格 / 口癖 / 句长偏好 / 常用词 / 不会说什么
【经典台词】1–2 句（体现性格 + 语言指纹）
```

> **语言指纹（speech_profile）**：每个主要角色记一张——说话粗/雅/毒舌、口头禅、句子长短、爱用/绝不用的词。这是长篇后期"所有人一个口气"的解药：写对话前对照注入（用 `dialogue` 技能），存进 `01_人物.md`。

## 一致性
- 设定后建议存为人物档案（含语言指纹），写正文时对照，避免 OOC、避免一人一腔。
- 大长篇用 `review` 查人设留存、用 `continuity` 核对人物状态/行为前后是否矛盾。

## 下一步
人设好了 → 用 `outline` 把人物弧光排进剧情；用 `expand` 写出人物的"声音"。

## 技能：coach
**触发场景**：中文网文写作教学与陪练（新手友好，选择性开启）。当用户想学怎么写网文、问写作技巧、要拆解某个套路为什么有效、要带教学点评的改稿、或说"教我写""我是新手"时使用。只在用户主动调用时出现，不会自动插入其他写作技能。

# 写作教学陪练（coach）

面向想**学会写网文**的作者，不只是替你写，而是讲清"为什么这样写"。新手友好。

## ⚙️ 选择性开启
- 本技能**只在你主动调用时**出现（`/webnovel:coach` 或明确说"教我"）。
- 其他写作技能（expand/spark 等）默认**不带教学解说**，专心出稿。
- 想让某次写作**附带教学点评**，明确说"边写边讲"，否则保持安静。

## 三种用法

### 1. 问答（学知识点）
回答具体写作问题，例：
- "黄金三章到底要写什么？" "章尾钩怎么留？" "金手指怎么设计不烂尾？"
- 调 `references/writing-craft.md` 及相关库，**讲原理 + 给例子 + 给可操作步骤**。

### 2. 拆解（懂套路为什么有效）
拿一个套路/桥段/范例，拆给作者看：
- 它用了什么结构（调 `references/trope-library.md`、`hook-library.md`）
- 读者为什么爽 / 为什么被钩住
- 怎么自己复用到不同题材

### 3. 陪练（带教学的改稿）
作者发来自己的片段，本技能：
- 先**肯定写得好的地方**（鼓励为主，新手易气馁）
- 指出 1–3 个最关键的问题，**讲清为什么是问题**
- 给"改前 vs 改后"对照，让作者看懂怎么改
- 留一个小练习/方向，让作者自己再试

## 教学原则
- **讲人话**：少术语，多打比方，新手能懂。
- **少而精**：一次只教 1–3 个点，别一股脑全倒。
- **鼓励优先**：先肯定再纠错，保护写作热情。
- **可操作**：每个建议都能落到"下一句怎么写"。
- **授人以渔**：解释原理，让作者下次自己会，而不是永远依赖 AI。

## 教学地图（常见学习路径）
新手建议顺序（详见 `references/writing-craft.md`）：
1. 开篇与黄金三章（留人）→ 配 `hook`
2. 爽点与节奏（追读）→ 配 `shuangdian`、`annotate`
3. 金手指与人设（立住）→ 配 `goldfinger`、`character`
4. 大纲与伏笔（不崩）→ 配 `outline`
5. 文笔与去 AI 味（手感）→ 配 `deslop`、`xiushi-ci`
6. 投稿与平台（变现）→ 配 `submission`、`trends`

## 下一步
学完某个点 → 用对应的写作技能实战，再回来 `coach` 复盘。

## 技能：continuity
**触发场景**：中文网文长篇连贯性与逻辑检查。当用户写长篇怕"前后矛盾/写到一百章崩了/设定漂移"、要查时间线乱不乱、人物行为合不合理、资源凭空变、身份忽暴忽藏、伤势没延续、战力越级、或要对一卷/全书做逻辑体检时使用。逐章核对六条逻辑链+时间线，揪出和前文/档案的矛盾并给改法。

# 连贯性与逻辑检查（continuity）

长篇崩盘的真正死因不是写不动，是**前后矛盾、时间线乱、设定漂移**——AI 写长篇尤其如此：50 章正常，100 章开始崩，300 章彻底失忆。本技能是长篇的"逻辑防线"，和 `memory`（记忆）配套：档案负责**记**，本技能负责**核对**。

## 何时用
- 写长篇，每写完一章/几章核对一次逻辑
- 怕"写到后面崩设定/人物漂移/伏笔忘了"
- 查时间线（"三天后""两个月后"到底乱没乱）
- 查资源/伤势/身份/战力前后对不对得上
- 每卷末做一次全量连贯性体检
- 和 `review` 的区别：`review` 是**读者留存**视角（钩子/爽点/追读）；本技能是**作者后台**视角（设定不矛盾、逻辑能自洽）

## 怎么查（六条逻辑链 + 时间线）
对照档案（`01_人物.md` / `03_伏笔追踪.md` / `07_时间线.md` / 战力表）逐章核对，详见 `references/continuity-check.md`：

1. **人物行为合理性** —— 行为有动机吗？和人设冲突吗？（通缉犯不会逛街买包子）
2. **剧情因果链** —— 新设定有铺垫吗？挖的坑记进伏笔表了吗？（防"突然冒出/挖了不填"）
3. **资源变化** —— 钱/道具/兵力只能按剧情增减（防"上章吃不起饭，下章掏十万灵石"）
4. **身份暴露状态** —— 维护"知情人名单"：谁知道主角的真实身份/实力/马甲（扮猪吃老虎流命脉）
5. **伤势 / 状态延续** —— 重伤/中毒/闭关要持续，到期要兑现（防"垂死→下章大战三百回合"）
6. **战力 / 等级一致** —— 对照战力表，没提升就别越级打平
7. **时间线** —— 每个"X 时间后"都要能换算成时间线上的一个点；并行线汇合不矛盾；设定的期限到点兑现

## 工作流
1. 先读档案（没建档先用 `memory` 初始化，至少要有人物状态、伏笔表、时间线、战力表）。
2. 读要检查的章节，逐条过六条链 + 时间线。
3. 出**连贯性体检报告**（格式见下），矛盾定位到段/章 + 给改法。
4. 提示把本章变化（人物状态/资源/时间线/知情人）**写回档案**，否则下章继续漂。

## 输出：连贯性体检报告
```
【连贯性体检 · 第 N 章 / 第 X 卷】
时间线：本章 = Day__ ｜跨度 __ ｜与主线 一致/冲突
六条逻辑链：行为/因果/资源/身份/伤势/战力 —— 逐条 ✓/✗ + 问题定位
矛盾清单：① 第_段 与 第_章/档案 冲突 → 改法（改本章/回改前文/补一句交代）
待兑现：到期未兑现的期限/伏笔（毒/闭关/约定）
档案待更新：人物状态/资源/时间线/知情人 哪些要写回
```

## 原则
- **定位到段/章**，指明和"哪一章/哪条档案"冲突，不空泛。
- 每个矛盾都给**可执行改法**。
- 分轻重：主线/战力/身份是硬伤先改；次要情绪线留白不算病（见 `references/human-texture.md`"不闭环≠烂尾"）。
- 查完必提示**回写档案**——这是防失忆的闭环。

## 配合
- 记忆/档案/时间线/战力表 → `memory`（`references/project-memory.md`）
- 读者留存五维诊断 → `review`

## 技能：deconstruct
**触发场景**：拆解对标平台爆火网文，学结构不抄字。当用户想拆书、分析爆款小说为什么火、对标某本热门书、提炼爆文的开篇/爽点/金手指/节奏套路、或要"照着这本爆款的写法写我自己的"时使用。输出可复用的结构模板，严守不抄袭原创底线。

# 拆书对标爆款（deconstruct）

爆款不是抄来的，是**拆出结构、复用到自己题材**。本技能把一本火书"拆零件"，提炼可复用的写作模板。

## ⚠️ 原创底线
**学结构、学套路、学节奏——不抄原文、不照搬剧情、不复制人物**。
套路（如打脸、扮猪吃老虎）是公共写作技法，可自由借鉴；具体文字、独创设定、剧情桥段受版权保护，不可复制。涉及同人/IP 见 `fanfic-guide.md`。

## 何时用
- "这本书为什么火，帮我拆一下"
- 对标某本爆款，想学它的写法
- 提炼爆文的开篇/爽点/金手指/节奏模板
- 想"照爆款结构写我自己的故事"

## 工作流

### 第 1 步：获取对标对象
用户提供书名/简介/章节文本，或描述这本书。
（要查它在平台的真实热度/同类爆款，用 `trends` 联网。已有拆解范例见 `references/case-studies.md`，可参照其拆解维度。）

### 第 2 步：六维拆解
按网文要素拆出**结构**（不复述原文）：
```
【对标书】+ 平台 + 频道 + 题材
【一句话爆点】它的核心卖点是什么
【开篇结构】用了哪种开篇钩（调 hook-library）+ 几句立危机
【金手指】什么流派、怎么上线、怎么成长（调 goldfinger-types）
【爽点节奏】爽点类型 + 密度 + 升级链（调 trope-library）
【人物模板】主角记忆点 + 反派强度 + 关系张力
【章节节奏】单章字数、章尾钩规律、付费节点设计
【成功要素】它做对了哪 3 件事
```

### 第 3 步：提炼可复用模板
把"它的结构"抽象成"你能用的模板"：
```
【可复用模板】
- 开篇公式：______（套你的题材）
- 金手指框架：______（换你的设定）
- 爽点链：______（换你的对手/场景）
- 节奏配方：______
【套用示范】用你的题材填一遍这个模板（原创内容）
【差异化提醒】别 1:1 复刻，至少换一个维度做出新鲜感
```

### 第 4 步：衔接创作
把模板喂给 `idea`/`outline`/`spark`，生成你**自己原创**的书。

## 原则
- 输出的是**抽象结构和方法**，不是原文段落。
- 永远提醒"换皮不够，要做差异化"（调 `trope-library.md` 反套路一节）。
- 多本同类爆款一起拆，能看出"这个题材的公共成功公式"。

## 下一步
模板提炼好 → `idea` 落选题、`outline` 排大纲、`spark` 直接孵化初稿。

## 技能：deslop
**触发场景**：去除中文网文里的 AI 味。当用户说正文"太像AI写的/有翻译腔/太工整/没人味/排比堆砌/读起来假"、嫌"无缘无故的修辞""引用奇怪跨度大/语域不一致"、嫌"重复的修辞句式""同义反复(他非常生气他怒了)""极端词太多(非常/极大)""莫名其妙的陈述句""句号当顿号""段落太对仗工整行数雷同"，或要把 AI 稿改得像真人时使用。产出更口语、更有网感、风格统一的网文文字。

# 去 AI 味（deslop）

AI 写的网文常有通病：太工整、爱排比、空洞抒情、翻译腔、用词高级但没网感。本技能把它改成"真人网文作者"的手感。

## 何时用
- 正文读起来"像 AI / 像作文 / 有翻译腔"
- AI 生成的稿子要落地成网文
- 文字太"端着"，没有网文的爽利感

## AI 味的典型症状 → 改法

| 症状 | 表现 | 改法 |
|---|---|---|
| **无缘无故的修辞** ⭐ | 为修辞而修辞的比喻/叠喻，对剧情情绪没帮助 | 每个比喻问"它让读者更懂更爽了吗"，没用就删；爽文优先具体动作不优先比喻 |
| **跨度大/不搭的引用** ⭐ | 朴实文里突然引古诗典故/学术概念/不合身份的梗，语域不一致 | 先定语域全文统一；问"这个叙述者会这么说吗""和全文一个频道吗"，不搭就删换 |
| **排比癖** | 三句排比抒情、"不是…而是…" | 删掉两句，留最有力的一句，或改成动作 |
| **喊情绪（不演）** | "我很心疼""他感到无比愤怒" | 用神态/动作/生理反应/潜台词演出来，让读者自己感受（详见 `references/show-emotion.md`，专项改用 `emotion` 技能）|
| **翻译腔** | "这是一个如此美丽的……" "令人难以置信的" | 改口语："美得不像话""离谱" |
| **过度工整/句子太顺** | 每句长度差不多、整齐划一、太顺；上下段字数行数雷同 | 句长不均(短长交错+偶尔50字长句)、加省略号/破折号/换行做停顿；段落打破对称（详见 `references/sentence-rhythm.md`）|
| **重复修辞/句式/词** | 同一个比喻、句式、词反复出现 | 通篇查重，删或换 |
| **同义反复** | "他非常生气，他怒了"——一个意思说两遍 | 留最准的一个，最好换成动作 |
| **极端词滥用** | 非常/极其/极大/无比/十分 堆砌 | 删九成，程度靠具体画面给 |
| **莫名陈述句/强行升华** | 突然插"生活总是充满挑战"这类空话 | 一律删 |
| **句号当顿号** | "苹果。香蕉。橘子。" | 并列用顿号「、」，逗号停顿，句号留给完整句 |
| **装饰符号癖** | 正文/标题撒 ✦✨◆●★ 当点缀或分隔 | 全删，分隔用空行或"——"，标题用纯文字 |
| **省略号用错** | 用 `...` 或 `。。。` | 中文省略号用「……」(六点一个标点) |
| **破折号滥用** | 「——」反复出现 | 少用，能用句逗就别用破折号 |
| **方括号入正文** | 正文用「【】」 | 正文不用【】(仅系统流弹窗例外) |
| **引号用错** | 平白用单引号''当主引号 | 主引号用双引号""，单引号''只用于嵌套 |
| **滥用高级词** | 堆四字成语和书面语 | 换网文常用的直白表达 |
| **解释过度** | 把人物情绪/动机全解释一遍 | 留白，让动作和对话自己说话 |
| **AI 转折词** | "然而""与此同时""不仅如此""值得一提的是" | 删掉或换成口语连接 |
| **端着的对话** | 角色说话像书面语 | 改成符合人设的口语，带语气词 |

## 工作流
1. 先读用户的文字，标出 AI 味最重的句子
2. 逐处按上表改写，给出"原句 → 改后"对照
3. 整体调节奏：加碎句、单句成段、对话提速
4. 保留剧情和爽点不变，只改"手感"

> 完整深度清单（含无缘无故修辞、跨度大引用、语域一致、show-don't-tell）见 `references/anti-ai-checklist.md`。

## 网文文字手感
- **短**：忌长难句，一句话一个意思
- **快**：对话和动作推进，少叙述
- **狠**：情绪用具体细节砸出来，不铺排形容词
- **活**：适当口语、网感、语气词，但别过火
- **一致**：**先定语域(朴实/华丽/古风/现代)，全文统一**——朴实老成的文就别乱引典故和怪东西，风格一致 > 单句出彩

## 注意
- 去 AI 味 ≠ 加网络烂梗，过度玩梗也会让稿子廉价
- 保持作者原有风格，只去"机器感"不强行改风格
- 改完可对照原文确认剧情/人设没改变

## 配合
- 写正文时就少出 AI 味 → 用 `expand`（已内置避免 AI 腔的要求）。
- 改之前想先**量化定位**"AI 味多重、扣分在哪几句" → 用 `aidetect`（AI 味指数 + 八项指标 + 定位清单，见 `references/ai-detector.md`），按它的优先级下刀更快。
- 本技能治**表面**（修辞/标点/符号/重复/极端词）。若表面改干净了还"像 AI/像说明书"，那是**深层缺人味**——用 `human` 技能加失控感（非理性行为/情绪不完整/不闭环/私货），见 `references/human-texture.md`。

**触发场景**：中文网文对话与台词。当用户要写/改对话、设计金句名场面台词、写打脸装13台词、虐心或甜宠台词、抱怨"对话太生硬/太书面/像AI/每个角色一个腔"、或要让人物说话有区分度时使用。产出口语化、有潜台词、符合人设声音的台词。

# 对话与台词（dialogue）

对话是"AI 味"最大的破绽，也是网文名场面的核心。AI 对话像书面陈述，网文对话要像真人交锋。

## 何时用
- 写一段对话 / 改一段生硬的对话
- 设计金句、名场面台词
- 写打脸/装13/反派/虐心/甜宠台词
- "对话太书面/像 AI/每个角色一个腔"
- 让不同角色说话有区分度

## 工作流

### A. 写/改对话
1. 明确：谁和谁、什么场景、要达成什么（推进剧情/暴露性格/制造张力）
2. 按好台词 5 原则写（调 `references/dialogue-library.md`）：口语化、有潜台词、符合人设声音、留白、每句有用
3. **去 AI 对话腔**：删复述/念设定、打碎长句、删"愤怒地说"改用动作、不同角色不同句式、加潜台词

### B. 设计专项台词
按类型从台词库取（打脸/装13/反派/虐/甜/日常）：
- **打脸**：越短越狠 + 围观反应（"就这？"）
- **装13**：不解释、越淡越唬人
- **反派**：文雅的威胁最瘆人
- **女频情感**：克制比嘶吼更虐，偏爱藏细节里

### C. 金句设计
要"可截图传播"：短 + 狠 + 反差/哲理。一章一两句即可，别硬凑。

### D. 人物声音区分（语言指纹 speech_profile）
给每个主要角色定一张**语言指纹**：风格（粗/雅/毒舌）+ 口癖 + 句长偏好 + 常用词 + 绝不会说的词。保证读者不看名字也知道谁在说。
- 自检每句："**只有这个角色会这么说**吗？"
- 写对话前先调出该角色的语言指纹**对照注入**；存进 `01_人物.md`（格式见 `references/project-memory.md` 七），由 `character` 技能产出。
- **长篇尤其重要**：后期最容易"所有人退化成一个口气"——每隔几卷用 `continuity` 抽查对话是否还守得住各自指纹。

## 输出
- 直接给台词/对话，必要时"改前 vs 改后"对照
- 设计台词时说明：用了什么潜台词、体现了什么人设
- 不靠"愤怒地/开心地"等副词，靠台词本身 + 动作

## 原则
- **口语 > 书面**："就这？"胜过"你的实力不过如此。"
- **潜台词 > 直白**：话里有话才有张力。
- **留白 > 说尽**：别把情绪解释完。
- **一人一声**：消灭"所有人一个腔"。
- 金句精而少，多了做作。

## 反例对照
- ❌「我对你的背叛感到非常愤怒和失望。」→ ✅「……行。」（转身，留白更狠）
- ❌「我的实力远超于你。」→ ✅「就这？」

台词配进正文 → `expand`；整体还有 AI 味 → `deslop`；打脸桥段结构 → `shuangdian`。

# 供修改的 Word 草稿（draft）

定位：**AI 先写个"大概"的草稿供人筛选修改**，不是定稿。按作者的排版习惯导成 Word，并标出可改的地方。

## 何时用
- 把小说/章节导成 Word(.docx)
- 要分章节、带目录、章标题左对齐
- 要 AI"标出可以改的地方"让作者自己改
- "先写个大概我来看看，再筛选修改"

## 输出规则（按作者习惯，务必遵守）
1. **格式 = Word(.docx)**。
2. **绝不覆盖原文**：命名 `《书名》修改1.docx`，已存在就自增 `修改2`、`修改3`…
3. **目录**：自动插入（Word 里右键目录→更新域 生成页码）。
4. **章标题"第X章"左对齐**（用 # 开头，脚本自动转 Heading 1 + 左对齐）。
5. **标记可改处**：在草稿里用 `【修改：你的建议】` 标出——脚本会渲染成**红色高亮**，方便作者定位修改。
6. **字数**：短篇总量 **8000–30000**；**草稿写到 10000–20000**，给作者留"既能删也能扩"的余地。各档篇幅/签约门槛见 `references/length-standards.md`（如番茄短故事 ≥6000、爽款 1万–1.8万）。

## 在哪里标"修改"建议
草稿里给自己挑刺，常见可改处（借 review/annotate/deslop 的眼光）：
- 节奏偏慢、铺垫过长 → `【修改：这段铺垫偏长，可删减】`
- 开头/章尾钩子弱 → `【修改：章尾再留个钩，比如…】`
- 情绪在"喊"没"演" → `【修改：'我很难过'建议改成动作，见 emotion】`
- 无缘无故的修辞 / 不搭的引用 → `【修改：这个比喻可删】`
- 爽点/打脸不够 → `【修改：此处可加一拍围观反应】`
- 可扩可删的弹性段 → `【修改：此处可扩写到XX字 / 可精简】`

## 工作流
1. 先确认：书名、频道/平台/题材、**人称（默认第一人称）**、目标字数。
2. 写草稿（调 expand/hook/shuangdian/emotion 等，遵守去 AI 味与第一人称默认），按"# 第X章"分章，可改处插 `【修改：…】`。
3. 把草稿存成 markdown，例如 `书名.md`。
4. 运行脚本生成 Word：
   ```bash
   python "${CLAUDE_PLUGIN_ROOT}/skills/draft/build_docx.py" 书名.md --title "书名" --outdir "你的输出文件夹"
   ```
   （缺库先 `pip install python-docx`。）
5. 告诉作者：生成的文件名、正文字数、有几处【修改】建议。

## 原则
- **是草稿不是定稿**：写"大概"，把精修留给作者；用【修改】把你的判断显式标出来，而不是替作者拍板。
- **永远新建文件**，不动原稿——作者的版本由作者掌控（也可配合 `memory` 的 git 快照）。
- 短篇控制在 8000–30000，草稿停在 10000–20000 留余地。

## 其它模型（ChatGPT/DeepSeek 等不能跑脚本时）
直接按上面 Markdown 约定输出（# 第X章 + 正文 + 【修改：…】），让作者自己粘进 Word；或在能生成文件的环境里下载。

## 下一步
作者改完 → 可再 `review` 复诊、`memory` 存档/快照。

## 技能：emotion
**触发场景**：中文网文情绪描写（show don't tell）。当用户要把"我很心疼/我好痛苦/他很愤怒"这类直白喊情绪的句子改成可感的描写、要让读者自己感受到情绪、嫌情绪写得太直白/太空/没代入、或要写虐点/感动/紧张等情绪场景时使用。用神态/动作/生理反应/潜台词把情绪演出来，而不是说出来。

# 情绪描写 · 演出来别喊出来（emotion）

网文最忌"喊情绪"——"我很心疼""我好痛苦"。本技能把情绪**演出来**，让读者自己感受到，而不是被通知。

## 何时用
- 把"我很X""他感到X"改成可感的描写
- 情绪写得太直白、太空、没代入感
- 写虐点/感动/紧张/愤怒/委屈等情绪场景
- "让读者自己感受到"，而不是作者替他下结论

## 核心原则
不告诉读者"我什么感受"，而是展示"我的身体/动作/语言怎么了"，让读者自己得出那个感受。
- ❌ 「我很心疼他。」
- ✅ 「我伸手想替他擦掉血，指尖快碰到时又顿住，最后只是把外套往他身上紧了紧。」

## 工作流

### 改稿三步（最常用）
1. **找**：圈出所有"我很X / 他感到X / 心里特别X"的直白情绪句。
2. **定**：这个情绪下，身体/动作/语言会有什么真实反应？查 `references/show-emotion.md` 的「情绪→外显对照表」。
3. **换**：用 1–2 个具体细节替换那句直白，必要时加一句潜台词，**删掉"我很X"**。给"原句 → 改后"对照。

### 写新场景
直接按 6 条通道写（从 `show-emotion.md` 取，挑 1-2 条最准的）：
神态/微表情 · 小动作 · 生理反应 · 对话潜台词 · 环境物件烘托(克制) · 留白。

## 铁律
- **1–2 个准细节就够**，别堆五个动作（堆了又油腻、又 AI）。
- **克制**：成年人的情绪是收着的，越克制越戳（女频虐点尤其）。"我累了"比嚎啕更狠。
- **环境烘托要具体要轻**，别写成"无缘无故的修辞"（"那杯茶早凉了，她也没动" 好过 "心如刀绞")。
- 极快节奏（番茄打斗/打脸）可适当直给，看场景，不必处处含蓄。

## 输出
- 改稿：逐句"原句 → 改后"对照，并说明换了哪条通道。
- 新写：直接给可感的描写片段。
- 始终保留剧情不变，只改"喊→演"。

## 反例对照
- ❌「我很伤心，八年感情成了笑话。」→ ✅「我扯了扯嘴角，挤出一个笑。」
- ❌「他很愤怒。」→ ✅「他后槽牙咬得咯咯响，把杯子轻轻放下——太轻了。」
- ❌「我好想你。」→ ✅「我给那个再也不会回的号码发了条消息，又一个字一个字删掉。」

## 下一步
情绪改完 → `expand` 接进正文；整体还有 AI 味 → `deslop`；台词潜台词 → `dialogue`。

# 英文网文出海（english）

帮中国作者写**像当地作者**的英文网文，而不是"中文翻译腔"。核心：**中文卖剧情爽点，英文卖情绪钩子；英文读者要被直接扔进故事**。一切规则调 `references/english-webfic.md`。

## 🔛 总开关：Native English Mode（默认开启）
写英文时**默认**按欧美母语网文叙事，不是"中文思维→翻译成英文"，而是"英文读者本来就会这样表达"。这条改的是叙事思维，最重要：
- **优先动作、对话、冲突、悬念；降低解释、降低抒情、降低文学化描写。**
- **提高人物真实感和口语化表达；让读者沉浸在事件中，不旁观作者讲故事。**
- 严格执行 `english-webfic.md` 第〇节的**十条核心规则**（禁翻译腔/Show>Tell/句长70-20-10/禁文学堆砌/每200字推进/钩子优先/对话口语/母语短句/减副词/章尾留钩）。
- 结构按英文逻辑：**谜团 → 情绪 → 爽点**（先埋疑问再给爽）；爽感包装用**个人成长**（Survivor/Underdog/Outcast/Chosen One），不是开局身份碾压。

## 何时用
- 直接写英文小说（Wattpad / Royal Road / Webnovel / Radish / KDP）
- 把中文故事**改写**成英文原生（不是逐句翻译）
- 问"出海写英文怎么写 / 选哪个平台"
- 去掉英文里的翻译腔 / AI 腔

## 四种用法

### A. 写英文原生稿
按英文网文规则直接写英文（调 `english-webfic.md`）：
- **前 100 词有事发生**，第一句就是钩子（身份反转/危机/秘密/禁忌关系）。
- **短段落、短句**；对话**一句一行、换行**。
- **不解释、不堆形容词**，直接动作。
- 情绪用**身体反应**（`I couldn't breathe.` / `My hands shook.`），不喊抽象。
- 章标题 `Chapter X`。

### B. 中转英（改写成英文原生）
不是翻译，是**用英文重讲**：保留剧情/钩子/反转，按英文读者习惯重写节奏、段落、情绪表达；砍掉中文式铺垫和解释；去文化梗。

### C. 去英文 AI 腔/翻译腔
扫英文稿，标出并改：
- 太正式：`Therefore/However/Furthermore` → 口语或删。
- 太工整机械：`He smiled. She smiled.` → 打破对称。
- 过度文学：`Moonlight danced gracefully...` → 直白快。
- 长段落 → 拆短；对话挤一行 → 换行。

### D. 选平台
按频道/题材推荐（调 `english-webfic.md` 平台表）：
- 女性向言情 → **Wattpad**（最易上手）/ Radish
- 男频系统/升级/LitRPG → **Royal Road**
- 系统/重生/都市（像番茄）→ **Webnovel**
- 完结本卖电子书 → **KDP**

## 原则
- **像本地作者，不像翻译**：宁可简单直白，不要中式华丽。
- **情绪钩子 > 剧情解释**：把读者扔进情绪，别替他总结。
- 你的反转/钩子/快节奏是优势，适合 Wattpad 短篇 / Webnovel / 短视频小说流。

## 配合
英文也要去 AI 味 → `deslop`/`human` 的思路通用（但用 `english-webfic.md` 的英文 AI 破绽清单）；钩子结构 → `hook`。

# 正文扩写（expand）

把章纲变成可发布的正文，控制好节奏、爽点、字数和钩子。

## 何时用
- 把章纲/细纲写成正文
- 扩写、续写某一章
- 把一句话剧情扩成完整场景

## 写作前确认
1. **人称**：**默认第一人称「我」**（更沉浸、像写亲身经历，尤其知乎/女频/短篇）。动笔前**用一句话表明**"默认用第一人称写，要第三人称告诉我"；用户指定第三人称就用第三。全书人称统一。（调 `references/pov-guide.md`）
2. **章纲/剧情点**：这一章要发生什么（没有就先用 `outline` 出章纲）
3. **平台 + 字数**：单章密度调 `references/platform-profiles.md`（番茄 1500–2500、起点 2000–3000）；全书还差多少到签约/分发门槛调 `references/length-standards.md`
4. **人设档案 + 语言指纹**：调出本章出场角色的 speech_profile 对照注入，保证人物"声音"一致、不 OOC、不一人一腔（见 `skills/dialogue`、`project-memory.md` 七）
5. **前情衔接**：上一章结尾的钩子要在本章开头回应；长篇还要核对时间线/资源/伤势/战力是否接得上（见 `skills/continuity`）

## 写作要领

### 节奏
- **开头接钩**：第一段回应上章悬念或立刻进入场景，不要拖
- **场景推进**：对话 + 动作 + 心理交替，避免大段叙述
- **爽点落地**：本章规划的爽点要写到位（调 `references/trope-library.md`），打脸用"嘲讽→沉默→碾压→围观"四拍
- **章尾断钩**：结尾用章尾钩收口（调 `references/hook-library.md`）

### 文字
- **短句快节奏**：网文忌长难句，多用短句和对话推进
- **画面感**：用具体细节代替抽象形容（"他攥紧了拳，指节发白"而非"他很愤怒"）
- **情绪演出来别喊**：忌"我很心疼/我好痛苦"，用神态/动作/生理反应/潜台词让读者自己感受（调 `references/show-emotion.md`，专项改用 `emotion` 技能）
- **爽感外显**：把读者要爽的点写出来——表情、围观者反应、对手的难堪
- **避免 AI 腔**：不堆砌排比和空洞抒情，写完可用 `deslop` 检查
- **不用装饰符号**：正文和标题一律不撒 ✦✨◆★ 等符号；分隔场景用空行或"——"，章标题纯文字
- **留一点失控感（人味）**：别写得太平均太干净太顺——关键处给一个非理性行为、一句没说完的情绪、或一段不收尽的留白，让它像"情绪记录"不像"说明书"（深度调 `human` 技能 / `references/human-texture.md`）
- **句式有呼吸**：句长不均，短句长句交错，偶尔甩个 50+ 字长句；用省略号/破折号/换行做停顿，别整齐划一（调 `references/sentence-rhythm.md`）
- **可出双版本**：用户要时，给"爽文版（情绪高/反转爽）"+"高级暗黑版（极端/灰度/留刀）"两稿（调 `references/dark-style.md`）；不要求则默认单版本

### 对话
- 每个角色说话方式不同（体现人设）
- 对话推进剧情或暴露信息，不闲聊
- 适当留白，不要每句都解释
- 口语化、有潜台词，避免书面 AI 腔（调 `references/dialogue-library.md`）；要专门打磨台词/金句 → 用 `dialogue` 技能

## 输出
- 直接产出正文（markdown），标题 + 正文
- 字数贴近目标
- 结尾自带章尾钩
- 写完简述：本章爽点是什么、埋/收了哪些钩、字数

## 连贯性
- 长篇每写几章：用 `review` 查读者留存（钩子/爽点/伏笔），用 `continuity` 核对逻辑/时间线/资源/身份/伤势/战力是否前后矛盾
- 重要设定/人物状态/时间推进/战力变化，写完即提示用户**更新档案**（`memory`），否则下一章继续漂

## 下一步
正文有 AI 味 → `deslop`（量化打分定位 → `aidetect`）；爽点不够 → `shuangdian`；查留存 → `review`；查逻辑/时间线 → `continuity`。

# 同人创作与版权检测（fanfic）

帮作者写同人，**同时守住版权红线**。同人是热门赛道，但踩线轻则锁文、重则被诉。

## ⚠️ 免责
本技能提供创作实务建议，**非法律意见**。商用变现前请取得 IP 方书面授权或咨询专业人士。

## 何时用
- 想写某个 IP 的同人（影视/动漫/小说/游戏/综艺）
- 问"这个同人能不能签约赚钱 / 在哪个平台发"
- 给已有同人做**版权风险检测**
- 需要"安全改写"方案规避侵权

## 两种模式

### A. 同人版权检测（合规校验）
对一个同人创意/稿子做风险体检（调 `references/fanfic-guide.md`）：
```
【IP 版权状态】公版(>50年) / 在版受保护 → 风险等级
【目标平台政策】该平台能否同人签约/变现（晋江✅唯一合法 / 番茄UC知乎❌ / 海外较宽松）
【借用角色数】__ 个（建议≤3 且大改）
【主线原创度】100%原创 / 含原著剧情 → 风险
【原文引用】（晋江规则：≤全文1/10 且单篇≤3000字）
【OOC/审核风险】是否恶意抹黑、三观、未成年等红线
【结论】可商用 / 仅免费 / 高风险勿碰
【整改建议】怎么改到安全线内
```

### B. 同人创作（安全方案）
帮用户写同人，但内置合规：
1. 先确认 IP 和目标平台 → 判断能不能变现
2. **优先推荐公版 IP**（西游/三国/红楼/封神/聊斋）做安全二创
3. 受保护 IP：建议"借名字不借主线"——角色≤3、大改人设、100% 原创主线
4. 按网文手法写（钩子/爽点照常，调其他技能）
5. 写完自动跑一遍 A 的合规检测

## 安全度铁律
- **公版 IP 最安全**：作者去世超 50 年的作品可自由商用二创。
- **受保护 IP 不授权 = 不能收费**：免费发也可能被下架/起诉。
- **晋江是国内唯一同人合法变现区**，且有引用比例硬规则。
- **海外同人较宽松**（海外华人写影视/网文同人是蓝海），但跨境商用仍建议授权/公版。

## 下一步

# 金手指设计（goldfinger）

金手指是男频网文的发动机。烂金手指让主角变工具人，好金手指让"努力和选择"被放大。

## 何时用
- 设计系统 / 签到 / 空间 / 模拟器 / 词条 / 复制吞噬 / 重生先知 / 召唤 等外挂
- 给主角设计变强机制

## 工作流

### 第 1 步：选流派
根据频道、题材、平台从流派库选型 → 调 `references/goldfinger-types.md`
（系统流/签到流/空间流/模拟器流/词条流/复制吞噬/重生先知/召唤流/赘婿扮猪/无限流）

### 第 2 步：四要素设计
为选中的金手指设计：
1. **获取门槛**：怎么得到的？要独特、绑定主角遭遇/性格，不能路边捡。
2. **成长曲线**：分阶段台阶，每阶解锁新玩法，列出 3–5 个升级节点。
3. **使用代价**：冷却 / 反噬 / 资源消耗 / 道德取舍——制造张力，避免无敌即无聊。
4. **主线绑定**：来源/进化挂钩主线大谜团（谁给的？为什么是我？），留后期戏份。

### 第 3 步：上线与首秀
- 上线时机：番茄第 1 章 / 起点≤第 3 章（调 `platform-profiles.md`）
- 设计"小试牛刀"：上线后 1–2 章内用它解决一个具体困境，给第一个爽点

### 第 4 步：防烂尾设计
- 给金手指设"成长上限的悬念"，避免主角无敌后无戏可写
- 规划后期金手指的进化/觉醒大节点

## 输出格式
```
【金手指名称】
【流派】
【获取方式】（含独特性说明）
【核心机制】一句话
【成长台阶】阶段1→2→3→4→5（各解锁什么）
【使用代价】
【主线绑定】来源谜团 + 后期进化方向
【上线时机 + 首秀困境】
```

## 避坑
- ❌ 万能无代价 ❌ 获取随意 ❌ 主角全靠外挂无能动性 ❌ 与世界观脱节
- ✅ 金手指是放大器，放大主角的选择和努力

## 频道提示
女频较少用硬金手指，多用"身份/重生/读心/空间"软外挂，重点在关系和信息差而非数值，见 `references/channel-guide.md`。

## 下一步
金手指定了 → 用 `outline` 排它的成长节点；用 `character` 配主角性格。

# 钩子设计（hook）

网文成败在留人。本技能专攻三类钩子：**开篇钩、章尾钩、付费节点钩**。

## 何时用
- 写小说开头 / 第一段 / 第一句
- 设计每章结尾的断章悬念
- 入 V / 付费墙前的钩子
- 用户说"我的开头不抓人""读者看两章就走"

## 工作流

### A. 开篇钩（最常用）
1. 确认频道（男频/女频）和题材 → 调 `references/channel-guide.md`
2. 从七种开篇结构选 2–3 种各写一版（调 `references/hook-library.md`）：
   - 悬置危机 / 身份反差 / 结局倒挂 / 替身挡刀 / 金手指激活 / 日常爆破 / 反派开局
3. 每版给出：
   - **3 句以内的开篇文字**（直接能用）
   - **钩点解析**（读者为什么必须往下看）
   - **承接建议**（这个钩怎么往下接黄金三章）

### B. 章尾钩
1. 看用户这一章的内容，找到"即将揭晓/即将爆发"的那个点
2. 从八种章尾钩选最合适的（悬念/危机/反转/打脸预告/关系/升级/信息/倒计时）
3. 在情绪/动作的**前一秒**断章，给出 2–3 个收尾句方案
4. 提醒回收：这个钩计划第几章兑现

### C. 付费节点钩
- 把全书/本卷最强悬念或爽点压到付费墙前一页
- 给出节点章的收尾设计

## 钩子铁律
- **开篇**：3 句内立危机或反差，信息差要让读者"必须读下去才能消解"
- **章尾**：在刀尖断章，绝不在情绪最高点写完
- **必兑现**：埋的钩 1–3 章内回收，否则读者弃书

## 技能：human
**触发场景**：给中文网文加"人味/失控感"，治深层AI味。当用户嫌稿子"读着像AI/像说明书/太工整太顺/情绪太平均/人物太正常/没人味/太中立"、或想让文字更像真人写的（不是改修辞，而是加失控感）时使用。用非理性行为、情绪不完整表达、逻辑不完全闭环、私货感，把"说明书"改成"情绪记录"。

# 人味 / 失控感（human）

最高级的一层去 AI 味。**"AI 味"的本质是缺少人类失控感**——AI 写得像说明书（都解释、都合理、都闭环、都中立），人写得像情绪记录（有失控、有偏执、有省略、有偏心）。

## 何时用
- 稿子"读着像 AI / 像说明书 / 太顺太工整 / 情绪太平均 / 人物太正常 / 没人味"
- 表面修辞都改干净了（`deslop` 过了），但还是"差点意思"
- 想让文字更像真人写的——不是改字，是加**失控感**

> 和 `deslop` 的分工：deslop 治**表面**（修辞/标点/符号/重复/极端词）；本技能治**深层**（情绪/冲突/人物/不闭环/私货）。两个一起用最彻底。

## 工作流

### A. 诊断（按 5 种感受找"太 AI"处）
扫稿子，指到段标出（调 `references/human-texture.md`）：
1. **情绪太平均**：都在合理反应，没有真正失控/极端/偏执的爆点。
2. **冲突太干净**：误会→解释→解决，缺"越滚越脏"的链条。
3. **句子太顺**：没有长短变化、断句、停顿、不完整表达 → 句长不均+停顿标点，详见 `references/sentence-rhythm.md`。
4. **人物太正常**：动机合理但不极端，没有"明知错还做"的不理性。
5. **太中立**：没有偏见、情绪判断、私货感。

### B. 注入人味（3 个手法，关键处加）
1. **非理性行为**：让角色做"逻辑上不该、情绪上必须"的事（明知会输还赌、当场翻脸不解释、突然极端选择）。
2. **情绪不完整表达**：留没说出口的缺口（"他没说完那句话。""她突然不想解释了。""那一刻他有点后悔，但没停。"）。
3. **逻辑不完全闭环**：次要的事不解释尽、次要的关系不收干净。
给"原句/原段 → 改后"对照，说明加了哪种。

## 原则
- **点睛不发疯**：关键处放 1–2 个失控点，不是通篇不理性。
- **不牺牲爽和钩**：番茄仍要节奏；失控感是让爽更可信、人物更立得住。
- **不闭环 ≠ 烂尾**：主线大坑必须回收，留白只留情绪性/关系性的小东西。
- 自检一句话：**这段像说明书，还是像一个人在记录情绪？**

## 反例对照
- ❌（太干净）「她误会了他，他解释清楚，两人和好。」
- ✅（越滚越脏）「她没等他说完就挂了电话。他想再打，手指悬在屏幕上，又放下了——算了，解释什么呢。第二天，闺蜜把那张照片发到了群里。」

## 下一步
加完人味 → `deslop` 收表面、`review` 复诊、`emotion` 强化关键情绪。

# 创作档案与记忆系统（memory）

给每本书一套**持久化记忆**：设定、人物、大纲、伏笔、灵感、进度、投稿记录全存在书的文件夹里，跨会话不丢。**像 git 一样**可做版本快照、可回顾"写到哪了"。

## 何时用
- 记录/保存大纲、人设、世界观，下次接着用
- 隔了几天回来续写："我上次写到哪了？"

- 管理多本书的进度

## 档案结构（书的"记忆"）
在书的文件夹里维护这套文件（没有就用本技能初始化）：
```
《书名》/
├── 00_设定.md        立项卡 + 金手指 + 世界观 + 势力库 + 战力体系（book-bible）
├── 01_人物.md        人设卡 + 角色当前状态 + 语言指纹(speech_profile)
├── 02_大纲.md        卷纲 / 章纲
├── 03_伏笔追踪.md     埋点 → 回收 表（编号/重要度，防烂尾命脉）
├── 04_灵感本.md       spark 速记的灵感
├── 05_进度.md        写到第几章 / 累计字数 / 下一步
├── 06_投稿记录.md     每次投稿：平台/日期/结果/拒因/改动
├── 07_时间线.md       主时间线 + 知情人名单（长篇防失忆）
└── 正文/
    ├── 第001章.md
    └── ……
```
> 短篇可只用 00/01/02；**长篇务必加 07_时间线 与 战力体系**，配合 `continuity` 防写到上百章崩设定。
> 模板见 `templates/book-bible-template.md`、`templates/submission-log-template.md`、`templates/progress-template.md`；格式详见 `references/project-memory.md`。

## 四个功能

### 1. 建档（init）
为新书创建上面的档案结构，从模板填入已有信息（调 `idea`/`outline`/`character` 的产出）。

### 2. 回顾续写（resume）
读 `05_进度.md` + `02_大纲.md` + `03_伏笔追踪.md` + `01_人物.md`，**一句话汇报**：
```
《书名》写到第 N 章（累计 X 万字）
上一章结尾钩：……（本章要回应）
待回收伏笔：……
主角当前状态：……
下一步建议：写第 N+1 章，本章大纲是……
```
解决"隔几天忘了写到哪"。

### 3. 更新（update）
写完/改完后同步档案：
- 写完一章 → 更新 `05_进度.md`（章号/字数/下一步）
- 出现新伏笔/回收伏笔 → 更新 `03_伏笔追踪.md`
- 人物状态变化（升级/关系/获得物/伤势） → 更新 `01_人物.md`
- 时间推进 / 谁知道了什么秘密 → 更新 `07_时间线.md`（时间线 + 知情人名单）
- 战力变化（境界突破/新强敌登场） → 更新 `00_设定.md` 战力对照
- 新灵感 → 追加 `04_灵感本.md`
> 不回写就会"下一章继续漂"——这是防失忆的闭环。要主动核对前后矛盾用 `continuity`。

### 4. 版本快照（像 git）
两种方式，按用户情况推荐：
- **真·git（推荐，能回滚看历史）**：在书的文件夹 `git init`，每次定稿章节后 `git add . && git commit -m "第N章定稿"`。本技能可代为执行。想回到旧版本 `git log` + `git checkout`。
- **轻量快照（不用 git）**：把关键文件复制成带日期版本，如 `02_大纲_2026-06-05.md`，留存历史版本。

## 原则
- 档案是**单一事实来源**：写正文前先读档案，写完后必更新，避免设定漂移。
- `03_伏笔追踪.md` 是防烂尾命脉，每章必查必更。
- 多本书各自独立文件夹，互不干扰。
- 涉及隐私/未发布稿件，提醒已在 `.gitignore` 排除，不会误传公开仓库。

## 配合
- `spark` 写灵感本、`outline` 写大纲、`character` 写人物（含语言指纹）、`submission` 写投稿记录 —— 本技能统管这些档案的回顾与版本。
- `continuity` 读本档案核对逻辑/时间线/战力一致性 —— 记忆负责"记"，continuity 负责"核对"。
- 全书篇幅/字数规划 → `references/length-standards.md`。

# 大纲设计（outline）

把选题展开成可执行的多层大纲，保证百万字长线不崩设定、不断爽点。

## 何时用
- 用户要整本书大纲 / 分卷大纲 / 单章细纲
- 规划主线、支线、伏笔
- 已有设定，需要排剧情节奏

> 可带"平台 + 频道 + 题材"参数，如 `/webnovel:outline 番茄 男频 都市战神`，背后自动调 `references/platform-profiles.md`（节奏）+ `genre-library.md`（题材套路）+ `channel-guide.md`（频道逻辑）。

## 大纲三层结构

### 层一：全书主线（故事骨架）
- **篇幅规划**：先定档位（短/中/长篇）和目标总字数 → 算卷数/章数（调 `references/length-standards.md`：总字数 ÷ 单章字数 ≈ 章数，每卷 25万–35万切卷；长篇先冲 ≥8万过分发坎）
- **核心矛盾**：贯穿全书的主线冲突（主角 vs 什么）
- **成长弧线**：主角从 A 到 Z 的实力/心境跃迁（分阶段）；同步定**力量体系/战力刻度**（存档案，防越级，见 `project-memory.md`）
- **大爆点规划**：每 10–20 万字一个卷级高潮（身份揭晓/势力翻盘/打脸大佬）
- **终局**：结局画面 + 主线大谜团的答案
- **伏笔总表**：编号 + 埋点章节 → 回收章节 + 重要度（这是防烂尾的关键）

### 层二：卷纲（每卷 = 一个完整的爽点闭环）
每卷给出：
- 卷主题 + 本卷主要敌人/目标
- 入卷钩子 + 出卷爆点
- 本卷爽点链（参考 `references/trope-library.md`）
- 本卷金手指成长台阶（参考 `references/goldfinger-types.md`）
- 埋下/回收的伏笔

### 层三：章纲（单章蓝图）
每章给出：
- 一句话剧情
- 本章爽点/钩子（章尾钩调 `references/hook-library.md`）
- 出场人物 + 关键道具
- 与主线/伏笔的关联
- 目标字数（按平台，调 `references/platform-profiles.md`）

## 工作流
1. 确认要做哪一层（全书 / 某一卷 / 某几章）。缺前置信息就先问或调 `idea` 的产出。
2. 按平台定节奏：番茄高密度爽点、起点可铺长线（调 `platform-profiles.md`）。
3. 先列骨架，再填细节，**显式标注伏笔的埋点和回收点**。
4. 输出结构化大纲（markdown 表格/列表），方便用户直接用 `expand` 写正文。

## 原则
- **黄金三章**：前 3 章必须有完整的小钩-冲突-小高潮，决定收藏。
- **爽点不断档**：检查每 3 章是否有情绪释放，番茄类每章都要有。
- **伏笔必回收**：埋的每个钩都要在总表里有回收计划。
- **节奏呼吸感**：紧张（战斗/打脸）后给短暂舒缓（日常/感情），再拉紧。

## 下一步
章纲好了 → 用 `expand` 写正文；想强化某段爽点 → 用 `shuangdian`；查读者留存 → `review`；长篇查逻辑/时间线/战力一致 → `continuity`。
> 长篇建议大纲一出就用 `memory` 建档（设定/人物/伏笔/时间线/战力），边写边更，防百万字失忆。

**触发场景**：中文网文错别字校对与敏感词/违禁内容过审检查。当用户要查错别字(的地得/在再/做作等)、查敏感词违禁词、投稿前过审自检、问"这段会不会被屏蔽/被标红/过不了审"、或要"像番茄那样标出错字并给一键替换建议"时使用。逐句扫描，给出"原词→建议词"对照供作者一键替换。

# 错别字校对与过审检查（proofread）

模仿番茄/起点提交后台的"标红 + 一键替换"体验：扫一遍正文，揪出**错别字**和**可能触发平台审核的敏感/违禁内容**，每处给"原句 → 改后"对照，作者照着替换即可。

## 何时用
- 查错别字（的/地/得、在/再、做/作、即/既…）
- 投稿/发布前过审自检："这段会不会被标红/屏蔽/过不了"
- 要"像番茄那样"标出问题词 + 给替换建议

> ⚠️ **能力边界（务必如实告诉作者）**：
> - **错别字**：按句子意思判断同音/形近误用，准确度高，可放心用。
> - **敏感词/违禁**：各平台的**确切屏蔽词库是内部机密且时常变动**，本技能给的是基于**公开审核常识**的启发式排查（覆盖政治/色情/暴恐/赌毒/迷信/真实品牌人名等大类 + 常见踩雷），**是"安全网"不是"保证命中番茄后台的每一个词"**。最终以平台实际审核为准；拿不准的词建议作者自行替换或上报平台。
> - 需要核对"某平台当下的公开内容规范"时，可联网查（同 `trends` 的联网能力）。

## 工作流

### 第 1 步：错别字扫描
逐句按意思判断，常见类型（详见 `references/sensitive-words.md` 错别字高频对）：
- **同音误用**：的/地/得、在/再、做/作、需/须、即/既、账/帐、应该/因该
- **形近误用**：辩/辨/辫、急燥(→躁)、迫不急待(→及)
- **多字/漏字/重字**：「他他走了」「走了进去去」
- **成语写错**：「迫不及待」「川流不息」「黄粱一梦」
- **标点**：`...`/`。。。`→「……」、并列误用句号→顿号（与 `deslop` 一致）

### 第 2 步：敏感/违禁扫描（按大类排查，见 `references/sensitive-words.md`）
- 涉政/时政、领导人/真实政治人物
- 色情低俗、过度性描写、擦边
- 暴力血腥、自杀自残的具体方法
- 赌博、毒品、违禁药物的具体操作
- 封建迷信渲染（区别于"修真设定"，看是否引导现实迷信）
- 真实品牌/商标/药名/真实人名地名（易被判广告或侵权）
- 辱骂脏话、地域/群体歧视
- 价值观红线、未成年相关红线

### 第 3 步：输出"标红 + 一键替换"清单
```
【校对报告 · 第 N 章 / 选段】
字数：__  错别字：__ 处  敏感/违禁疑似：__ 处

—— 错别字 ——
① 第_段「他在做什么」→「他在作什么」？… 按"作为"应为「作」｜原句:……
   建议替换：做 → 作
② 第_段「迫不急待」→「迫不及待」
   建议替换：急 → 及

—— 敏感/违禁（疑似，按平台审核常识）——
① 第_段「<触发词/句>」类别:涉政｜风险:高
   建议改写：……（给 1–2 个安全替换）
② 第_段「<真实品牌名>」类别:商标/广告嫌疑｜风险:中
   建议替换：用虚构名「XX」

【优先处理】高风险敏感项先改；错别字可批量替换。
```

## 原则
- **每处都给"原词→建议词"**，让作者像点番茄的"一键替换"一样照改。
- 错别字**按句子意思**判断，不是机械匹配（"做/作"看语境）。
- 敏感项**标类别 + 风险等级 + 替换**，并说明这是启发式、以平台为准。
- 不误伤：修真/玄幻的"法术/境界"、合理剧情冲突不算违禁；只标真正可能触审的。

## 配合
- 投稿三章签约体检（节奏/钩子/字数门槛）→ `submission`
- 文字手感/AI 味 → `deslop`、`aidetect`
- 当下平台公开规范联网核对 → `trends`

# 审稿与追读诊断（review）

不只是挑错，而是从**读者留存**视角诊断稿子，给出能直接改的建议。这是本插件区别于通用写作工具的核心。

## 何时用
- 审一章 / 一卷 / 全书
- 查剧情漏洞、伏笔断点、人设 OOC
- 诊断追读率/完读率为什么掉
- 发布前体检

## 六维诊断

### 1. 钩子诊断（留人）
- 开篇 3 句有没有立住危机/反差？
- 每章是否有有效章尾钩？标出"平淡收尾"的章（调 `references/hook-library.md`）
- 付费节点前是否压了强钩？

### 2. 爽点诊断（追读）
- 爽点密度是否匹配平台？（番茄每章、起点每 3 章，调 `references/platform-profiles.md`）
- 标出"爽点断档"的连续章节
- 爽点是否原地重复没升级？（调 `references/trope-library.md` 升级链）
- 打脸四拍是否完整（尤其"围观"拍有没有漏）

### 3. 伏笔诊断（防烂尾）
- 列出所有已埋伏笔 → 标记：已回收 / 待回收 / 疑似遗忘
- 标出"挖了没填的坑"和"突然冒出没铺垫的设定"

### 4. 人设诊断（一致性）
- 人物言行是否前后一致、有没有 OOC
- 主角是否有主观能动性（还是被金手指牵着走）
- 反派强度是否撑得起主角的胜利

### 5. 节奏诊断（呼吸感）
- 是否一直紧绷或一直平淡？
- 战斗/打脸后有没有舒缓，舒缓后有没有再拉紧

### 6. 人味/失控感诊断（像不像真人写的）
- 情绪是不是太"平均"（都合理反应，没人真的失控/偏执/在痛）？
- 冲突是不是太"干净"（误会→解释→解决，缺"越滚越脏"）？
- 句子是不是太"顺"（无长短变化、无停顿、无不完整表达）？
- 人物是不是太"正常"（没有"明知错还做"的不理性/偏执/赌）？
- 是不是太"中立"（没有偏见、私货感）？
- 一句话标准：读起来像**说明书**还是**情绪记录**？太像说明书 → 用 `human` 技能加失控感（调 `references/human-texture.md`）。
- 想要**量化**到"AI 味指数 X/10 + 扣分在哪几句" → 用 `aidetect`（比喻密度/连续同句式/句长波动/段落分布/情绪词重复，见 `references/ai-detector.md`）。

## 输出：诊断报告
```
【整体评分】留人 / 追读 / 连贯 各打分 + 一句话结论
【致命问题】最影响数据的 1–3 点（按优先级）
【钩子】哪几章收尾平淡 → 怎么改
【爽点】哪段断档 / 没升级 → 怎么补
【伏笔】待回收清单 + 疑似遗忘的坑
【人设】OOC 或强度问题
【节奏】哪里需要松/紧
【修改优先级】先改哪个见效最快
```

## 原则
- 诊断要**具体到章/段**，不说空话
- 每个问题都给**可执行的改法**，不只是指出问题
- 优先级排序：先改最影响读者留存的

# 剧本写作（script）

剧本和小说是两套语言。铁律：只写镜头能拍到的，看得见、听得见。不写心理活动、不写"他想起三年前"，情绪靠动作、表情、台词演出来。一切格式调 `references/script-format.md`。

## 何时用
- 写剧本 / 短剧 / 分集剧本 / 台词本
- 把小说（网文）改成剧本
- 问剧本格式怎么排

## 默认：竖屏短剧（当下最火）
不特别说明就按竖屏短剧写：约 80–100 集，单集 1–2 分钟（剧本约 300–600 字），黄金 3 秒抓人，爽点密集，每集结尾强卡点。

## 四种用法

### A. 写短剧剧本
1. 先定题材/频道/主线爽点（调 `trope-library.md`、`hook-library.md`）。
2. 按标准格式写（场景标题 + 客观动作描述 + 人物台词 + 表演提示），见 `script-format.md`。
3. 单集结构：开场钩子（前 3 句立冲突）→ 冲突升级 → 小反转/爽点 → 集尾卡点。
4. 每集结尾在最炸的前一秒切断，留钩子。

### B. 小说改剧本
砍心理描写、留动作和对白；按地点/时间拆场景；把叙述信息变成可拍的动作或说出口的台词；按短剧密度重排节奏，爽点提前、卡点压集尾。

### C. 影视/长剧本
标准剧本格式（场景标题/动作行/人物台词），节奏可比短剧缓，重人物和结构。投正规渠道可用好莱坞格式（INT./EXT.）。

### D. 格式校验
检查：是否有非镜头内容（心理/旁白解释）、场景标题是否规范、台词是否能演、每集有没有卡点。

## 格式要点（速记）
```
第 N 集

场景 N   地点   日／夜   内／外

客观动作描述，只写能拍的。

人物名（表演提示）：台词。

（集尾卡点：一行动作或台词制造悬念。）
```
- 表演提示用圆括号（），不用【】。
- 画外音标（V.O.），闪回标 闪回：。

## 原则
- 只写能拍的：删一切心理描写和作者解释。
- 台词能演：短、口语、一句一意思（调 `dialogue-library.md`）。
- 一场一冲突，没戏就删。
- 去 AI 味同样适用：台词别书面别工整，动作别堆形容词。

# 爽点设计（shuangdian）

"爽"是网文的核心商品。本技能负责设计和强化爽点桥段，尤其是打脸。

## 何时用
- 设计打脸/装13/扮猪吃老虎/逆袭/复仇桥段
- 剧情太平、读者说"不爽"
- 一个爽点要写得更解气

## 三段式爽感闭环
任何爽点都遵循：
```
压抑 → 蓄势 → 释放
```
- **压抑**：把不公/嘲讽/危机写具体（越具体，释放越爽）
- **蓄势**：主角隐忍/布局/觉醒（番茄类 1–2 章内必释放，起点类可铺 3–5 章）
- **释放**：打脸/翻盘/碾压，把爽感外显写足

## 打脸四拍（最常用）
```
1. 嘲讽：反派/众人看不起主角，越嚣张越好
2. 沉默：主角不辩解，淡淡出手（蓄势的张力）
3. 碾压：实力/真相/反转一击致命
4. 围观：旁观者的震惊反应——这一拍最容易被新手漏掉，但最爽
```

## 爽点升级链（防疲劳）
同一种爽点不能原地重复，必须升级对象/规模/方式：
```
打脸同窗 → 打脸长辈 → 打脸宗门 → 打脸天才 → 打脸大势力 → 打脸天道
```
设计前先问：上一次同类爽点的"量级"是多少？这次要更高。

## 工作流
1. 确认爽点类型（调 `references/trope-library.md` 的 23 个元素选型）
2. 确认频道：男频重实力碾压，女频重情绪解气/体面打脸（调 `references/channel-guide.md`）
3. 按三段式 + 打脸四拍设计桥段
4. 检查是否比同类前作升级了
5. 输出桥段大纲或直接写片段

## 频道差异
- **男频打脸**：实力碾压、越级挑战、围观惊呼
- **女频打脸**：让瞧不起我的人后悔、体面解气、打白莲不掉价（作者素材库的宅斗场景）

## 输出
```
【爽点类型】
【压抑】具体的不公/嘲讽
【蓄势】主角怎么忍/怎么布局
【释放】打脸四拍逐拍
【升级】比上次同类爽在哪
```

# 网络热梗 · 实时查 + 不突兀（slang）

热梗能让现代向网文有网感，但**用错地方就立刻廉价又出戏**。本技能负责"搜到当下的梗"+"用得不突兀"。

## 何时用
- 想给现代都市/校园/职场/现实向小说加点网感
- 想知道当下抖音/小红书(RedNote)/微博/B站(TikTok/Douyin) 流行什么梗
- 问某个梗"现在还火不火 / 会不会过气了"
- 担心"加梗突兀、显廉价"

## ⚠️ 时效性
热梗每月在变，**优先用联网搜索（WebSearch）拉当前的梗**；无联网时基于通用知识给判断并**明确标注"可能已过期，建议联网核对"**。

## 三种用法

### A. 查当前热梗（联网）
用 WebSearch 检索（示例）：
- "抖音/小红书/微博 最新 热梗 流行语 <当前年月>"
- "<某个梗> 还火吗 出处 用法"
聚合多条、去重，输出：梗 + 含义 + 出处 + 适用场景 + **是否还在火（标时间）**。

### B. 怎么用不突兀（重点）
给定场景/题材/人物，按 6 条铁律判断（调 `references/slang-guide.md`）：
1. 看题材（古风/玄幻/正剧 → 不用现代梗）
2. 贴人物身份（年轻有网感才说）
3. 适量（一篇 1–3 个）
4. 避开快过气的
5. 放进对话/心理，别硬塞旁白
6. 服务情绪/人物，不为玩而玩

输出：能不能用 / 用哪个 / 放哪句 / 怎么融，给"示例改写"。

### C. 体检（查突兀/过气/语域不符）
扫稿子里的梗，标出：
- 题材/语域不符的（古风里的现代梗 → 删）
- 可能已过气的（→ 提示核对或换）
- 硬塞旁白、堆太多的（→ 减量、移进对话）

## 原则
- **宁可不用，不要硬蹭**：拿不准就不用，比用错强。
- **越"一句话场景梗"越短命，越"概念词"越长青**。
- 梗是调味，不是主菜；少而准。

## 下一步
确定用哪个梗 → `expand`/`dialogue` 融进正文；整体网感与去 AI 味 → `deslop`。

# 灵感记录与一键成稿（spark）

灵感转瞬即逝。本技能帮作者**随手记灵感**，并能**把一个灵感直接孵化成大纲和初稿**——可指定字数、自动分章节。

## 何时用
- 突然有个点子，想先记下来（不打断当前写作）
- 翻出灵感本里某个点子，想展开成大纲/初稿
- "我就这一句话想法，帮我写出来"
- 要按指定字数、指定章节数出稿

## 两种模式

### 模式 A：灵感速记（记录）
把用户的灵感**追加**到灵感本（默认 `灵感本.md`，在当前工作目录；没有就创建）。每条记录：
```
## [YYYY-MM-DD HH:MM] #标签
灵感原文……
（可选）一句话延展：这个点子可以怎么写
```
- 不展开、不评判，快速存下即可，保持作者心流。
- 自动打标签（题材/频道/类型），方便日后检索。
- 用户问"我有哪些灵感"时，读灵感本列出清单。

### 模式 B：灵感孵化（一键成稿）
把**一个灵感**沿"点子 → 卖点 → 大纲 → 初稿"孵化。先和用户确认产出深度和规格：

**① 规格确认（缺就用默认并说明）**
- **人称：默认第一人称「我」**（更沉浸、像亲历）；用户要第三人称也行，动笔前表明用哪个（调 `references/pov-guide.md`）
- 频道：男频/女频（调 `references/channel-guide.md`）
- 题材大类（调 `references/genre-library.md`）
- 目标平台 → 决定节奏字数（调 `references/platform-profiles.md`）
- **总字数 或 单章字数**：如"每章 3000 字"或"先写 2 万字试水"
- **章节数**：如"分 8 章"；没给就按字数自动推算
- 产出深度：只要大纲 / 大纲+前三章初稿 / 完整初稿

**② 孵化流程**
1. **立意**：把灵感提炼成一句话卖点 + 核心爽点（借 `idea` 逻辑）
2. **大纲**：生成分章细纲，**每章标注目标字数和本章爽点/钩子**（借 `outline` 逻辑）
3. **分章**：按总字数÷章节数算每章字数，列出章节表
4. **初稿**：按章逐章写正文，控字数、接钩子、章尾留扣（借 `expand` 逻辑）

**③ 字数与分章规则**
- 单章字数贴近目标（番茄 1500–2500、起点 2000–3000）
- 每章写完报"本章 XX 字 / 累计 XX 字 / 进度 X/N 章"
- 长稿分批写，每批确认后继续，避免一次性超长

## 输出示例（孵化）
```
【灵感】一句话原灵感
【卖点】提炼后的一句话卖点
【规格】男频/番茄/共8章/每章2500字/目标2万字
【分章表】
  第1章《章名》2500字 — 剧情一句话 — 爽点/钩子
  第2章……
【初稿】（按章输出，每章带字数统计）
```

## 原则
- 速记模式**快**，不要展开成大段。
- 孵化模式**先确认规格再动笔**，尤其字数和章节数。
- 一个灵感能走多远取决于它本身，单薄的点子先帮作者补设定再孵化。

## 下一步
大纲出来 → `expand` 精写、`annotate` 看节奏、`review` 体检、`submission` 投稿自检。

# 投稿过稿诊断（submission）

写得好不一定签得了约。本技能从**编辑/签约视角**体检稿子，帮作者过稿。

## 何时用
- 准备投稿/签约，想先自检
- 写/优化书名、简介、标签
- 黄金三章过稿体检
- "为什么我的书没签约 / 怎么提高过稿率"

## 工作流

### A. 过稿体检（最常用）
1. 确认目标平台（番茄/晋江/起点/UC/知乎盐选/Webnovel）→ 调 `references/submission-guide.md` + `references/platform-profiles.md`
2. **核对篇幅/字数门槛**（调 `references/length-standards.md`）：番茄短故事 ≥6000 才能签、长篇首签 ≥2万、全平台分发 ≥8万；字数没到门槛先提示作者补够再投。
3. 按"黄金三章过稿标准"逐条核对前三章（调 submission-guide 的自检清单）
4. 输出**签约可能性评估 + 致命劝退点 + 逐条改法**：
```
【目标平台】
【签约可能性】高/中/低 + 一句话结论
【致命劝退点】最可能被拒的 1-3 点（按编辑视角排序）
  - 开篇是否 3 句抓人
  - 金手指/卖点上线时机
  - 三章内有无小高潮
  - 有无大段背景说明文（头号劝退）
  - 单章字数/节奏是否合平台
【逐条改法】每个问题给具体改法
【优先级】先改哪个最能提过稿率
```

### B. 书名/简介/标签
- **书名**：3 个候选，突出核心卖点和爽点
- **简介**：用公式 `身份钩 + 金手指转折 + 爽点预期 + 悬念尾钩`（见 submission-guide）
- **标签**：建议 3–5 个，第一个=核心卖点；当下热门标签用 `trends` 联网查

### C. 投稿策略
- 一稿多投的平台改造建议（番茄砍铺垫、起点补设定、晋江加情感线）
- 存稿与日更考核提醒

### D. 投稿记录联动（记录"为什么没过"）
和创作档案 `06_投稿记录.md` 联动（格式见 `references/project-memory.md`、模板 `templates/submission-log-template.md`）：
- **投稿前**：先读 `06_投稿记录.md`，把历史**高频拒因标签**（如 #开篇慢 #爽点晚 #同质化）拎出来，本次体检重点查这些，避免重犯。
- **投稿后**：用户告诉你结果，**追加一条记录**：平台/日期/结果/编辑反馈/你的判断/改动/复盘标签。
- **归纳**：累计 3 次以上的拒因标签 = 系统性问题，明确指出并给针对性改法。
> 没有创作档案就提示用 `memory` 建档。

## 原则
- 站在**编辑和系统数据**的角度看，不是站在作者自嗨的角度。
- 最常见的拒稿原因是"开篇铺垫太长/爽点太慢"——优先查这个。
- 平台政策会变，涉及"当下平台要什么题材"时，提示用 `trends` 联网核对。

## 下一步
体检出问题 → 用 `hook` 改开篇、`idea` 调卖点、`expand` 重写三章、`review` 复检。
> 投稿前再用 `proofread` 过一遍**错别字 + 敏感/违禁词**（像番茄后台标红+一键替换），免得因错字/触审被打回。

# 网文起名（title）

书名决定点击率。本技能专治"AI 味/大众味"——AI 爱起抽象文艺名，网文要的是**关键词 + 爽点 + 钩子**。

## 何时用
- 起新书名 / 改名 / 要一批候选
- 嫌名字"太普通、太 AI、太文艺、没记忆点"
- 给已有书名做点击率优化

## 工作流

### 第 1 步：明确信息
确认 频道 + 平台 + 题材 + 核心卖点（金手指/爽点）。缺就问，或从已有设定取。
（番茄/七猫爱长名带卖点；起点偏精炼——调 `references/platform-profiles.md`）

### 第 2 步：起 6-8 个候选（分风格）
从起名套路各取几种（调 `references/title-library.md`），覆盖不同风格供挑：
- 关键词+夸张型、第一人称口语型、身份反差型、开局体、系统题材型、反套路调侃型…
每个名字必须满足**反 AI 三要素**：
1. 带**题材关键词**（系统/读心/侧写/赘婿/重生…，让算法识别）
2. 带**爽点或钩子**（无敌/打脸/倒计时/十年悬案/跪求…）
3. 用**具体意象**，不用抽象形容词（恐怖屋＞惊魂；倒计时＞宿命）

### 第 3 步：标注 + 自检
每个候选标：风格 + 卖点 + 适配平台。然后过"起名自检清单"（`title-library.md`），剔除：
- 抽象文艺名（《浮世清欢》类）
- 烂大街通用名（《重生之路》类）
- 没题材关键词的名

### 第 4 步：推荐 + 说明
挑 2-3 个最优，说明为什么点击率高；可给"安全款"和"大胆款"各一。

## 输出格式
```
【题材/频道/平台】
【推荐书名】Top 2-3（标注：为什么抓人）
【全部候选】
  1.《___》关键词型 — 卖点：___ — 适配：番茄
  2.《___》第一人称型 — …
  …
【避坑提醒】哪些常见起法会显 AI/大众，已规避
```

## 原则
- **宁可土爽，不要文艺空**：网文书名是广告，不是诗。
- 没题材关键词的名一律重起（算法识别不出 = 推不出去）。
- 抽象形容词换成具体意象。
- 热门关键词会变，需要时用 `trends` 联网查当下平台热词。

## 反例对照（教学用）
- ❌《暗夜追凶》→ ✅《读心刑警：我能看见凶手的犯罪倒计时》
- ❌《重生之路》→ ✅《重生后我把渣男全家送进了局子》
- ❌《修仙日记》→ ✅《我的灵田能种出神仙》

**触发场景**：写温暖现实向短篇/治愈系小说。当用户要写有温度、有现实意义、不狗血不煽情不说教的故事，写普通人的小困境/小成长/治愈系，或嫌稿子"太爽文/太空/想要点意义但别灌鸡汤"时使用。用小冲突+生活细节+温暖收束，让读者看到现实、带走余味。

# 温暖现实短篇（warmth）

写"有温度、有现实感、能落地"的故事——**不狗血、不煽情、不说教、不鸡汤**。番茄不吃大道理，所以温暖要**轻、具体、藏在细节里**。

## 何时用
- 想写温暖向/治愈系/现实向短篇
- 想让故事"有点意义"但**怕变成鸡汤说教**
- 写普通人的小困境、小选择、小成长
- 嫌稿子太纯爽文、太空，想加真实的温度

## 三条铁律（先记这个）
1. **温度藏在克制里，不在台词里**：一个不说话的小动作（默默把伞塞过去）＞一段暖心独白。让读者自己捂热那一下（调 `references/show-emotion.md`）。
2. **先冷后暖，暖才有重量**：纯甜是假的——先有现实的累/委屈/孤独，再给一点点暖，对比之下才砸到心里。
3. **余味=不说满**：结尾留一个没说出口的善意/一个开放的小希望，让读者自己补完。

## 工作流

### 第 1 步：定主题（一句话）
想让读者**感受到什么**？例："普通人的努力也有价值""善意可以被传递""小选择也能改变心境"。**只定一个**，别贪。

### 第 2 步：搭人物（平凡+缺点+微成长）
- 主角：普通人 + 一个弱点（犹豫/胆小/会妥协）+ 一个小兴趣/怪癖
- 配角：有动机、有瑕疵，不洗白不刻意黑
- 关系：微冲突 + 小温暖

### 第 3 步：排情节（起承转合，现实逻辑）
- **起**：普通生活 + 一个小困扰（要有钩子，别平淡）
- **承**：困扰升级 / 小冲突出现
- **转**：主角尝试 → 碰壁 → 一个小亮点（现实逻辑：不是贵人开挂）
- **合**：结局落地 + 留温暖余味（成长式/释怀式/平凡式，调 `references/warm-realism.md`）

### 第 4 步：写（番茄手感 + 温暖收束）
- 开头照样要钩子、要那个具体的小困扰勾住人——**温暖≠平淡**。
- 用 **1–2 个生活小细节**撑现实感（外卖、删了又写的消息、独自排队看病…）。
- 情绪"演出来别喊"（调 `emotion`）；对话用潜台词（调 `dialogue`）。
- 结尾给 **1 个温暖亮点**，让人轻微释怀或微笑。

## 反鸡汤（关键）
把道理翻译成"**一个具体的人做了一件具体的小事**"：
- ❌「善良总会被世界温柔以待。」
- ✅「她把多打的那份粥推给隔壁桌的老人，没说话，低头继续扒自己的饭。」

## 避雷
强行煽情/鸡汤 · 过度狗血 · 人物非黑即白 · 只为刷爽点。
但也别走反面：**温暖向仍要有钩子和冲突**，只是收束用温暖而非打脸。

## 输出
