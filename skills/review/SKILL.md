# 审稿与追读诊断（review）

不只是挑错，而是从**读者留存**视角诊断稿子，给出能直接改的建议。这是本插件区别于通用写作工具的核心。

## 何时用
- 审一章 / 一卷 / 全书
- 查剧情漏洞、伏笔断点、人设 OOC
- 诊断追读率/完读率为什么掉
- 发布前体检

## 六维诊断

### 1. 钩子诊断（留人）
- 开篇 3 句有没有立住危机/反差？
- 每章是否有有效章尾钩？标出"平淡收尾"的章（调 `references/hook-library.md`）
- 付费节点前是否压了强钩？

### 2. 爽点诊断（追读）
- 爽点密度是否匹配平台？（番茄每章、起点每 3 章，调 `references/platform-profiles.md`）
- 标出"爽点断档"的连续章节
- 爽点是否原地重复没升级？（调 `references/trope-library.md` 升级链）
- 打脸四拍是否完整（尤其"围观"拍有没有漏）

### 3. 伏笔诊断（防烂尾）
- 列出所有已埋伏笔 → 标记：已回收 / 待回收 / 疑似遗忘
- 标出"挖了没填的坑"和"突然冒出没铺垫的设定"

### 4. 人设诊断（一致性）
- 人物言行是否前后一致、有没有 OOC
- 主角是否有主观能动性（还是被金手指牵着走）
- 反派强度是否撑得起主角的胜利

### 5. 节奏诊断（呼吸感）
- 是否一直紧绷或一直平淡？
- 战斗/打脸后有没有舒缓，舒缓后有没有再拉紧

### 6. 人味/失控感诊断（像不像真人写的）
- 情绪是不是太"平均"（都合理反应，没人真的失控/偏执/在痛）？
- 冲突是不是太"干净"（误会→解释→解决，缺"越滚越脏"）？
- 句子是不是太"顺"（无长短变化、无停顿、无不完整表达）？
- 人物是不是太"正常"（没有"明知错还做"的不理性/偏执/赌）？
- 是不是太"中立"（没有偏见、私货感）？
- 一句话标准：读起来像**说明书**还是**情绪记录**？太像说明书 → 用 `human` 技能加失控感（调 `references/human-texture.md`）。
- 想要**量化**到"AI 味指数 X/10 + 扣分在哪几句" → 用 `aidetect`（比喻密度/连续同句式/句长波动/段落分布/情绪词重复，见 `references/ai-detector.md`）。

## 输出：诊断报告
```
【整体评分】留人 / 追读 / 连贯 各打分 + 一句话结论
【致命问题】最影响数据的 1–3 点（按优先级）
【钩子】哪几章收尾平淡 → 怎么改
【爽点】哪段断档 / 没升级 → 怎么补
【伏笔】待回收清单 + 疑似遗忘的坑
【人设】OOC 或强度问题
【节奏】哪里需要松/紧
【修改优先级】先改哪个见效最快
```

## 原则
- 诊断要**具体到章/段**，不说空话
- 每个问题都给**可执行的改法**，不只是指出问题
- 优先级排序：先改最影响读者留存的

## 下一步
