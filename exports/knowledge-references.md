# AI 味量化检测库（AI-Slop Detector）

> `deslop` / `human` 讲"怎么改"，本库讲"**怎么量出来、打多少分、问题在第几句**"。
> 把"我觉得有点 AI 味"变成"AI 味指数 7.2/10，扣分在比喻密度和连续同句式，位置见下"。
> 供 `aidetect` / `deslop` / `review` / `annotate` 调用。阈值是给**快节奏中文网文**调的经验值，按平台微调。

> ⚠️ 模型无法真正"逐字计数"，这些指标是**估算式启发规则**：扫一遍正文，估每千字次数、找连续段、挑最密的位置。重在**定位 + 趋势**，不必纠结精确到个位。

---

## 一、八项量化指标 + 阈值

### 1. 比喻密度（像/似/仿佛/宛如/犹如/如同/好比）
- 健康：**≤ 3 次 / 千字**，且不连用。
- 报警：连续 2–3 句都带比喻（"他像猎豹。眼神似利刃。声音宛如寒风。"）= 比喻污染。
- 查法：标出每个比喻，问"它让读者更懂/更爽/更有画面吗"，没用就删（见 `anti-ai-checklist.md` 一）。

### 2. 排比 / 三段式密度
- 健康：每章 **≤ 1 处**排比，且服务情绪高点。
- 报警："不是A，不是B，不是C，而是D""他愤怒。他不甘。他绝望。"这类工整三连。
- 查法：留最有力的一句，其余删或改成动作。

### 3. 连续同句式（最毁气氛）
- 报警：连续 **≥ 3 句**同一结构——尤其"主语+动作"（他走进去。他看向前方。他停下脚步。他皱起眉头。）。
- 健康：连续同句式 ≤ 2，第 3 句就换结构/换主语/合并/加停顿。
- 查法：扫描相邻句开头词与主谓结构，标出 ≥3 连发的句群。

### 4. 句长波动
- AI 特征：句长**过于均匀**（一段里每句都 14–16 字）。
- 健康：一段内**最长句 ÷ 最短句 ≥ 3 倍**；通篇要有 <8 字的碎句，也要有 >30 字的长句穿插。
- 报警：连续 5+ 句长度都在 ±3 字内浮动 = 流水线。
- 查法：估每句字数，看波动；太平 → 拆长句、并短句、甩一个长句（见 `sentence-rhythm.md`）。

### 5. 段落长度分布
- AI 特征：要么**全长段**（每段 400+ 字读着累），要么**全短段**（一句一段刷屏，像短视频文案）。
- 健康：**长段—短段—中段—对话—短段**交错，有呼吸感；上下段字数/行数**不要雷同**（防"对仗工整"）。
- 报警：连续 4+ 段长度雷同，或全篇段落方差极小。

### 6. 高频情绪词 / 生理反应重复
- AI 特征：一个情绪反复用同一个词；生理反应排列组合就那几样。
- 报警词族（**同章各 ≤ 1 次**，全书别滥用）：
  - 震惊 / 错愕 / 愕然 / 倒吸一口凉气
  - 瞳孔一缩 / 眼神一冷 / 眼底闪过一丝 X
  - 心头一震 / 心头一紧 / 呼吸一滞 / 呼吸一窒
  - 嘴角勾起 / 勾起一抹 X 的弧度
  - 不易察觉地 / 几不可闻地
- 健康：同类生理反应同章 ≤ 2 种；同一个词全书别"一天用八遍"。
- 查法：建本书"高频表达清单"，统计重复率，超标的换具体动作/细节。

### 7. 极端词 / 强化词
- 报警词：非常 / 极其 / 极大 / 无比 / 十分 / 特别 / 简直 / 完全 / 彻底 / 瞬间 / 顿时 / 猛地。
- 健康：**删九成**，每千字残留 ≤ 2–3 个；程度靠具体画面给（"三天没合眼，烟灰缸堆成小山" > "压力非常大"）。

### 8. AI 转折词 / 空泛升华 / 标点误用
- 转折词癖：然而 / 与此同时 / 不仅如此 / 值得一提的是 → 删或换口语。
- 空泛升华：突然插"生活总是充满未知与挑战""命运总是如此"→ 一律删。
- 标点：`...` 或 `。。。`（应为「……」）；破折号「——」反复出现；句号当顿号（"苹果。香蕉。橘子。"）；正文滥用【】；单引号当主引号。（详见 `anti-ai-checklist.md` 七）

---

## 二、AI 味指数评分（0–10，越高越浓 = 越该改）

按八项各自的超标程度给"扣分点"，汇总成指数。建议权重（连续同句式和句长波动最毁人味感，权重高）：

| 指标 | 权重 | 满分含义（贡献的"AI味分"） |
|---|---|---|
| 连续同句式 | 高 | 多处 ≥3 连发 → +2.5 |
| 句长波动 | 高 | 通篇均匀无碎句无长句 → +2.0 |
| 比喻密度 | 中 | 多处连用/超 3 千字 → +1.5 |
| 高频情绪词重复 | 中 | 词族反复 → +1.5 |
| 段落分布 | 中 | 全长/全短/雷同 → +1.0 |
| 极端词 | 低 | 堆砌 → +0.8 |
| 排比三段式 | 低 | 多处工整三连 → +0.7 |
| 转折词/升华/标点 | 低 | 频繁 → +0.5（叠加封顶 1.0）|

**指数分档**：
- **0–3 像真人**：偶有小瑕，可发。
- **4–6 半成品**：能看出机器痕迹，按定位清单改重点项。
- **7–10 浓 AI 味**：一眼假，整章过 `deslop` + `human`。

> 评分是**相对参照**，不是考试。重点永远是"扣分在哪几项、具体哪几句"，给作者可下刀的位置。

## 原则
- **先定位后评分**：分数服务于"改哪里"，不是为了打分。
- **结合平台**：番茄本就短句多、节奏快——短句密集 ≠ AI，关键看**有没有波动、有没有碎句和长句穿插**。
- **检测 ≠ 修改**：本库出诊断，改交给 `deslop`（表面）/ `human`（深层失控感）/ `emotion`（情绪演出来）。

# 去 AI 味深度清单（Anti-AI Checklist）

> 比"AI 腔"更隐蔽的几种破绽，专治读者一眼看出"这是 AI 写的"。供 `deslop` / `expand` / `dialogue` 调用。
> 两个最高频、最该删的病：**无缘无故的修辞** 和 **跨度大/不搭的引用**。

---

## 一、无缘无故的修辞（头号病）

AI 爱"为修辞而修辞"——加比喻、排比、华丽形容，但对剧情/情绪/画面毫无帮助，只是显得"文艺"。爽文里这是减分项。

**判断法**：每一个比喻/形容，问一句——「它让读者更懂、更爽、更有画面了吗？」没有就删。

- ❌「她的心情如同秋日里飘零的落叶，又似深海中无声的叹息，泛起层层涟漪。」（三个比喻，零信息，拖节奏）
- ✅「她笑了笑，没说话。」（克制，留白，更戳）

- ❌「愤怒像火山一样在他胸中喷涌，又如潮水般席卷了他的理智。」
- ✅「他攥紧了拳，指节发白。」（一个具体动作 > 两个比喻）

**规则**：
1. 一段最多一个比喻，且必须服务情绪/画面。
2. 爽文优先"具体动作/细节"，不优先"比喻"。
3. 删掉所有"为了好看"的排比和叠喻。

---

## 二、跨度大 / 不搭的引用（语域不一致）

AI 常突然引一个**和故事调性不搭**的东西：通俗爽文里冒出古诗典故、学术概念、西方哲学、或不合身份的网络梗。你写得朴实老成，它却引个怪东西——**本质是"语域(register)不一致"**。

**判断法**：问两句——
1. 「**这个叙述者/这个角色**，会知道、会这么说吗？」（市井女主不会随口引黑格尔）
2. 「这个引用，和**全文的语气**是一个频道吗？」（朴实白描里别塞华丽典故）

- ❌（通俗都市文里）「他的背影消失在街角，恰如《追忆似水年华》中那不可追回的时光。」（跨度太大，出戏）
- ✅「他头也不回地走了。我站在原地，看着那个背影越来越小。」

- ❌（古风文里）「公主气场全开，仿佛人间清醒的 MBTI 之王。」（现代梗污染古风语域）
- ✅「公主眼皮都没抬：『退下。』」

**规则**：
1. **先定语域**（朴实/华丽/古风/现代/冷感/俏皮），全文统一，不中途跳。
2. 通俗爽文：少引经据典，少掉书袋；要引也引读者秒懂的。
3. 古风别用现代网络梗；现代别硬塞古诗酸文；第一人称市井叙述者别用超出其认知的概念。
4. 宁可"老陈朴实"，不要"花哨不搭"——**风格一致 > 单句出彩**。

---

## 三、show, don't tell（番茄铁律）

少喊情绪，用**动作+细节+对话**显情绪。

- ❌「他很愤怒。」→ ✅「他拳头攥得咯咯响，指节发白。」
- ❌「她内心悲愤交加，两行清泪滑落。」→ ✅「她气得浑身发抖，眼泪唰地掉下来。」
- ❌「我很伤心，八年感情成了笑话。」→ ✅「我扯了扯嘴角，挤出一个笑。」

虐点尤其要克制：成年人的失望是"我累了""我侧开了身"，不是嚎啕大哭。

---

## 四、大白话 + 短句（手机阅读友好）

- 拒绝文艺修辞、生僻字、复杂长句。
- 多用口语词。段落 ≤ 3 行，一句一段也行；对话独立成行。
- ❌「她内心悲愤交加。」→ ✅「她气炸了。」

---

## 五、其它常见 AI 破绽（快查）
- **过度完整**：每句主谓宾齐全 → 真人有省略、停顿、打断。
- **解释过度**：把情绪/动机/已知信息全复述 → 留白。
- **副词喊情绪**："愤怒地说""开心地笑" → 删副词，用台词+动作。
- **转折词癖**："然而""与此同时""不仅如此" → 删或换口语。
- **一人一腔**：所有角色说话一个味 → 见 `dialogue-library.md`。
- **装饰符号癖** ⭐：正文/标题里撒 `✦ ✨ ◆ ● ▪ ★ ◇ ❖ ➤` 等符号当点缀或分隔 → **全删**。网文正文不用装饰符号；要分隔场景用**空行**或 `——`，章标题就是纯文字、不加符号不加颜色。

---

## 六、最招人烦的 6 种 AI 痕迹（重点查）

### 1. 重复（修辞 / 句式 / 用词）
AI 爱反复用同一个比喻、同一种句式、同一个词。通篇读一遍，重复的删或换。
- ❌ 三章里出现四次"像潮水般涌来" → 留一次，其余换具体写法。

### 2. 同义反复（换汤不换药）
一个意思用几个同义词说两三遍，以为在加强，其实是注水。
- ❌「他非常生气，他怒了，胸中怒火中烧。」→ ✅「他一脚踹翻了椅子。」（留最准的一个，最好换成动作）

### 3. 极端词 / 强化词滥用
"非常、极其、极大、无比、十分、特别、巨大"——AI 一激动就堆。**删掉九成**，用具体代替。
- ❌「他感到非常非常大的压力。」→ ✅「他三天没合眼，烟灰缸堆成了小山。」
- 程度靠**具体画面**给，不靠"非常""极"。

### 4. 莫名其妙的陈述句 / 强行升华
突然插一句和上下文没关系的空泛总结或感慨，像作文凑字。
- ❌「……他走进了雨里。生活总是充满了未知与挑战。」→ 删掉后半句。
- 凡是"这是一个……的时代""人生就是如此""命运总是……"这类空话，**一律删**。

### 5. 标点：句号当顿号 / 句号碎句癖
该用顿号、逗号的地方用句号一刀刀切开。
- ❌「他买了苹果。香蕉。橘子。」→ ✅「他买了苹果、香蕉、橘子。」
- 并列用**顿号「、」**，半句停顿用**逗号**，句号留给真正说完的一句。
- 注：番茄"短句独立成段"是手法，但**标点本身要用对**——成段 ≠ 滥用句号。

### 6. 过度对仗 / 段落雷同
上下两段字数差不多、行数一样、结构镜像，整齐得发假，一看就是机器排的。
- 改：**长短段交错**，故意打破对称；该长的长、该短的短，让段落参差不齐才像人写的。

---

## 七、中文标点规范（行文别用错/滥用标点）

AI 常把标点用错或用偏，一眼露馅。中文行文守这几条：

1. **省略号用「……」**（六个点，一个标点，占两个字位），**不要** `...`（三个英文点）、不要 `。。。`。
   - ❌ 他愣了一下... / 他愣了一下。。。 → ✅ 他愣了一下……
2. **破折号「——」少用**：AI 极爱滥用破折号，是高频破绽。能用句号/逗号就别用破折号；一段里别反复出现。
3. **引号**：对话和引语用**双引号「"」「"」**为主；**单引号「'」「'」只用于引号里再套引号**（嵌套），不要平白用单引号当主引号。
4. **不用「【】」于正文**：这种方括号正文一般没人用。**例外**：系统流的系统弹窗提示可用（如「【叮，系统激活】」）。其余场景一律不用 【】。
5. 并列用**顿号「、」**，不是逗号也不是句号（见第六节）。

## 去 AI 味自检（改稿过一遍）
- [ ] 有没有"为修辞而修辞"的比喻/排比？删。
- [ ] 有没有和全文语气不搭、跨度过大的引用/典故/梗？删或换。
- [ ] 语域全文一致吗？（朴实就一直朴实）
- [ ] 情绪是"喊"出来的还是"演"出来的？改成演。
- [ ] 句子够短、够白话吗？段落 ≤3 行吗？
- [ ] 有没有重复的比喻/句式/词？有没有同义反复（一个意思说两遍）？删。
- [ ] "非常/极其/极大"这类极端词是不是堆太多？删九成，换具体画面。
- [ ] 有没有莫名其妙的空泛陈述句/强行升华？删。
- [ ] 标点对吗？并列用顿号「、」不是句号。
- [ ] 段落是不是太对仗工整、长度行数雷同？打破对称，长短交错。
- [ ] 正文/标题里有没有 ✦✨◆★ 等装饰符号？全删，分隔用空行或"——"。

# 爆款拆解案例库（Case Studies）

> 真实爆款的结构拆解，供 `deconstruct`（对标学结构）和 `coach`（教学）调用。
> **学结构、学手法，不抄原文**。

---

## 案例 1：《等过风雨，也曾等你》（女频 · 追妻火葬场 · 信息差打脸）

### 爆款公式
**信息差 + 追妻火葬场 + 打脸反转 + 男二上位**，爽虐结合。

### 开篇钩（场景+人物+事件，一句立冲突）
> 闺蜜的婚礼上，手捧花被一个女孩抢到，又弹了一下，落进我怀里。

- 一句话同时给：**场景**(婚礼)、**人物**(女主/男主/女配)、**事件**(捧花落到女主手里，男主却转手给女配)。
- 效果：读者一眼懂"这对情侣有问题"，立刻追读。
- **手法**：开篇不铺垫，用"场景+人物+事件"三要素直接砸冲突。

### 五类钩子层层递进
信息差(读者知女主下周结婚、男主不知) + 悬念(男主以为婚礼是和自己) + 情绪(男主轻慢 vs 女主平静) + 对比(对女主/女配双标) + 倒计时(十月二十八号)。详见 `hook-library.md` 第五节。

### 虐点设计（6 个，层层磨碎八年期待）
公开敷衍 / 偏爱忽视对比 / 未来规划落空(画饼) / 态度双标 / 自我价值否定 / 回忆现实反差。详见 `trope-library.md` 第四节。
- 核心：用男主的"不作为+轻慢+双标"，把女主八年期待一点点磨碎。

### 爽点设计（虐越深爽越足）
女主清醒离场(不哭闹) → 信息差打脸(所有人以为她嫁男主，其实早卖婚房另嫁) → 男主追悔(备好西装车队却没接亲地址) → 女配落空 → 女主穿婚纱比从前更美。

### 三个可复用手法（模仿要点）
1. **开篇"场景+人物+事件"**，别铺垫废话。
2. **虐点用细节代替喊口号**：不写"我很伤心"，写"我扯了扯嘴角，挤出一个笑。秦奈不知道，我下周也要结婚了，和别人。"
3. **爽点用男主信息差打脸，代替女主自嗨**：让男主活在自己认知里("觉得她只是闹脾气，让兄弟准备车队")，读者知道真相 → 打脸比"他后悔了"爽十倍。

### 真人质感要点
- 渣男台词要现实："别急，以后给你办个完美婚礼""别小心眼了"——不是"我不爱你了"。
- 女主情绪克制成年化："我累了""我侧开了身""我把文件袋推到门缝底下"。
- 加真实社交场景：闺蜜吐槽、兄弟起哄，不悬浮。

### 可直接改写的模仿版开头（原创范式，非抄原文）
```
闺蜜的婚礼上，手捧花被伴娘抢了，又弹了一下，落进我怀里。
全场起哄炸开，所有人都看向我和陈屿。
我攥着捧花的手在抖，等他像偶像剧里那样当众说一句"我娶你"。
可他只是笑着抽走花，转身递给身边的实习生，声音温柔得像哄小孩：
"她第一次参加婚礼，让她沾沾喜气。乖，咱们等下一次。"
我扯了扯嘴角，没说话。
陈屿不知道，没有下一次了。
我的婚礼，就在下周六。
```

# 男频女频分流指南（Channel Guide）

> 男频和女频是两套不同的"爽"逻辑。写之前先定频道，节奏、爽点、人物、情感配比全不同。
> 本库供所有写作技能在判断"该往哪个方向写"时调用。

---

## 一、核心差异

| 维度 | 男频 | 女频 |
|---|---|---|
| **底层爽点** | 实力升级、资源扩张、打脸对手、地位攀升（我变强、掌控全局） | 情感拉扯 + 独一份偏爱、女主自我成长、仇人报应（被偏爱 + 自我圆满） |
| **代入方式** | 成为强者（向上） | 被珍视/掌控全局（向内 or 俯视） |
| **剧情主线** | 世界线推进，从弱小→登顶，世界观越写越大 | 人际关系线推进，CP 磨合、误会反转、双向奔赴 |
| **开篇逻辑** | 主角受辱 → 金手指落地 → 当场打脸 | 女主身陷困境（被背叛/穿书/被害）→ 男主登场/开启自救线 |
| **金手指** | 硬外挂（系统/功法/空间） | 软外挂（身份/重生/读心/空间） |
| **冲突** | 实力冲突、势力争斗 | 情感冲突、身份冲突、家族博弈 |
| **反派** | 挡路的强者 | 抢爱的、害我的、看不起我的 |
| **爽点节奏** | 打脸→升级→更大的脸 | 误会→反转→双向奔赴/打脸白莲 |
| **结局诉求** | 站上巅峰、一览众山小 | 圆满的爱 + 体面的赢 |

---

## 二、男频主要类型

- **玄幻/修真**：升级流，宗门、功法、天才打脸。番茄/起点主力。
- **都市**：战神归来、赘婿、医武、扮猪吃老虎。
- **系统流**：签到、任务、数值，番茄爆款集中地。
- **历史/种田**：策略、经营、争霸。
- **科幻/末世**：空间、进化、求生。

**男频铁律**：主角要有**主观能动性**，金手指是放大器不是替身；爽点要**升级**不能原地踏步。

---

## 三、女频主要类型

- **古言/宅斗宫斗**：身份、算计、打脸白莲与恶毒女配（作者素材库主场景）。
- **现言/豪门**：虐恋、重生复仇、打脸渣男渣女。
- **穿书/快穿**：穿进剧情改命，反套路。
- **纯爱/甜宠**：情绪流，双向奔赴。

**女频铁律**：情感线是主菜，实力线是配菜；"被偏爱"和"掌控感"是核心爽点；打脸要**体面、解气、不掉价**。

---

## 四、女频爽点的特殊性（结合作者素材）

作者素材库中的开篇范例（穿越女斗世家、白绫挂断、替身逆位、挡刀反转）体现女频典型爽感：
- **弱势开局 + 不认命**：身份弱（穿越女/替身/庶女）但姿态强（决绝反抗）。
- **关系反转**：替身→正主、网恋对象→恶霸本人，靠**关系的意外**制造钩子。
- **情绪解气**：女频的"打脸"更重情绪解气（让瞧不起我的人后悔），而非单纯实力碾压。

---

## 五、写作前必答三问

1. **男频还是女频？** → 决定爽感逻辑。
2. **哪个细分类型？** → 决定套路池（调 [trope-library.md](trope-library.md)）。
3. **投哪个平台？** → 决定节奏密度（调 [platform-profiles.md](platform-profiles.md)）。

# 连贯性与逻辑检查库（Continuity & Logic）

> 长篇崩盘不是"写不下去"，是**前后矛盾、时间线乱、资源凭空变、身份忽暴忽藏**——读者一眼看穿，弃书。
> AI 写长篇的规律：**50 章正常，100 章开始崩，300 章彻底失忆**。本库是防失忆的检查协议。
> 供 `continuity` / `review` / `expand` / `memory` 调用。与 `project-memory.md` 配套：档案是"记忆"，本库是"核对"。

---

## 一、六条逻辑链（逐章必查）

每写完一章，对照档案（`01_人物.md` / `03_伏笔追踪.md` / `07_时间线.md` / 战力表）核对这六条：

### 1. 人物行为合理性
人物这一章做的事，和**他的性格、动机、当前处境**对得上吗？
- ❌ 主角被全城通缉 → 下一章大摇大摆逛街买包子。
- ❌ 一向谨慎多疑的人，毫无理由地轻信陌生人。
- 查：行为有没有动机支撑？和人设档案的"性格3词"冲突吗？被金手指牵着走、没有主观能动性也算病。

### 2. 剧情因果链
这一章发生的事，**前面有没有因？后面有没有果？**
- ❌ 反派突然知道主角的秘密，但前文没有任何"他怎么得知"的铺垫。
- ❌ 一个关键道具/线索用完就消失，再也没人提。
- 查：新出现的设定有没有铺垫（防"突然冒出"）？挖的坑有没有记进伏笔表（防"挖了不填"）？

### 3. 资源变化（最容易穿帮）
钱、灵石、道具、兵力、人脉——**只能按剧情增减，不能凭空跳变**。
- ❌ 上一章穷得吃不起饭 → 下一章随手掏出十万灵石。
- ❌ 物资/储备数量前后对不上（昨天还剩 3 颗丹药，今天吃了 5 颗）。
- 查：本章资源进出有交代吗？和档案"重要道具/资源"一致吗？大额变动要落进度档。

### 4. 身份暴露状态
谁知道主角的**真实身份 / 实力 / 秘密 / 马甲**？这是一张需要持续维护的名单。
- ❌ 主角藏得好好的马甲，某章被某人识破 → 几章后那人又像完全不知道一样对待他。
- ❌ "无人知晓的秘密"被随口说出来，却没人反应。
- 查：维护一张"知情人名单"（每个秘密谁知道）。扮猪吃老虎流尤其要管——什么时候露、对谁露、露多少。

### 5. 伤势 / 状态延续
受伤、中毒、被废、虚弱、闭关——**状态要持续，不能下一章就活蹦乱跳**。
- ❌ 这章重伤垂死 → 下一章生龙活虎大战三百回合，中间没有任何疗伤交代。
- ❌ 中了"七日内必死"的毒，第八章还没解毒也没死。
- 查：负面状态有没有"恢复过程"？设定的期限（X天解毒/Y天出关）到点兑现了吗？

### 6. 战力 / 等级一致
谁强谁弱、境界高低，要符合**已建立的战力体系**。
- ❌ 上一卷被 A 一招秒，这一卷实力没明显提升却能跟 A 打平。
- ❌ 配角越级暴打 BOSS，破坏了体系的可信度。
- 查：对照战力对照表（见 `project-memory.md`）。境界跨度要有"为什么变强"的交代。反派强度要撑得起主角的胜利。

---

## 二、时间线核对（AI 最爱忘）

AI 写"三天后""五分钟后""两个月后""第二天"全凭手感，写到后面作者自己都晕。

**维护一条主时间线**（存进 `07_时间线.md`，格式见 `project-memory.md`）：用 Day1 / Day2 / 第3月 这类**绝对锚点**，别只用"几天后"这种相对量。

逐章核对：
- **本章经过了多久？** 落到时间线上是哪一天。
- **跨度合理吗？** "三天后"主角就突破了原本要闭关一年的瓶颈？
- **并行线对得上吗？** A 线过了一个月、B 线只过了三天，两线汇合时矛盾吗？
- **季节/年龄/节日**：写到换季、过年、生日，和时间线推进对得上吗？
- **设定的期限**：闭关 X 天、约定 Y 月后决战、毒发倒计时——到点要兑现，别忘。

> 一句话：**所有"X 时间后"都要能换算成时间线上的一个点。** 换不了，就是要出事的地方。

---

## 三、长篇失忆的三个临界点（重点防）

| 章数 | 症状 | 重点查 |
|---|---|---|
| ~50 章 | 基本正常，偶有小漏 | 资源、伤势这类细节 |
| ~100 章 | 开始崩：配角名字记混、设定微漂移 | 人物状态、战力体系、知情人名单 |
| ~300 章 | 彻底失忆：早期伏笔全忘、时间线乱、人设漂移 | **全部六条 + 时间线 + 伏笔总表**，建议每卷末做一次全量核对 |

**防线**：① 档案是单一事实来源，写前读、写后更（见 `skills/memory`）；② 每章过六条逻辑链；③ 每 10–20 章 / 每卷末做一次全量连贯性体检。

---

## 四、检查输出格式

```
【连贯性体检 · 第 N 章（或第 X 卷）】
时间线：本章 = Day__ ｜跨度 __ ｜与主线 一致 / 冲突(说明)
六条逻辑链：
  行为合理性  ✓/✗  问题在第__段：……
  因果链      ✓/✗  ……
  资源变化    ✓/✗  ……
  身份暴露    ✓/✗  知情人名单变化：……
  伤势/状态   ✓/✗  ……
  战力一致    ✓/✗  ……
矛盾清单：① 第_段 与 第_章/档案 冲突 → 改法
待兑现：到期未兑现的期限/伏笔（毒/闭关/约定）
档案待更新：人物状态 / 资源 / 时间线 / 知情人 哪些要写回档案
```

## 原则
- **定位到段/章**，指明和"哪一章 / 哪条档案"冲突，不空泛。
- 每个矛盾给**改法**（改本章 or 回改前文 or 补一句交代）。
- 区分轻重：主线/战力/身份矛盾是硬伤先改；次要情绪线留白不算病（见 `human-texture.md`"不闭环≠烂尾"）。
- 查完提示把变化**写回档案**，否则下一章继续漂。

# 暗黑/灰度风格 + 双版本输出（Dark Style）

> 和 `warm-realism.md`（温暖）相对的另一极：**暗黑、灰度、极端**。
> 也定义"爽文版 / 高级暗黑版"**双版本输出**。供 `expand` / `draft` / `spark` 调用。

---

## 一、暗黑/灰度风格要点

- **情绪极端**：恨、执念、疯、孤注一掷——敢写到顶，不和稀泥。
- **人物复杂/灰色**：没有纯好纯坏；主角也可能自私、狠、有隐秘欲望（呼应 `human-texture.md` 的"私货感/不理性"）。
- **事件更黑暗**：背叛、复仇、算计、人性崩塌；后果真实、有代价。
- **逻辑留空**：不把一切解释干净，留冷感和余味（次要线不闭环，主线坑仍回收）。
- **黑暗/开放结局**：可不圆满，可反转到底，可留一刀。
- **张力靠克制**：越黑的内容越要冷静地写，不嘶吼、不渲染，更瘆人。

> 注意红线：暗黑 ≠ 三观崩坏/违规。极端的是情绪和命运，不是价值观导向；平台审核红线仍守（违法犯罪不美化、不渲染）。

---

## 二、爽文版 vs 高级暗黑版（双版本）

| 维度 | 爽文版 | 高级暗黑版 |
|---|---|---|
| 情绪 | 偏高、解气、明确 | 极端、压抑、灰色 |
| 人物 | 冲突适中、立场清晰 | 复杂、灰度、有隐秘动机 |
| 反转 | 明确、爽快 | 黑暗、意外、留刀 |
| 逻辑 | 基本闭环 | 故意留空、冷感 |
| 结局 | 高光/圆满 | 开放/黑暗/反转 |
| 适合 | 番茄/下沉/追读 | 知乎盐选/进阶/重口读者 |

---

## 三、双版本输出怎么做
当用户要"两个版本"时：
1. 先写**爽文版**：强化主角高光、爽点、明确反转，情绪偏高。
2. 再写**暗黑版**：同一核心故事，情绪推到极端、人物加灰度、事件更黑、逻辑留空、结局可黑。
3. **各自标注版本名**，保持原人物/故事核心，字数按要求（短篇常 20000–30000，也可更短，见 `manuscript-format.md`）。

> 不要求时默认单版本；用户说"给我爽文版和暗黑版两稿"才出两版。

---

## 四、和句式/人味配合
暗黑版尤其要：句式长短不均、留停顿（`sentence-rhythm.md`）+ 非理性/不闭环/私货（`human-texture.md`）——冷、断、狠，才是"高级暗黑"而非"中二黑暗"。

---

- 温暖那一极 → `warm-realism.md`、`skills/warmth`
- 句式节奏 → `sentence-rhythm.md`　人味/失控 → `human-texture.md`
- 成稿/导出 → `skills/expand`、`skills/draft`

# 台词与对话库（Dialogue Library）

> 对话是"AI 味"最大的重灾区，也是网文名场面的核心。本库供 `dialogue` / `expand` / `deslop` 调用。

---

## 一、AI 对话腔的病（先认清）

| 病症 | AI 写法 | 问题 |
|---|---|---|
| **太书面** | "我对此感到十分愤怒。" | 没人这么说话 |
| **太完整** | 每句主谓宾齐全、逻辑严密 | 真人说话有省略、有停顿、会打断 |
| **没潜台词** | 心里想啥嘴上说啥 | 没有言外之意，无张力 |
| **人人一个腔** | 主角反派配角说话一个味 | 角色没声音、分不清谁说的 |
| **解释过度** | 对话里塞设定、复述已知信息 | 假、出戏、像在念说明书 |
| **情绪靠形容词** | "他愤怒地吼道" | 该靠台词本身和动作，不靠副词 |

**一句话**：AI 对话像"书面陈述"，网文对话要像"真人在交锋"。

---

## 二、好台词 5 原则

1. **口语化**：短句、省略、有语气词，敢不完整。"就这？"胜过"原来你的实力也不过如此。"
2. **有潜台词**：话里有话，言外之意 > 字面意思。
3. **符合人设声音**：每个角色有自己的说话方式（用词/句长/口头禅/温度）。
4. **留白**：不把话说尽，不把情绪解释完，让读者自己品。
5. **推进或交锋**：每句对话要么推进剧情、要么暴露性格、要么制造张力，不闲聊。

---

## 三、台词类型库（按场景取用）

### 打脸 / 反击台词（男频女频通用）
- 轻蔑反问："就这？" "你也配？" "说完了？"
- 淡定碾压："我尽力了，没想到还是一拳。"
- 反向羞辱："你说得对，所以你输了。"
- 要点：**越短越狠**，配围观反应（调 `xiushi-ci.md`）。

### 装13 / 强者台词
- 不解释、不辩白，越淡越唬人："随便。" "无所谓，反正你们也拦不住。"
- 反差落差：平静说出炸裂信息。
- 忌：长篇大论解释自己多强（一解释就 low）。

### 反派台词（要压迫感）
- 文雅的威胁最瘆人："别紧张，我只是想看看你绝望的样子。"
- 不动声色的掌控："你以为你在选择？"

### 女频情感台词
- 虐：克制比嘶吼更虐。"你走吧。"（不说舍不得，更舍不得）
- 甜：偏爱藏在细节。"别人不行，你可以。"
- 拉扯：欲说还休、口是心非。

### 日常 / 插科打诨
- 调节节奏，体现关系亲疏，用得好增加角色魅力。

---

## 四、金句设计（可截图传播的台词）

好金句 = **短 + 狠 + 反差/哲理**，读者会截图发评论区。
- 公式：日常词 + 颠覆性断言。"成年人的崩溃，都是静音模式。"
- 网文场景金句：在爽点峰值甩一句立 flag 或定调的话。
- 别硬凑：金句一章一两句就够，多了显做作。

---

## 五、人物声音区分（让读者不看名字也知道谁在说）

给每个主要角色定 3 个说话特征：
- **句长**：急性子短句蹦，沉稳的长句缓。
- **用词**：粗人用糙词，书生用文词，反派用冷词。
- **口头禅 / 温度**：有人爱反问，有人爱沉默，有人嘴贱。

写对话前先想：这句话，**只有这个角色会这么说**吗？

---

## 六、去 AI 对话腔技巧（改稿用）
- 删掉对话里的"复述已知信息"和"念设定"。
- 把"主谓宾完整长句"打碎成口语短句、加省略和停顿。
- 把"愤怒地说/开心地说"删掉，改用台词本身+一个动作。
- 给不同角色换不同句式，消除"一个腔"。
- 加潜台词：让角色说"反话/半句话/答非所问"。

# 英文网文出海库（English Web Fiction）

> 出海写英文不是把中文翻译过去——**中文网文卖"剧情爽点"，英文网文卖"情绪钩子"**。
> 中文读者愿意看你解释为什么爽；英文读者要被**直接扔进故事**，自己感受。
> 目标：让 AI 写的英文读起来像当地作者，而不是"中国网文翻译腔"。供 `english` 调用。

---

## 〇、总开关：Native English Webnovel Mode（默认）

> 写英文时**默认开启**这一条——它改的是**叙事思维**，比几十条零散规则都重要：
>
> - **默认按欧美母语网文风格输出，不是中文翻译过去。**
> - **优先动作、对话、冲突、悬念；降低解释、降低抒情、降低文学化描写。**
> - **提高人物真实感和口语化表达。**
> - **让读者沉浸在事件中，而不是旁观作者讲故事。**
>
> 区别：不是"中文思维→翻译成英文"，而是"英文读者本来就会这样表达"。

### 十条核心规则（Native English）
1. **禁止翻译腔**：情绪用动作/身体反应，不直译中式比喻。
   ❌ `Her heart felt like it was cut by knives.` / `Flames of anger burned in his chest.`
   ✅ `My chest tightened.` / `He clenched his jaw.` / `She looked away.`
2. **Show > Tell**：不解释情绪。❌ `She was sad/nervous/angry.` ✅ `Her hands shook.` `She stared at the floor.` `He slammed the door.`
3. **句长 70/20/10**：70% 短句、20% 中句、10% 长句，避免连续长句。
   > `I stopped. / The room was empty. / Too empty. / Then I heard breathing behind me.`
4. **禁止文学堆砌**：避免 `moonlight danced`、`crystal tears`、`breathtaking beauty`、`cold smile` —— 直接写动作。
5. **每 200 字必须推进**：新信息/新冲突/新秘密/新反转/新危险，禁止无意义描写。
6. **钩子优先**：章节开头不要环境描写，开头给 危险/秘密/死亡/身份反转/禁忌关系/倒计时/悬念。
7. **对话像真人**：❌ `I am extremely disappointed in your behavior.` ✅ `What the hell was that?` —— 口语 > 书面。
8. **母语短句节奏**：多用 `Maybe.` `Fine.` `Seriously?` `No way.` `Wait.` `Damn.` 增加真实感。
9. **减少副词**：避免 `slowly / carefully / suddenly / angrily`，用动作表达。
10. **每章结尾留钩**：读完让读者想"再看一章"。
    > `The message appeared. / WELCOME BACK, PLAYER. / Level 1 begins.`

---

## ⓪+ 中国爽文 vs 英文爽文（结构差异，最易忽略）

| | 中文爽文 | 英文爽文 |
|---|---|---|
| 结构 | **情绪 → 爽点**（直接给爽） | **谜团 → 情绪 → 爽点**（先制造疑问再给爽） |
| 例 | "前男友后悔了" → 直接爽 | 前男友**为什么**后悔？他隐瞒了什么？那晚发生了什么？→ 先疑问，再爽 |

> 英文读者要先被"为什么"勾住，再给情绪和爽点。**先埋谜，后给糖。**

### 爽感包装：身份碾压 vs 个人成长
- **中文爱"身份碾压"**：首富、战神、龙王、神医（一出场就高位）。
- **英文爱"个人成长"**：**Survivor（幸存者）/ Underdog（弱者逆袭）/ Outcast（被排挤者）/ Chosen One（被选中者）**（从低到高、自己挣来）。
- 同样是爽，**包装方式不同**：英文要让读者陪主角"挣"，而不是开局就赢。

## 二、英文 vs 中文 底层逻辑（最重要）

| | 中文网文 | 英文网文 |
|---|---|---|
| 卖点 | 剧情**爽点** | 情绪**钩子** |
| 读者关注 | 世界观、爽感、文学性 | 人物、感情、情绪 |
| 节奏 | 可铺垫 | 前 100 词必须有事发生 |
| 解释 | 愿意看"为什么爽" | 讨厌解释，直接动作，自己理解 |
| 开头 | 「血色残阳下，少女眸光冷冽。」 | `I never expected my life to change that night.` / `The dead man opened his eyes.` |

> 一句话：**英文读者更喜欢被直接扔进故事里，让他自己感受**，而不是你告诉他该感受什么。

---

## 三、英文读者讨厌的 3 件事（AI 最容易翻车）

### 1. 长段落
英文一大段就弃书。要**短，非常短**。
- ❌ 一大段叙述 + 回忆 + 心理。
- ✅ `I looked outside. / Rain hit the window. / The memory came back.`

### 2. 解释
不要写"她之所以这么做是因为……"，直接动作。
- ❌ `She was angry because...`
- ✅ `She slammed the door.` （让读者自己理解）

### 3. 堆形容词
- ❌ `Her beautiful, gorgeous, stunning eyes...`
- ✅ `Her eyes stayed on me.` （简单）

---

## 四、英文排版（必须遵守）

- 章标题：`Chapter 1`（不是"第一章"）。
- 段落短、段间空行。
- **对话必须换行，一句一行**：
```
"Who are you?"

I froze.

"Answer me."
```
- ❌ 绝不把对话和动作挤在一行：`"Who are you?" I froze. "Answer me."`（阅读体验差）。

---

## 五、英文节奏：前 100 词有事发生
- ❌（中文式铺垫）女主起床→洗脸→吃饭→出门→撞见男主。
- ✅ 第一句直接开始：`The dead man opened his eyes.`

---

## 六、英文最爱的钩子（开篇第一句）

- **身份反转**：`My husband murdered me. Then I woke up three years earlier.`
- **危机/倒计时**：`The timer on my arm showed 00:59.`
- **秘密**：`Everyone in town knew the truth except me.`
- **禁忌关系**：`I accidentally married my enemy.`

---

## 七、英文情绪表达：用身体反应，不喊抽象
- ❌ `Her heart was breaking.`（≈ 心如刀割，英文很少这样写）
- ✅ `I couldn't breathe.` / `My hands shook.` / `My throat went tight.`
> 和中文的 show-don't-tell 一脉（见 `show-emotion.md`），但英文更偏**生理反应**、更克制。

---

## 八、AI 写英文最容易暴露的 3 个破绽

1. **太正式**：`Therefore... / However... / Furthermore...` → 像论文/学术。改口语连接或删。
2. **太工整/机械**：`He smiled. She smiled. He walked. She walked.` → 句式镜像、节奏机械。打破对称（见 `sentence-rhythm.md`）。
3. **过度文学**：`Moonlight danced gracefully upon the crystal lake...` → 网文读者直接跳。要直白、要快。

- **短篇/广告流（TikTok/Reels）**：强钩子、强反转、超短段落。

你的**反转意识、钩子意识、快节奏**很适合 Wattpad 短篇 / Webnovel / 短视频小说流。把"解释爽点"改成"直接扔进情绪"，就立刻接近当地作者，而不是翻译腔。

# 同人文合规指南（Fan-fiction Compliance）

> 同人是热门赛道，但**版权红线**踩不得。本库供 `fanfic` 创作与版权检测技能调用。
> ⚠️ 免责：以下为创作实务经验，**非法律意见**；商用变现前请咨询专业人士或取得 IP 方书面授权。

---

## 一、版权法律红线（重中之重）

1. **商用避雷**：没有 IP 方**书面授权**，任何平台都**不能签约变现、不能收费**。一旦盈利，易被原作方起诉（参考江南《此间的少年》侵金庸案）。
2. **公版 IP 优先**（最安全）：作者去世超 50 年、版权进入公有领域的作品可自由二创——**西游记、三国演义、红楼梦、水浒传、封神演义、聊斋**等。
3. **安全改写技巧**（降低侵权风险）：
   - 原作角色 **≤ 3 人**，且大幅改写人设/身世/世界观
   - **完全抛弃原作主线**，只借用名字，故事 100% 原创
   - 不照搬原著标志性桥段、设定体系

---

## 二、分平台同人放行规则

| 平台 | 同人政策 | 能否变现 |
|---|---|---|
| **晋江** | ✅ 专属同人分区，授权 IP / 原创衍生可签约 | **国内唯一合法变现** |
| **番茄** | ❌ 商用同人全封禁 | 免费发文也有下架风险 |
| **UC** | ❌ 基本不接收 IP 同人 | 仅原创过审 |
| **知乎盐选** | ❌ 不允许版权 IP 同人签约 | 同人只能发专栏无稿费 |
| **海外 Claude / Webnovel 端** | 海外 IP 同人法律较宽松 | **蓝海**：海外华人写影视/网文同人 |

### 晋江同人硬性规则（唯一合法区，务必记牢）
- 引用原著原文 **≤ 全文的 1/10**
- 单篇引用原文**不超过 3000 字**
- **不能照搬原著剧情，必须原创主线**
- 超量 = 违规锁文

---

## 三、内容审核红线（全平台统一）

- 严禁 **OOC 过度抹黑**原作角色、毁原著核心人设
- 禁止未成年人早恋、色情改编
- 禁止歪曲原作世界观、三观不正

---

## 四、海外同人蓝海（你的目标市场）

- 海外对同人创作（fan-fiction）法律环境相对宽松，欧美有成熟同人文化（AO3 等）。
- 海外华人写**影视/动漫/网文同人**用 Claude 创作，是本插件的差异化场景之一。
- 即便如此，**商用仍建议走授权或公版**，避免跨境维权风险。

# 金手指流派库（Golden Finger Types）

> 金手指（外挂）是男频网文的发动机，也是区分"爽文"与"流水账"的关键。
> 一个好的金手指 = **明确的获取门槛 + 清晰的成长曲线 + 与主线绑定的代价**。
> 本库供 `goldfinger` / `idea` / `outline` 技能调用。

---

## 一、主流金手指流派

| 流派 | 机制 | 爽点来源 | 代表 | 风险 |
|---|---|---|---|---|
| **系统流** | 任务发布 + 数值奖励 | 确定性反馈、升级闭环 | 番茄爆款主力 | 易写成任务流水账 |
| **签到流** | 每日/地点签到领奖 | 零成本白嫖、复利滚雪球 | 《我在地球当神明》类 | 前期爽后期乏力 |
| **空间流** | 随身世界/灵田/商城 | 资源囤积、乱世安全感 | 种田、末世、女频通用 | 节奏易拖沓 |
| **模拟器流** | 推演人生/预演结果 | 信息差、试错无代价 | 《大奉打更人》模拟器 | 逻辑要自洽 |
| **词条流** | 给万物附加属性词条 | 组合 build 的策略爽 | 近年新兴 | 数值要平衡 |
| **复制/吞噬流** | 复制他人天赋/吞噬变强 | 无限叠加的成长感 | 《天赋复制》类 | 易破坏世界平衡 |
| **重生/先知流** | 携带未来信息重来 | 降维打击、弥补遗憾 | 都市、商战、玄幻 | 蝴蝶效应要处理 |
| **召唤流** | 召唤名将/英灵作战 | 名场面 + 卡牌收集 | 历史、玄幻 | 易沦为抽卡 |
| **赘婿/扮猪流** | 隐藏实力/身份 | 扮猪吃老虎、打脸 | 都市无敌文 | 装太久读者烦 |
| **无限流** | 闯关副本/恐怖游戏 | 智斗 + 求生 + 团队 | 《无限恐怖》后裔 | 设定门槛高 |

---

## 二、金手指设计四要素

1. **获取门槛**：怎么得到的？越独特越好，别"路边捡到"。绑定主角性格/遭遇更佳。
2. **成长曲线**：从弱到强的台阶要清晰，每一阶解锁新玩法，避免一步到位。
3. **使用代价**：无代价的外挂=无聊。设定冷却/反噬/资源消耗/道德取舍，制造张力。
4. **主线绑定**：金手指的来源/进化要和主线大谜团挂钩（谁给的？为什么是我？），后期才有戏。

---

## 三、金手指与节奏

- **上线时机**：番茄类第 1 章上线；起点类最晚不超过第 3 章。
- **首次爽点**：金手指上线后 1–2 章内必须用它解决一个具体困境（小试牛刀）。
- **防烂尾**：给金手指设"成长上限的悬念"，否则主角无敌后无戏可写。

---

## 四、避坑清单

- ❌ 金手指万能无代价 → 失去紧张感
- ❌ 获取方式随意（路上捡个戒指） → 读者无代入
- ❌ 升级全靠金手指、主角无主观能动 → 主角变工具人
- ❌ 与世界观脱节 → 设定割裂
- ✅ 金手指是放大器，放大的是**主角的选择和努力**

---

- 女频较少用硬金手指，多用"身份/系统/重生"软外挂，见 [channel-guide.md](channel-guide.md)
- 金手指带来的爽点元素见 [trope-library.md](trope-library.md)

# 钩子库（Hook Library）

> 网文的命脉是**留人**。读者在前 3 句、每章结尾、每个付费节点都在做"继续/退出"的决定。
> 本库收录可复用的钩子结构与实战范例，供 `hook` / `expand` / `idea` 技能调用。

---

## 一、开篇钩（黄金第一段）

网文开篇 = 在 3 句话内制造一个**信息差 + 危机/反差**，让读者必须往下看才能消解。

### 1.1 七种高转化开篇结构

| 结构 | 公式 | 适用 |
|---|---|---|
| **悬置危机** | 主角已身处绝境，但原因/出路未知 | 通用 |
| **身份反差** | "不过是个废物/穿越女/赘婿，却……" | 男频逆袭、女频宅斗 |
| **结局倒挂** | "等我醒来，剧情已经大结局很久了" | 穿书、无限流 |
| **替身/挡刀** | 主角替别人承受了命运的一击 | 女频、悲剧反转 |
| **金手指激活** | 第一句就给系统/外挂上线 | 男频爽文、番茄 |
| **日常爆破** | 平静日常被一条消息/一个电话击碎 | 都市、悬疑 |
| **反派开局** | 以反派/恶役视角开场，命运将倾 | 快穿、反派文 |

### 1.2 实战范例（女频·爽点开头）

> 以下范例来自作者素材库，体现"一句话立危机/立反差"的女频开篇手感。

1. **身份反差 + 实力悬殊**
   > 「不过是一个穿越女，怎么斗得过世家贵族培养了十几年的闺秀。」 ——《碎京华》
   - 钩点：弱势方 vs 庞然大物，读者想看她怎么翻盘。

2. **危机置顶 + 决绝姿态**
   > 「我把圣上赐我的白绫挂断了。」 ——《新政》
   - 钩点：赐死=死局，但"挂断"=反抗，一句话立人设立冲突。

3. **结局倒挂 + 信息差**
   > 「穿越后我隐世了十年，才发现剧情已经大结局很久了。」 ——《穿越后我成了男主的水母》
   - 钩点：主角错过全部主线，读者好奇"那现在怎么办"。

4. **挡刀 + 命运反转**
   > 「反派即将下场，而我却刚好穿来替他挡了一击。」 ——《我为反派挡刀后》
   - 钩点：替反派挡刀=违背常理的选择，制造关系张力。

5. **替身逆位**
   > 「替身文完美结局后，正主回来了。」 ——《流景》
   - 钩点：本该退场的替身成了主角，颠覆套路预期。

6. **日常爆破 + 巧合钩**
   > 「当被校园恶霸逼到墙角，我给网恋对象发了最后一条消息，结果恶霸的手机响了。」 ——《不是月亮》
   - 钩点：网恋对象=恶霸本人，巧合制造强烈反转预期。

### 1.3 实战范例（男频·爽点开头）

> 男频开篇核心：**屈辱/危机 + 金手指或实力反差**，3 句内立"逆袭预期"。下面按男频主要类型各给范例。

**玄幻/修真**
1. **实力倒挂（扮猪吃老虎）**
   > 「全宗门都以为我是个炼气期废物，没人知道我随手画的符能轰平一座山。」
   - 钩点：废物身份 vs 隐藏实力，等着看打脸。
2. **越级危机**
   > 「筑基期的我，被丢进了只有金丹真人才能活着出来的秘境。」
   - 钩点：必死困局 + "他凭什么能活"的悬念。

**都市**
3. **金手指激活 + 屈辱现状**
   > 「老婆给别人当备胎的第七年，我的签到系统终于醒了。」
   - 钩点：屈辱触底 + 外挂上线 = 逆袭预期拉满。
4. **赘婿/扮猪开局**
   > 「岳母把离婚协议拍在桌上时，没人知道这座城市一半的产业都姓我的姓。」
   - 钩点：被看扁的赘婿=隐藏巨佬，打脸预期。

**系统/签到（番茄主力）**
5. **第一章上外挂**
   > 「叮——宿主签到失败，因为你穷得连个能签到的地方都没有。系统沉默了。」
   - 钩点：系统第一句就上线 + 反差幽默，番茄最爱。

**历史/末世**
6. **悬置危机 + 谜团**
   > 「我睁眼时，正躺在自己的葬礼上，而棺材盖正在合拢。」
   - 钩点：活人参加自己葬礼，生死危机 + 重生/穿越谜团。

> 男频套路池（升级/打脸/越级/系统/扮猪）见 [trope-library.md](trope-library.md)；金手指流派见 [goldfinger-types.md](goldfinger-types.md)；男频用词偏"猛/爽/燃"的强化词与围观反应，见 [xiushi-ci.md](xiushi-ci.md)。

## 二、章尾钩（断章艺术）

每章结尾必须留一个"未完成"，读者才会点下一章。**严禁在情绪最高点收尾**——要在"即将揭晓/即将爆发"处断。

### 2.1 八种章尾钩

| 类型 | 手法 | 例 |
|---|---|---|
| **悬念钩** | 抛出谜题不解答 | "门开了，站在门口的人让她血液瞬间冻结——" |
| **危机钩** | 危险降临未化解 | "他扣动扳机的瞬间，主角才发现枪口对着的是自己母亲。" |
| **反转钩** | 真相即将颠覆 | "直到他翻开档案最后一页，才明白自己追查的凶手……就是自己。" |
| **打脸预告钩** | 嘲讽者即将翻车 | "众人哄笑时，主角只是淡淡抬手——下一秒，全场死寂。" |
| **关系钩** | 情感即将质变 | "她说'我们分手吧'，他却笑了：'你以为，你还走得掉？'" |
| **升级钩** | 实力/资源即将跃迁 | "系统提示音响起：'检测到宿主突破条件已满足。'" |
| **信息钩** | 关键情报半遮半掩 | "信封里只有一张照片，照片上的人，本该在三年前就死了。" |
| **倒计时钩** | 给出明确时限 | "距离碧落宗灭门，还有七天。" |

### 2.2 章尾钩铁律

- **一章一钩**：每章至少 1 个有效钩子，付费章节加倍。
- **钩在刀尖**：在动作/真相即将发生的**前一秒**断章，不要写完。
- **钩要兑现**：埋的钩必须在 1–3 章内回收，否则读者弃书。
- **大钩套小钩**：卷级大悬念（主线）下挂章级小悬念（支线），层层咬合。

---

## 三、付费节点钩（VIP 章 / 追读节点）

- **第 1 章**：3 句内立危机 + 立爽感预期。
- **第 3 章末（黄金三章收口）**：第一个小高潮 + 主线悬念落地，决定收藏。
- **入 V 前一章**：把最大悬念/最强爽点压到付费墙前一页，逼读者订阅。
- **每 10 万字**：一个卷级大爆点（打脸大佬/身份揭晓/势力翻盘）。

---

## 四、番茄黄金 300 字开头公式（免费流/短平快主力）

番茄/免费流开头 300 字定生死，公式：
```
开局低谷 + 强冲突 + 金手指 + 钩子
```
- **第一句**：直给困境（被弃/被追杀/被绿/病危/葬礼），不铺垫。
- **300 字内**：引爆冲突（打脸/背叛/生死危机）。
- **第一章末**：亮金手指（系统/重生/异能/祖传宝物）。
- **章尾**：留钩子（悬念/反转/新危机）。

**开头四禁忌**：❌ 慢热铺垫(写景/回忆/背景) ❌ 大段设定(世界观/等级) ❌ 多人扎堆(开篇≤3人) ❌ 长段落(手机阅读窒息)。

### 6 种万能开头模板（直接套）
1. **打脸逆袭**：被未婚夫当众甩耳光"你个废物配得上我？"我反手一巴掌："现在，你高攀不起。"
2. **重生复仇**：烧死前我看见丈夫搂着白月光笑。一睁眼，重回出嫁日——这次我要他们血债血偿。
3. **系统激活**：被逐出家族修为尽废，天降系统：【最强战神系统绑定】神级功法+千亿资产。
4. **穿书求生**：穿成虐文女配，原主刚被男主推下悬崖，我睁眼就见男主冰冷的脸："你又玩什么把戏？"
5. **危机悬念**：葬礼第三天，母亲把我拉进祠堂，指着族谱上我的名字——后面朱砂写着"已除籍"。
6. **情绪暴击**：产房门开，医生说"孩子保不住"，丈夫却把签字笔递给白月光："签了字，我们就结婚。"

## 五、五类追读钩子（贯穿全文拉好奇）

> 不止"信息差"，下面 5 类层层叠用，把读者好奇心拉满（女频追妻火葬场最常用）。

| 钩子类型 | 机制 | 示例 |
|---|---|---|
| **信息差钩** | 读者知真相、剧中人被蒙在鼓里 | "他不知道，没有下一次了——我的婚礼就在下周。" |
| **悬念钩** | 把期待拉满再瞬间反转 | 他以为婚礼是和自己，备好西装，兄弟却问"你和她的婚房怎么卖了？" |
| **情绪钩** | 角色态度/心境反差带共情 | 他把她的失落说成"小心眼"，她平静道"我累了，分手吧"。 |
| **对比钩** | 双标行为放大矛盾 | 对自己暧昧立刻澄清，对别人暧昧却放任；外人羞辱她，他不维护。 |
| **倒计时钩** | 明确时间节点制造紧张 | "十月二十八号，酒店订了，婚纱也选好了。" |

# 人味 / 失控感库（Human Texture）

> 比表面"AI 腔"更深的一层。**"AI 味"的本质不是"是不是 AI 写的"，而是"缺少人类的失控感"。**
> 一句话判断：**AI 写得像"说明书"，人写得像"情绪记录"。**
> 供 `human` / `expand` / `review` / `character` 调用。与 `deslop` 互补：deslop 治表面文字，本库治深层结构。

---

## 一、别人说"像 AI"，其实在说这 5 种阅读感受

### 1. 情绪太"平均"
每个人都"合理反应"，没有特别极端的情绪爆点。
👉 读者感受："都在反应，但没一个真的在痛、在疯。"
→ 人写的情绪有**偏心、失控、偏执**。

### 2. 冲突太"干净"
AI 爱写：A 误会 B → B 解释 → 解决。
真实好看的是：A 误会 B → **B 不解释** → C 插刀 → 误会升级 → 滚成别的事件。
👉 AI 缺少"**越滚越脏**"的冲突链。

### 3. 句子太"平均顺滑"
每句都顺、都读得懂、没有长短变化、没有突然卡一下的语气。
人写作有断句、有情绪停顿、有不完整表达。
👉 "**太顺**"反而成了 AI 特征。

### 4. 人物太"正常"
动机合理、行为符合逻辑，但**不极端**。
爆款人物会"明知道错还做"、会自毁、会偏执、会赌。
👉 **人味来自"不理性"。**

### 5. 缺少"私货感"
AI 尽量中立；人写作有偏见、情绪判断、不公平的评价。
👉 **中立 = 没味道。**

---

## 二、加"人味"的 3 个具体手法（照着加）

### ✅ 1. 插入"非理性行为"
- 明知道会输还要赌
- 当场翻脸、不解释
- 突然做一个极端选择
> 让角色做一件"逻辑上不该做、但情绪上必须做"的事。

### ✅ 2. 加一句"情绪不完整表达"
- "他没说完那句话。"
- "她突然不想解释了。"
- "那一刻他其实有点后悔，但没停。"
> 不把情绪说圆，留一个没说出口的缺口。

### ✅ 3. 允许"逻辑不完全闭环"
- 有些事没解释清楚
- 有些关系没收干净
> 人写作最不像 AI 的地方：不是所有线都收得整整齐齐。

---

## 三、核心判断标准
- ❌ 像**说明书**：一切都解释、都合理、都闭环、都中立。
- ✅ 像**情绪记录**：有失控、有偏执、有省略、有没收的线、有作者的偏心。

改稿自检一句话：**这段读起来像说明书，还是像一个人在记录自己的情绪？**

---

## 四、注意（别过头，失控是点睛不是发疯）
- **点睛，不是通篇**：在关键处放 1–2 个失控点（情绪爆、非理性选择、不完整表达），不是每段都发疯。
- **不牺牲节奏**：番茄仍要爽、要钩；失控感是让爽更可信、人物更立得住，不是拖节奏。
- **不闭环 ≠ 烂尾**：主线大坑必须回收（见 `references/...`/伏笔追踪）；留白的是**次要的、情绪性的、关系性的**小东西。
- **私货有度**：偏见/不公平评价用在叙述者视角和人物身上，不是三观出问题。

---

# 篇幅字数标准库（Length Standards）

> 写之前先定篇幅：短篇 / 中篇 / 长篇，决定签约门槛、分发逻辑、卷数规划、爽点密度。
> 平台调性/单章字数见 `platform-profiles.md`；本库专管**全书总篇幅**。
> 供 `idea` / `outline` / `expand` / `submission` / `draft` / `spark` 调用。

---

## 一、番茄小说官方划分（最精准，分两套标准）

番茄把作品分成「短故事」（一次性完结）和「连载长篇」两条线，标准不同。

### 1. 番茄短故事（一次性完结短文）

| 档位 | 字数 | 说明 |
|---|---|---|
| 微短 | < 6000 字 | **无法走短故事签约**，篇幅不够 |
| 超短篇（短篇） | 6000 – 24999 字（6千–2.5万） | 主流爆款 **1万–1.8万字**，单篇完整结局，**适合短剧改编** |
| 中短篇（中篇） | 25000 – 80000 字（2.5万–8万） | 剧情线更丰富，**平台主推、稿费档位更高** |

> 短故事签约门槛：**≥6000 字**。冲爆款/短剧改编，瞄准 **1万–1.8万字**。

### 2. 番茄连载长篇（分章节长期更新）

| 档位 | 字数 | 常见频道 |
|---|---|---|
| 首签门槛 | ≥ 2 万字 | 可申请首次签约 |
| 全平台分发门槛 | **≥ 8 万字** | 达标才能正常全平台分发推荐 |
| 小长篇 | 8万 – 30万字 | **女频主流完结篇幅** |
| 标准长篇 | 30万 – 100万字 | 男女频通用 |
| 大长篇 | 100万字以上 | 男频玄幻/历史常见 **200万+** |

> 关键两道坎：**2万字可申请首签**，但要**≥8万字**才进正常全平台推荐。新书别停在 2万–8万的"分发洼地"。

---

## 二、全网通用行业标准（起点 / 七猫 / 晋江 / 知乎通用）

### 1. 短篇
- **微型小说**：500 – 2000 字（知乎盐选小故事、公众号短文）
- **标准短篇**：2000 – 30000 字（单本完结，无长期连载）

### 2. 中篇
- **30000 – 100000 字（3万–10万）**：多条支线、完整人物成长，介于短文与连载长篇之间。

### 3. 长篇（10万字以上，分章节连载）
- **女频言情 / 现言**：普遍 **10万–60万字**完结
- **男频玄幻 / 仙侠 / 历史**：普遍 **80万–300万字**，甚至千万字

---

## 三、文学出版传统划分（和网文区别很大，仅作参考）

> 传统出版的"中长篇"门槛比网文低得多，别拿这套标准套网文签约。

- **短篇**：1万 – 3万字
- **中篇**：3万 – 13万字
- **长篇**：13万字以上（茅盾文学奖标准）

---

## 四、怎么选篇幅（开书前决策）

| 你的目标 | 选篇幅 | 关键数字 |
|---|---|---|
| 知乎盐选 / 公众号付费 | 微型~标准短篇 | 2千–3万，第一人称沉浸 |
| 番茄短故事冲爆款 / 短剧改编 | 番茄超短篇 | **1万–1.8万**，单篇完整结局 |
| 番茄短故事拿高稿费档 | 番茄中短篇 | 2.5万–8万 |
| 女频长篇完结 | 小长篇 | 8万–30万 |
| 男频连载吃推荐 | 标准长篇起步 | 先冲 **≥8万** 进分发，再向 30万+ 滚 |
| 男频玄幻/历史大长篇 | 大长篇 | 100万+，按 30万/卷 规划卷数 |

**规划公式**：目标总字数 ÷ 单章字数（见 `platform-profiles.md`，番茄 1500–2500、起点 2000–3000）≈ 章数；再按 **每卷 25万–35万** 切卷。

---

## 五、各技能怎么用本库

- `idea` / `spark`：立项时定篇幅档位 + 给目标总字数。
- `outline`：按总字数算卷数/章数，长篇务必先过 **8万分发坎**。
- `expand`：单章字数仍看 `platform-profiles.md`；本库管"还差多少到签约/分发门槛"。
- `submission`：体检时核对是否够签约/分发字数（番茄短故事≥6000、长篇首签≥2万、分发≥8万）。
- `draft`：短篇导出留字数余地，别卡死在门槛线下。

# 稿件排版与修改规范（Manuscript Format）

> 作者的 Word 排版与改稿习惯。供 `draft` 调用。

---

## 一、核心理念：先"大概"，后筛选修改
让 AI 写小说，**先写个大概的草稿来看**，再由人去筛选和修改。所以 AI 的产出是**草稿/初稿**，不是定稿——要：
- 写得"差不多"，把精修和拍板留给作者；
- **显式标出可改的地方**（让作者一眼找到、自己改），而不是默默写死。

## 二、Word 输出规范
| 项 | 规则 |
|---|---|
| 格式 | Word `.docx` |
| 命名 | `《书名》修改1.docx`，已存在则 `修改2`、`修改3`… |
| **不覆盖原文** | 永远新建文件，绝不改动作者原稿 |
| 目录 | 自动插入（Word 右键目录→更新域 生成页码）|
| 章标题 | "第X章…" **左对齐**（Heading 1）|
| 正文 | 宋体、**每段首行缩进 2 字符**（Word firstLineChars=200，跟字号走）；章标题/书名不缩进 |
| 修改标记 | `【修改：建议】` → 红色高亮，便于定位 |

## 三、字数规范（短篇）
- **短篇总量**：8000–30000 字。
- **草稿目标**：10000–20000 字 —— 留出"既能删也能扩"的余地（作者筛选时好操作）。
- 低于 8000 提示扩写，高于 30000 提示拆分。

## 四、【修改】标记该标在哪
节奏慢/铺垫长、钩子弱、情绪在喊没演、无缘无故的修辞、不搭的引用、爽点/打脸不足、可扩可删的弹性段——都用 `【修改：…】` 标出，给出**具体怎么改**的建议。

- 生成脚本 → `skills/draft/build_docx.py`
- 诊断/标注思路 → `skills/review`、`skills/annotate`、`skills/deslop`、`skills/emotion`
- 版本快照 → `skills/memory`

# 角色起名库（Character Name Library）

> 角色名要"贴时代、贴身份、好读好记、有记忆点、不撞大众款"。本库供 `name` / `character` 调用。

---

## 一、AI/大众起名通病（先认清）

| 病症 | 典型 | 问题 |
|---|---|---|
| **烂大街男主名** | 叶辰、林轩、楚枫、萧炎、苏宇、陈枫 | 撞名几千本，零记忆点 |
| **烂大街女主名** | 苏念、沈清欢、林婉儿、慕容雪、白浅 | 太"仙太软"，满大街 |
| **姓挤在网文专用姓** | 叶/萧/林/楚/慕容 | 一看就 AI、就网文流水线 |
| **堆生僻字** | 玥/璟/翀/砼 | 读者打不出、记不住、念不顺 |
| **配角名太像** | 一群人都姓李/都两字/都一个调 | 读者分不清谁是谁 |

---

## 二、起名 5 步法（不会起也能起好）

1. **定时代/世界观** → 决定风格（古风/现代/西幻/修仙）
2. **选姓**：避开烂大街网文姓，从下面"姓氏池"挑有质感的
3. **定性格意象** → 取 1-2 个能体现性格/命运的字（但别晦涩）
4. **配音韵**：声调有起伏、好念、两到三字为主
5. **查重 + 查歧义**：别撞知名角色、别有谐音歧义；一群配角拉开区分度

---

## 三、姓氏池（避开烂大街，按质感分）

- **有质感常用姓**（推荐主角）：沈、顾、陆、江、temp裴、温、谢、宋、霍、纪、闻、陆、商、燕、裴
- **生活化姓**（现代都市更真实）：周、吴、郑、许、唐、孙、何、罗、夏
- **世家/古风姓**（古言）：谢、王、崔、卢、裴、萧(古言可用)、独孤、上官、慕容(慎用，已烂)
- **避开/慎用**（网文用烂）：叶、萧(现代)、林、楚、苏(女主)、墨、慕容

## 四、按题材的起名风格

### 现代都市 / 刑侦悬疑
- 风格：正常但有质感，别太"古风"也别太土。
- 女主（干练/利落）：温野、江照、裴知秋、商雨棠、顾舒、闻笙
- 男主（沉稳/冷峻）：陆沉、霍珩、闻时、宋野、纪川、江停
- 反派（连环案，越文雅冷静越瘆人）：谢明烛、宋邃、纪寒、严漾

### 古言 / 古风
- 姓(世家)+单字雅名 或 双字：谢知微、裴砚、温辞、沈鹭、崔莞、卢窈
- 男：裴砚、谢临、萧决(古言可)、崔珩

### 玄幻 / 修仙
- 半雅半气，可带意境但别堆生僻：江尘、墨渊(慎)、苏明阙、叶轻眉(借鉴结构非照抄)
- 宗门/势力命名见第六节

### 西幻 / 星际
- 音译为主，统一语感：凯尔、塞拉、奥兰、薇恩、洛安

---

## 五、按身份/性别的取名手感

- **女主**：要有**记忆点 + 反差**，别落"清欢/婉儿"俗套。利落字（野、照、棠、辞、笙）比软字更出挑。
- **男主**：沉稳大气，避免"凌绝战逆"工业霸总味，一个有质感的字胜过三个霸气字。
- **反派**：阴鸷或文雅都可，文雅冷静的反派名最瘆人（谢明烛 > 黑龙王）。
- **长辈/配角**：符合年代（长辈用建国/秀兰/桂芬等年代名才真实），配角之间拉开区分度。

## 六、势力 / 宗门 / 地名命名

- 宗门：避开"天X门""万剑宗"流水线，可用意象（听雪楼、青冥观、忘川阁）。
- 组织（现代/悬疑）：简洁有压迫感（暗渠、回声、第七室）。
- 地名：有画面感，别全是"天元大陆""玄黄界"。

## 七、起名自检清单
- [ ] 姓避开了 叶/萧/林/楚/苏 这些烂大街？
- [ ] 撞知名角色没？（撞了重起）
- [ ] 有没有生僻到读者打不出来的字？
- [ ] 好念吗？声调有起伏吗？
- [ ] 贴合时代/身份吗？（古人别叫现代名，反之亦然）
- [ ] 一群配角彼此区分得开吗？

---

- 角色起名/改名 → `skills/name`　人设设计 → `skills/character`
- 书名起名 → `skills/title`（同源反 AI 思路）

# 平台调性库（Platform Profiles）

> 同一个故事，投番茄和投晋江要写成两个样子。平台决定**节奏、字数、爽点密度、男女频偏好、付费结构、同人政策**。
> 本库供所有写作技能调用；写作前先定"平台 + 频道"。**实时榜单/热门题材用 `trends` 联网核对**。

> 技能用法：可在命令后带"平台 + 频道"参数，如 `/webnovel:outline 番茄 男频`、`/webnovel:hook 知乎盐选 女频`。

---

## 一、番茄小说（免费流量天花板 · 算法推荐 · 长短篇通吃）

> 字节系，免费阅读，广告分成 + 付费，**完读率/追读**是命脉。读者下沉、量大。

### 男频（约 70% 男性读者，短平快爽文）
- **热门题材**：都市战神/赘婿、重生历史、玄幻修仙、系统种田、灵异探案、穿越回乡种田
- **写作规则**：
  1. 前 3 章必须打脸 + 身份反转，**第 1000 字就要第一次小爽点**（被羞辱→亮出底牌）
  2. 主线 = 主角持续变强、资源收割、仇敌逐个清算，**全程高频爽点，不能长时间憋屈**
  3. 长篇 100w+ 起步，短句多、少大段心理描写

### 女频（免费女频头部，下沉读者多）
- **热门题材**：闪婚霸总、重生宝妈逆袭、穿书炮灰、种田宅斗、灵异后妈、校园甜宠、追妻火葬场
- **写作规则**：核心 = 女主搞事业 + 专属偏爱男主，**虐完立刻补偿，3 章一个小反转**；短篇 6000 字标准篇幅（番茄短篇签约门槛）

### 同人政策 ⚠️

# 人称 / 视角指南（POV Guide）

> 用第一人称还是第三人称，决定读者的"代入方式"。供 `expand` / `spark` / `idea` 调用。
> **本插件默认第一人称**（更沉浸、更像在写自己的亲身经历）；用户要第三人称也支持——**动笔前必须明确告知当前用的是哪个人称**。

---

## 一、默认规则（重要）

- **默认第一人称「我」**：代入感最强，读起来像作者在讲自己的经历，最适合知乎盐选、女频情感/现实向、短篇、悬疑沉浸。
- **用户指定第三人称** → 就用第三人称，并照单执行。
- **每次动笔前，用一句话表明**：例如"默认用**第一人称**写，如需第三人称告诉我。"——让用户清楚当前人称，可随时切换。
- **同一本书人称统一**：开了第一人称就别中途跳第三人称（反之亦然）。

---

## 二、第一人称「我」

- **特点**：强代入、强沉浸、像亲历；情绪和悬念都透过"我"的感官和认知传递。
- **优势**：
  - 读者和"我"绑定，**像在看自己的故事**（知乎盐选爆款核心）。
  - 信息天然受限——"我不知道的，读者也不知道"，**天生制造悬念和钩子**（信息差、反转）。
  - 情感戏更贴、更戳（女频虐点、现实向）。
- **写好要点**：
  - 像在跟读者**讲自己的事**，口语、贴近、可带点自嘲或评论。
  - 情绪仍要"演出来别喊"（"我"也别直接说"我好痛苦"，见 `show-emotion.md`）。
  - 善用"我"的认知盲区埋钩："我当时还不知道，这是最后一次见他。"
  - 知乎盐选格式：段落 ≤3 行、多用感叹和破折号、开篇第一句埋悬念（见 `platform-profiles.md`）。
- **适合**：知乎盐选、女频现实/情感/悬疑、短篇、追妻火葬场、第一人称沉浸爽文。

---

## 三、第三人称「他/她」

- **特点**：视角灵活，可多线、可群像、可拉远拉近。
- **优势**：
  - 能写"我"看不到的场面（反派密谋、多线并进）。
  - 男频长篇升级流、宏大世界观更顺手。
- **写好要点**：
  - 最常用**第三人称限知**：贴着主角一个人的视角走，别一会儿钻这个脑子一会儿钻那个（"全知乱跳"是大忌）。
  - 需要切换视角时，**分章/分节切**，别一段里跳。
- **适合**：男频玄幻/系统/历史长篇、多主角群像、宏大世界观。

---

## 四、按平台/频道的人称倾向（仅供参考，默认仍第一人称）

| 场景 | 常见人称 |
|---|---|
| 知乎盐选 | **第一人称**（沉浸、像亲历）✅ 强推 |
| 女频现实/情感/短篇 | 第一人称多 |
| 女频长篇言情 | 第一/第三都常见 |
| 男频玄幻/系统/长篇升级 | 第三人称多 |
| 悬疑/恐怖沉浸 | 第一人称多 |

> 注意：男频长篇升级流虽常用第三人称，但**默认仍按本插件规则用第一人称**；若你判断第三更合适，**提示用户**"这类通常用第三人称，要切吗？"，最终以用户选择为准。

# 剧本格式与写法库（Script Format）

> 剧本和小说是两套语言。剧本铁律：**只写镜头能拍到的——看得见、听得见**。不写心理活动、不写"他想起三年前"（除非用闪回镜头）。情绪靠动作、表情、台词演出来。
> 当下最火的是**竖屏短剧**，本库以它为主，兼顾影视长剧本与"小说改剧本"。供 `script` 调用。

---

## 一、剧本铁律（和小说最大的区别）

1. **只写能拍的**：镜头能拍到的画面、动作、对白、音效。删掉一切心理描写、内心独白、抒情旁白。
2. **现在时、客观**：场景描述用现在进行的客观镜头语言，不评论、不解释。
3. **情绪靠演**：不写"她很伤心"，写她的动作和台词（和小说的 show-don't-tell 一脉，更彻底）。
4. **台词能说出口**：短、口语、能演；一句台词一个意思。
5. **一场一冲突**：每场戏有一个目标或冲突，没戏就删。

---

## 二、标准格式（中文剧本通用排版）

```
第 1 集

场景 1   医院走廊   日／内

苏晚跑过走廊，一只高跟鞋甩飞在地。

苏晚（喘着气）：医生呢？我老公在哪？

护士拦住她。

护士：你先冷静。

苏晚一把推开护士，撞开急救室的门。

（卡点钩子：病床是空的，床头卡片上写着另一个女人的名字。）
```

格式要点：
- **场景标题行**：`场景N  地点  时间(日/夜)  内／外景`。一换地点或时间就换一场。
- **场景/动作描述**：单独成段，客观、能拍。
- **台词**：`人物名（表演提示）：台词`。表演提示用圆括号（），写语气或小动作，**不用【】**。
- **画外音**：`人物名（V.O.）` 画外旁白；`（O.S.）` 画外发声；闪回标 `闪回：`。
- **转场**：需要时单独一行写 `转场` 或 `切`；多数短剧直接换场即可。
- **卡点钩子**：每集结尾用一行（动作或台词）制造悬念，逼着看下一集。

> 投影视/正规渠道可用好莱坞标准格式（INT./EXT. — DAY/NIGHT、动作行、人物名居中、对白），但国内短剧/平台多用上面的中文排版。

---

## 三、竖屏短剧写法（最火，重点）

- **体量**：约 80–100 集，每集 1–2 分钟，单集剧本约 300–600 字；竖屏构图（近景、特写多）。
- **黄金 3 秒 / 第一集前 30 秒**：必须抛冲突或反转，3 秒不抓人就划走。
- **爽点密度比网文还高**：几乎每集一个爆点（打脸/反转/打脸/身份揭穿）。
- **每集结尾强卡点**：在最炸的前一秒切，钩子比网文章尾更狠。
- **人物脸谱化、台词直给**：反派嚣张、主角隐忍后爆发；台词要"出口即爽"。
- **强冲突前置**：误会、打脸、当众羞辱、身份反转，越快越好。
- **题材**：逆袭、打脸、重生复仇、赘婿战神、豪门虐恋、追妻火葬场、甜宠（和女频/男频套路通，调 `trope-library.md`、`hook-library.md`）。

### 短剧单集结构（模板）
```
开场（钩子，前 3 句立冲突）
→ 冲突升级（对手加码 / 误会加深）
→ 小反转或小爽点
→ 集尾卡点（最炸处切断，留悬念）
```

---

## 四、小说改剧本（网文→短剧）

1. **砍心理、留动作对白**：把"她心里又气又怕"改成她摔门、她颤声说的台词。
2. **拆场景**：按地点/时间把章节切成一场场戏。
3. **提炼台词**：把叙述里的信息变成人物说出口的话或可拍的动作。
4. **重排节奏**：按短剧密度，把爽点提前、卡点压到集尾。
5. **删旁白式解释**：除非用 V.O. 画外音，否则不留作者解释。

---

## 五、剧本里的"去 AI 味"
- 台词别书面、别工整、别每句完整（口语、停顿、半句话，调 `dialogue-library.md`、`sentence-rhythm.md`）。
- 动作描述别堆形容词，写具体可拍的动作。
- 情绪靠表演提示和动作，不靠旁白喊。

# 错别字 + 敏感/违禁内容库（Proofread & Censor）

> 供 `proofread` / `submission` / `deslop` 调用。两部分：**错别字高频对**（可放心用）+ **敏感/违禁大类排查**（启发式安全网，非平台确切词库）。

> ⚠️ **重要边界**：平台（番茄/起点/晋江等）的**确切屏蔽词库是内部机密、动态更新**，谁也拿不到完整实时版。本库是按**公开审核常识**整理的排查框架，目标是"投稿前先自查、少踩明显的雷"，**不能保证命中平台后台的每一个词**。最终以平台实际审核结果为准。

---

## 一、错别字高频对（按句子意思判断，不是机械替换）

### 1. 同音误用（最高频）
| 易错 | 用法口诀 |
|---|---|
| 的 / 地 / 得 | 名词前「的」、动词前「地」、动词后补语「得」（漂亮的花 / 飞快地跑 / 跑得快）|
| 在 / 再 | 「在」表正在/处所，「再」表又一次（在写 / 再写一遍）|
| 做 / 作 | 具体动作多用「做」（做饭做事），抽象/书面多用「作」（作为、作出、写作）|
| 须 / 需 | 「必须」（一定要）/「必需」「需要」（需求）|
| 即 / 既 | 「即」=就是/立刻，「既」=既然/既…又（即刻 / 既然）|
| 账 / 帐 | 钱用「账」（账户账目），布幔用「帐」（帐篷蚊帐）|
| 以 / 已 | 「以」介词，「已」=已经 |
| 象 / 像 | 比喻/相似用「像」（像…一样）|

### 2. 形近误用
辩(辩论)/辨(分辨)/辫(辫子)；燥(干燥)/躁(急躁烦躁)；竟(竟然)/竞(竞争)；蓦(蓦然)/暮(暮色)；缘(缘分)/原(原因)。

### 3. 成语易写错
迫不**及**待（非"急"）、川流不息（非"穿"）、黄**粱**一梦（非"梁"）、再接再**厉**（非"励"）、走**投**无路（非"头"）、一**筹**莫展、信口**雌**黄、变本加**厉**、首**屈**一指、不**胫**而走。

### 4. 多字 / 漏字 / 重字 / 顺序
「他他走了」（重字）、「走了进去去」（多字）、「把饭吃」（漏"了/完"）、「我的妈妈的」（多"的"）。

### 5. 标点（与 `anti-ai-checklist.md` 七一致）
`...`/`。。。` → 「……」；并列误用句号 → 顿号「、」；主引号用双引号""，单引号''只用于嵌套；正文不滥用【】。

---

## 二、敏感 / 违禁内容大类（按类自查 + 替换思路）

> 每一类给"为什么踩雷 + 怎么改"。不逐字列违禁词（无意义且易过时），按**类别**自查更可靠。

### 1. 涉政 / 时政
真实政治人物、政权、敏感历史事件、现实政治影射。
→ 网文一律**架空**：朝代、国名、人名全虚构；不影射现实政治。

### 2. 色情低俗 / 擦边
露骨性描写、性器官直写、未成年性相关（红线，绝不可碰）。
→ 情欲场景**点到为止/转场留白**；晋江等有"限制级"分区也需守线；番茄等免费向更严。

### 3. 暴力血腥 / 自杀自残
过度血腥细节、**自杀自残的具体方法步骤**、虐待细节。
→ 冲突写"结果与情绪"不写"操作教程"；不提供可模仿的具体方法。

### 4. 赌博 / 毒品 / 违禁品
赌术、制毒吸毒、违禁药物的**具体操作/配方/剂量**。
→ 剧情需要可"提及存在"，但不写可操作细节。

### 5. 封建迷信（与"修真设定"区分）
区别：**架空world的修真/玄幻法术=设定**，没问题；**引导读者现实中算命/跳大神/治病=迷信渲染**，踩雷。
→ 把超自然牢牢锁在架空设定里，别和现实生活的"求神治病"挂钩。

### 6. 真实品牌 / 商标 / 药名 / 真实人名地名
写真实品牌（可口可乐/某手机）、真实药名、真实明星/企业家姓名、真实小区/学校全名——易判**广告嫌疑或侵权**。
→ 一律换**虚构名**（"某品牌""临江市""第一中学"），或泛化。

### 7. 辱骂 / 歧视
脏话粗口、地域黑、性别/职业/群体歧视、人身攻击。
→ 角色"毒舌"可以有锋芒但不带真实歧视；脏话用"他骂了句粗话"带过或软化。

### 8. 价值观 / 三观红线
美化犯罪不受罚、宣扬极端报复/反社会、洗白施暴者、鼓励违法。
→ 爽文打脸/复仇没问题，但**别让"作恶且正确"成为价值导向**；坏人要有代价（也利于过审）。

### 9. 未成年相关（红线最严）
涉及未成年的性、恋爱擦边、暴力——平台零容忍。
→ 角色设定成年；校园文也守线。

# 句式节奏库（Sentence Rhythm）

> AI 写的句子"整齐划一、太顺"，像流水线；人写的句子**长短不均、有呼吸、有停顿、有断裂**。
> 这是"句子太顺"那条 AI 味的具体解法。供 `deslop` / `human` / `expand` 调用。

---

## 一、人写 vs AI 写（句式对照）

| 维度 | AI 短句 | 人写短篇 |
|---|---|---|
| 句长 | 多为 5–15 字短句，均匀 | **长短不一**，短句+长句混合，偶尔 50–60 字甚至更长 |
| 节奏 | 整齐划一 | 不均匀，**有呼吸感** |
| 修辞 | 直接简洁 | 偶有比喻、重复、内心独白（但不堆，见 `anti-ai-checklist`） |
| 情绪 | 明显但规范、每句说完 | 会拖、会断、会模糊，**话说一半** |
| 停顿 | 少 | 常用顿号、省略号、破折号、换行，模仿思考/呼吸 |

> 一句话：**句长不均 + 加停顿 + 情绪拖一拍/断一下**，就像人写的了。

---

## 二、5 条句式节奏改写法

### 1. 句长不均匀
- 短句与长句**交错**；该短就一两个字成句，该长就一口气 50–60 字不断。
- **避免每句长度都差不多**——整齐 = AI。
- 例：「他站住了。门后那道熟悉又陌生的影子，和他记忆里那个总在厨房哼歌、把最后一块排骨夹给他、临走还回头看一眼的人，慢慢重合，又慢慢碎开。他没动。」（短—长—短）

### 2. 用停顿标点做呼吸
- **省略号**……拖一下、留余地。
- **破折号**——转折、补充、被打断。
- **换行/单句成段**：制造停顿和重音。
- **顿号**用于并列（不是句号，见 `anti-ai-checklist`）。

### 3. 情绪不说完 / 话说一半
- 「他张了张嘴，最后什么也没说。」
- 「她想解释，可是……算了。」
- 内心独白突然中断，比说圆更真。

### 4. 节奏配情绪
- 紧张/爆发：短句、碎句、单句成段，砸下来。
- 舒缓/回忆/抒情：长句铺一铺，缓口气。
- **紧—缓—紧**交替，别一个节奏到底。

### 5. 允许一点口语化和"不完美"
- 偶尔的口语、自言自语感、轻微的逻辑跳跃，让它像人不像说明书（深层见 `human-texture.md`）。
- 注意：是"自然的不完美"，不是病句和错别字。

---

## 三、改写指令（喂给任何 AI 都能用）
> "句子长度不要整齐划一：短句与长句交错，偶尔出现 50 字以上的长句；适当用省略号、破折号、换行做呼吸停顿；人物情绪不必每句说完，可以话说一半或内心独白突然中断；保持原剧情和爽点，只调句式节奏，让它读起来像人写的、有呼吸感，而不是流水线短句。"

---

## 四、注意（别走另一个极端）
- 不是"全写长句"——是**长短交错**；通篇长句一样累。
- 停顿标点是调味，**别每句都……都——**，那又成了新的整齐。
- 番茄等快节奏平台仍以短句为主，长句少而精地穿插用。

# 情绪外显库（Show, Don't Tell）

> 网文最忌"喊情绪"——"我很心疼""我好痛苦"。要把情绪**演出来**，让读者自己感受到。
> 供 `emotion` / `expand` / `deslop` / `dialogue` / `shuangdian` 调用。

---

## 一、核心原则

**不告诉读者"我什么感受"，而是展示"我的身体/动作/语言怎么了"，让读者自己得出那个感受。**

- ❌ 告诉（Tell）：「我很心疼他。」——读者无感。
- ✅ 展示（Show）：「我伸手想替他擦掉血，指尖碰到他脸的瞬间又顿住，最后只是把外套往他身上紧了紧。」——读者自己心疼了。

**为什么有效**：读者不爱被"通知"情绪，爱"亲眼看见"再自己感受。看见的情绪才可信、才代入。

---

## 二、展示情绪的 6 条通道

写情绪时，从这 6 条里挑 1–2 条具体化（别全堆，1-2 个最准的细节胜过一堆）：

1. **神态/微表情**：眼神、嘴角、眉头、视线落点（"嘴角扯了扯""别开眼""盯着一个点出神"）。
2. **小动作**：手、姿势、下意识的动作（"攥紧又松开""反复搓着杯壁""把烟摁灭"）。
3. **生理反应**：呼吸、喉咙、体温、眼眶、颤抖（"喉咙发紧""指尖发凉""眼眶热了一下"）。
4. **对话潜台词**：说反话/半句话/答非所问（难过时说"我没事，你忙吧"）。
5. **环境/物件烘托**（克制用）：用一个具体物件或动作折射，**别写无缘无故的比喻**（"那杯茶早就凉了，她也没动" 好过 "心如刀绞般凄凉"）。
6. **留白**：不解释，动作完就停（"她转身上楼，没回头。"）。

---

## 三、常见情绪 → 外显对照表（直接取用）

| 情绪 | ❌ 别写 | ✅ 这样演 |
|---|---|---|
| **心疼** | "我好心疼" | 伸手又收回；别过脸不让他看见红的眼眶；声音哑了一瞬；把外套替他拢紧 |
| **痛苦/难过** | "我好痛苦" | 笑比哭难看；指甲掐进掌心；喉咙像堵了东西；盯着一处失神；"扯了扯嘴角，挤出一个笑" |
| **愤怒** | "他很愤怒" | 拳头攥得咯咯响，指节发白；后槽牙咬碎；声音反而低下来；把杯子轻轻放下——太轻了 |
| **委屈** | "我很委屈" | 眼眶热了，仰头把眼泪逼回去；"我没事"说得太快；低头扒饭，咽得很慢 |
| **失望/心寒** | "我对他失望透顶" | 平静得反常；"我累了"；侧开身；把文件袋从门缝推回去 |
| **喜悦** | "我开心极了" | 嘴角压都压不住；走路带风；反复看手机那条消息；声音不自觉扬起来 |
| **紧张/害怕** | "我好紧张" | 手心出汗在裤子上蹭；反复咽口水；心跳声盖过别的声音；腿有点软 |
| **震惊** | "他震惊了" | 僵在原地；瞳孔骤缩；倒吸一口凉气；手里的东西啪地掉了 |
| **尴尬** | "气氛很尴尬" | 没人说话；茶杯碰桌面的声音格外响；他低头摆弄打火机 |
| **思念** | "我很想你" | 给那个再也不会回的号码发了条消息，又删了；留着他没喝完的那半杯 |
| **嫉妒** | "她很嫉妒" | 笑得很标准，眼睛没笑；指甲在掌心掐出月牙；"挺好的啊"语气淡了一度 |

---

## 四、改稿三步（把"喊"改成"演"）

1. **找**：圈出所有"我很X""他感到X"的直白情绪句。
2. **定**：想这个情绪下，身体/动作/语言会有什么真实反应（查上表）。
3. **换**：用 1–2 个具体细节替换那句直白；必要时加一句潜台词；**删掉"我很X"**。

> 例：「我很伤心，八年感情成了笑话。」
> → 「我扯了扯嘴角，挤出一个笑。秦奈不知道，我下周也要结婚了，和别人。」

---

## 五、注意（别矫枉过正）
- **1-2 个准细节就够**，别一个情绪堆五个动作，那又成了油腻。
- **克制**：成年人的情绪是收着的，越克制越戳（尤其女频虐点）。
- **环境烘托要具体、要轻**，别变成"无缘无故的修辞"（见 `anti-ai-checklist.md`）。
- 极快节奏处（番茄打斗/打脸）可以直给一点，不必处处含蓄——看场景。

# 网络热梗使用指南（Slang Guide）

> 热梗能让现代向网文有网感、贴近年轻读者，但**用错地方/用过量/用过气梗 = 立刻廉价又突兀**。
> 本库给"怎么用不突兀"的规则；**具体梗有时效性，靠 `slang` 技能联网实时搜**，别信写死的列表。

---

## 一、用得不突兀的 6 条铁律（核心）

1. **看题材**：现代都市/校园/职场/现实向/轻松搞笑 → 可用；**古风/玄幻/历史/正剧/严肃 → 基本不用**。现代网络梗塞进古风=污染语域，大忌（呼应 `anti-ai-checklist.md` 的"语域一致"）。
2. **贴人物身份**：年轻、有网感的角色才说网络梗；老人、古人、严肃/高位角色不说。一个 50 岁老干部不会冒出"City不City"。
3. **适量**：一篇撒 **1–3 个**就够。堆梗 = 廉价 + 过气快 + 像在硬蹭。
4. **避开快过气的**：用相对有生命力的；上周才火、靠单一事件的梗，下周就尬。**烂梗比不用更尬**。
5. **放进对话/心理，别硬塞旁白**：梗在角色嘴里、吐槽里才自然；作者旁白强行玩梗最出戏。
6. **服务情绪/人物**：梗要体现性格或推进情绪，不是为玩而玩、为蹭而蹭。

---

## 二、长青 vs 易过气

- **相对长青**（已沉淀进日常口语，风险低）：情绪价值、内卷、躺平、破防、emo、e人/i人、活人感、City不City（也在退）…
- **易过气**（火得快烂得快，用前必确认还在火）：当月新出的句式梗、谐音梗、特定综艺/事件梗。

> 经验：**越是"一句话场景梗"越短命，越是"一个概念词"越长青**。拿不准就别用，或用 `slang` 查它还火不火。

---

## 三、按平台/频道

- **番茄/下沉/年轻向**：可适度用直白热梗增加网感。
- **晋江/精品**：克制，偏生活化口语而非硬核网络梗。
- **知乎盐选/现实向**：可用，但要贴第一人称叙述者的身份和语气。
- **古言/玄幻/正剧**：不用现代网络梗。

---

## 四、当前热梗快照（会过期，仅示例，用前用 `slang` 刷新）

> 截至 2026 年初：活人感、赛博对账、情绪价值、City不City、高雅企鹅、如何呢又能怎、来财、村咖、浪浪山小妖怪 等。
> ⚠️ 这只是某一时刻的样子，**热梗每月在变，正式用前必须用 `slang` 联网核对当前是否还在火**。

---

# 投稿与过稿经验库

> 写得好 ≠ 签得了约。过稿是另一套游戏：编辑在 3 章内、读者在 3 分钟内做判断。
> 本库供 `submission` / `idea` / `review` 调用。各平台具体政策会变，**实时数据用 `trends` 联网核对**。

---

## 一、编辑/系统在看什么（签约判断点）

网文签约本质看**两个数据预期**：
1. **留存**：黄金三章能不能让读者点到第 4 章（收藏率/追读率）
2. **商业性**：题材是不是当下平台要的、爽点能不能持续供给

编辑快速过稿时盯的点（按顺序）：
- **前 3 句**：有没有立刻抓人（钩子）
- **第 1 章**：金手指/核心卖点上没上、主角立没立住
- **前 3 章**：有没有第一个小高潮、主线悬念清不清楚
- **简介/标签**：卖点一句话说得清吗、题材标签对不对平台
- **更新稳定性**：能不能日更稳定（签约后考核）

## 二、黄金三章过稿标准（自检清单）

- [ ] 第 1 句立危机或反差
- [ ] 第 1 章内金手指/核心设定上线（番茄）或最晚第 3 章（起点）
- [ ] 第 1 章主角有一个让人记住的点
- [ ] 第 3 章前有第一个明确爽点/小高潮
- [ ] 三章内主线悬念清晰（读者知道"接下来要看什么"）
- [ ] 每章章尾有钩
- [ ] 没有大段世界观说明文/背景交代（最大劝退点）

## 三、各平台投稿要点

| 平台 | 投稿要点 | 易拒原因 |
|---|---|---|
| **起点** | 重设定深度与文笔，男频玄幻/都市为主；签约看三章+大纲 | 节奏太慢、开篇铺垫太长、同质化 |
| **番茄** | 极致快节奏，第1章上爽点上钩；数据流（完读/追读）说话 | 开篇拖、爽点慢、单章太长 |
| **七猫** | 下沉爽文，套路直给，悬念前置 | 太文艺、不够直白 |
| **晋江** | 女频精品，重情感与人物，文笔在意 | 三观不正、人设崩、感情线弱 |
| **UC** | 落魄主角身怀绝技开局，狗血强冲突，中老年下沉 | 太文艺、冲突不够狗血 |
| **知乎盐选** | 第一人称短篇，开篇第一句埋悬念，段落≤3行 | 开篇不抓人、不够沉浸 |
| **Webnovel（出海）** | 男频系统/都市最吃香，爽点直给，适配翻译读者 | 文化梗太重、节奏慢 |

## 四、简介与标签（过稿的"门面"）

### 简介公式
```
[身份/处境钩] + [金手指/转折] + [核心爽点预期] + [悬念尾钩]
```
- 例（男频）：被未婚妻退婚当天，他觉醒了吞噬万物的系统。三年后归来，当年看不起他的人，跪着求他收下门票。
- 例（女频）：穿成恶毒女配的第一天，她就把剧本烧了。男主追妻火葬场？不好意思，姐忙着升级打脸。

### 标签
- 选 **3–5 个平台热门标签**，第一个标签=核心卖点。
- 标签要"对平台胃口"，用 `trends` 查当下热标。

## 五、签约后的更新考核
- **日更字数**：多数平台要求日更 3000–6000 字稳定。
- **断更红线**：签约期断更易解约，开书前先存稿 3–5 万字。
- **数据维护**：追读掉了要会救（用 `review` 诊断）。

# 网文起名库（Title Library）

> 书名是点击率的第一道关，也是算法识别题材的关键。本库专治"AI 味/大众味"书名。
> 供 `title` / `idea` 调用。

---

## 一、为什么 AI 起的名又土又大众（病根）

| 病症 | 例子 | 问题 |
|---|---|---|
| **抽象文艺** | 《浮世清欢》《岁月如歌》《暗夜微光》 | 没信息、没爽点、读者不知道是啥题材 |
| **烂大街通用** | 《重生之路》《逆袭人生》《命运转折》 | 烂大街、零记忆点、撞名一堆 |
| **没题材关键词** | 《迷局》《追凶》《真相》 | 算法识别不出题材，推不出去 |
| **没钩子** | 《我的修仙生活》 | 平淡，看不出爽在哪 |

**一句话**：AI 爱起"形容词+意境"的名；网文要的是"**关键词+爽点+钩子**"的名。

---

## 二、反 AI 起名 6 原则

1. **带题材关键词**：系统/重生/穿书/神医/读心/侧写/赘婿——让算法和读者一眼识别题材。
2. **带爽点或钩子**：无敌/首富/打脸/反派/跪求/倒计时/十年悬案——一眼看出爽在哪。
3. **要具体意象，不要抽象形容词**："恐怖屋""倒计时""地契"＞"惊魂""宿命""微光"。
4. **敢口语化、有网感**：敢用"我"开头、敢用数字、敢用反差和调侃。
5. **长度配平台**：番茄/七猫爱长名带卖点（一句话剧透爽点）；起点偏精炼有逼格。
6. **避开烂大街壳**：《重生之X》《最强X》《我的X生活》——要用就加新意，别裸用。

---

## 三、男频起名套路（附公式）

| 套路 | 公式 | 例 |
|---|---|---|
| 关键词+夸张 | 题材词 + 极致化 | 《史上最强炼气期》《全球高武》 |
| 第一人称口语 | 我 + 反差设定 | 《我有一座恐怖屋》《我能看到经验条》 |
| 身份反差 | 低身份 + 隐藏强 | 《赘婿》《大奉打更人》 |
| 开局体 | 开局 + 爽点 | 《开局一座山》《开局签到荒古圣体》 |
| 反套路调侃 | 吐槽感 + 反差 | 《这游戏也太真实了》《我的徒弟都是大反派》 |

## 四、女频起名套路（附公式）

| 套路 | 公式 | 例 |
|---|---|---|
| 穿书重生+爽点 | 穿成X后我Y | 《穿成恶毒女配后我躺赢了》《重生后我成了首富》 |
| 身份反差 | 反差身份词 | 《农门辣妻》《国民老公带回家》 |

# 套路元素库（Trope Library）

> 网文的"爽"建立在读者熟悉的套路上——套路不是贬义，是**与读者的默契契约**。
> 本库把 23 个核心爽点元素拆成"机制 + 部署方式 + 适配频道"，供 `shuangdian` / `outline` / `idea` 技能调用。

---

## 一、23 个核心爽点元素

| # | 元素 | 触发机制（读者为什么爽） | 部署建议 | 频道 |
|---|---|---|---|---|
| 1 | **主角逆袭** | 弱→强的势能释放 | 开局压到最低，每卷抬一个台阶 | 男频核心 |

## 二、爽点节奏公式

### 2.1 三段式爽感闭环（单个爽点）
```
压抑（铺垫不公/嘲讽/危机）→ 蓄势（主角隐忍/布局/觉醒）→ 释放（打脸/翻盘/碾压）
```
- 压抑越具体，释放越爽。
- 蓄势别太长：番茄类 1–2 章内必释放，起点类可铺 3–5 章。

### 2.2 爽点密度（按平台）
- **番茄/短剧向**：几乎每章一个小爽点，3–5 章一个中爽点。
- **起点/精品向**：可以铺垫，但每 3 章必给一次明确情绪释放。
- 详见 [platform-profiles.md](platform-profiles.md)。

### 2.3 爽点升级链（避免读者疲劳）
同一种爽点（如打脸）不能原地重复，必须**升级对象/规模/方式**：
```
打脸同窗 → 打脸长辈 → 打脸宗门 → 打脸天才 → 打脸大势力 → 打脸天道
```

---

## 三、反套路与新鲜感

读者既要"熟悉的爽"，又怕"老套"。破解：
- **旧套路 + 新外壳**：经典打脸放进"美食/种田/科技"等新题材。
- **预期违背**：让读者以为要套路化发展，临门一脚反转一次（不能常用）。
- **情绪真实**：套路是骨架，人物的具体反应和细节才是血肉。

---

## 四、虐点设计（女频核心：追妻火葬场）

女频的"爽"常常先靠"虐"垫——**虐得越深，后面反转/打脸越爽**。虐点和爽点是绑定的。

### 6 种经典虐点（追妻火葬场常用）
1. **公开场合的敷衍**：当众冷落/打脸女主，只甩一句"乖，等下一次"。
2. **偏爱与忽视的对比**：女主疲惫时，男主却去接别的女人。
3. **未来规划的落空**：从不谈结婚，却画饼"以后给你办完美婚礼"。
4. **态度的双标**：女主的情绪是"小心眼"，别人的暧昧是"不懂事"。
5. **自我价值的否定**：旁人贬低女主，伴侣默认不维护。
6. **回忆与现实的反差**：当年"背你一辈子"的承诺，成了如今的嘲讽。

### 虐点铁律
- **用细节代替喊口号**：不写"我很伤心"，写"我扯了扯嘴角，挤出一个笑"。
- **成年人的克制**：失望是"我累了""我侧开了身"，不是嚎啕大哭——越克制越戳。
- **虐为爽服务**：每个虐点都为后面的"清醒离场 / 打脸反转 / 男主追悔"蓄势。

### 虐→爽闭环（女频追妻火葬场公式）
```
男主渣/傲慢/双标(虐) → 女主清醒平静离场 → 信息差打脸(男主蒙在鼓里) → 男主追悔莫及 → 女主自我实现(过得更好)
```
- 核心：**男主前期多渣多傲慢，后期打脸后悔就多爽**；女主从"卑微等待"到"掌控人生"，读者跟着解气。
- 打脸优先用**信息差**（男主活在自己认知里，读者知道真相），比直接写"他后悔了"爽十倍。

# 温暖现实短篇库（Warm Realism）

> 写"有温度、有现实感、能落地"的短篇——**不狗血、不煽情、不说教、不鸡汤**。
> 番茄不吃大道理，所以温暖要**轻、具体、藏在细节里**。供 `warmth` / `expand` / `idea` 调用。
> 框架来自作者整理，⭐ 标记处为本插件补充的"情感处理"心得。

---

## 一、核心逻辑（记牢）

1. **故事为主，温暖为辅**：情绪/冲突/反转是燃料，温暖/思考是收束，给读者余味。
2. **现实意义 ≠ 苦难堆积**：不强行写惨、不刻意制造困境；写生活、写普通人的小选择就够。让人"看到生活，也看到希望"，比"哭得稀里哗啦"高级。
3. **人物有缺点但可被理解**：主角不全能，会成长/尝试/选择；配角不完美但行为合理。人性复杂，最终带出温暖。

### ⭐ 补充：温暖要"可信"的三条底层
- **温度藏在克制里，不在台词里**：一个不说话的小动作（默默把伞塞过去）＞一段暖心独白。让读者**自己**捂热那一下（呼应 `show-emotion.md`）。
- **先冷后暖，暖才有重量**：纯甜是假的。先有现实的累/委屈/孤独，再给一点点暖，对比之下那点暖才砸得到心里。
- **余味=不说满**：结尾留一个没说出口的善意 / 一个开放的小希望，让读者自己补完，比写全更暖。

---

## 二、人物技巧

| 技巧 | 做法 | 注意 |
|---|---|---|
| 平凡但有特点 | 普通学生/打工人/小人物 + 小兴趣/怪癖 | 别"天才"，让读者代入 |
| 弱点与成长 | 犹豫/胆小/会妥协 + 微小成长 | 成长不是开挂，是心理/情绪的微调 |
| 给每人动机 | 行为背后都有原因 | 不洗白、不刻意黑化 |
| 留余地 | 角色不必全圆满 | 让读者自己感受温度 |

---

## 三、情节技巧

1. **生活化细节撑故事**（代替空洞抒情）：熬夜加班的外卖、微信删了又重写的消息、独自排队看病的狼狈。
2. **小冲突 + 小反转**（不用大灾大难）：同事不理解→忍一忍→意外得到支持；网恋失望→收获现实里的小温暖。
3. **现实逻辑 > 爽文逻辑**：遇困难→尝试→屡碰壁→学会妥协/微小成长。**避免"贵人出现+一夜开挂+圆满结局"**。
4. **温暖收束**：结尾留一线温度或希望；不必全圆满，但让人读完安心/微笑/有共鸣。

### ⭐ 补充：温暖的常见"载体"
陌生人的小善意 · 家人不说出口的爱 · 自己跟自己和解 · 把委屈咽下后的一个小确幸 · 一件被默默记住的小事。选 1 个做收束就够，别堆。

---

## 四、主题与意义

- **先定核心主题**：想让读者感受到什么？例："普通人的努力也有价值""善意可以被传递""小选择也能改变心境"。
- **用故事展现，不直白说教**：不写"你要善良/努力/勇敢"，让角色的行为和结果替你说。
- **模糊与多元**：允许遗憾、无力、失败，但保留温暖亮点。

### ⭐ 反鸡汤的具体手法
把道理**翻译成"一个具体的人做了一件具体的小事"**。
- ❌（鸡汤）"善良总会被这个世界温柔以待。"
- ✅（落地）"她把多打的那份粥推给隔壁桌的老人，没说话，低头继续扒自己的饭。"

---

## 五、结局模板

| 类型 | 写法 | 核心温度 |
|---|---|---|
| **成长式** | 问题没全解决，但心态/选择成熟了 | 给希望，示范心态成长 |
| **释怀式** | 接受遗憾，放下执念 | 生活不完美，但能继续向前 |
| **平凡式** | 回归日常，普通却安心 | 小温暖 + 真实生活感 |

> 温度要点：结尾让人**轻微释怀或微笑**，而不是悲伤或高喊口号。

---

## 六、避雷

1. 强行煽情/鸡汤 → 割裂故事质感
2. 过度狗血 → 冲突离谱、失真
3. 人物非黑即白 → 忽略人性复杂
4. 剧情只为刷爽点 → 刺激完很空

### ⭐ 但别走另一个极端
温暖向**不等于没钩子、没冲突、平平淡淡**。番茄仍要追读：开头照样要钩子、要一个具体的小困扰勾住人，只是**收束**用温暖而非打脸。温暖 ≠ 平淡。

---

## 七、可落地模板（Quick）

**人物**：主角=普通+弱点+微小成长；配角=有动机+有瑕疵；关系=微冲突+小温暖。
**情节**：起(普通生活+小困扰) → 承(困扰升级/冲突) → 转(尝试/碰壁+小亮点) → 合(结局落地+留温暖余味)。
**每篇三件套**：1–2 个生活小细节强化现实感；1 个情绪反转强化冲击；结尾 1 个温暖亮点。

> 一句话：**写普通人的小困境+小冲突+小成长，用生活细节和温暖亮点收束，让读者在轻松阅读中看到现实、感受共鸣、带走余味。**

# 网文写作基本功（教学库）

> 给新手的网文写作入门知识，供 `coach` 教学技能调用。讲原理，不只是给套路。
> 进阶细节散落在各专项库（hook/trope/goldfinger/platform/channel/xiushi-ci/submission）。

---

## 一、网文和传统小说的根本区别
- **商品属性**：网文是连载商品，读者随时能划走，**留人 > 文笔**。
- **即时反馈**：每章都在被数据检验（追读/完读/订阅）。
- **爽感驱动**：读者花钱买"爽"和"代入"，不是买"深刻"。
- 结论：先学会"留住读者"，再谈艺术性。

## 二、六大基本功

### 1. 开篇与黄金三章（决定收藏）
- 前 3 句立危机/反差，前 3 章给第一个小高潮。
- 最大新手错误：开头写大段世界观/背景介绍 → 必劝退。
- 练法：把第 1 章第一句改 5 版，挑最抓人的。→ 配 `hook`

### 2. 爽点与节奏（决定追读）
- 爽点 = 压抑→蓄势→释放。压抑越具体，释放越爽。
- 节奏要有呼吸：紧（冲突）—缓（日常）—紧。
- 打脸记得写"围观反应"，那一拍最爽最易漏。→ 配 `shuangdian`、`annotate`

### 3. 金手指与人物（决定能不能立住）
- 金手指是放大器，放大主角的努力和选择，别让主角变工具人。
- 人物要有记忆点 + 主观能动性 + 成长弧光。→ 配 `goldfinger`、`character`

### 4. 大纲与伏笔（决定会不会烂尾）
- 先有骨架再填肉；埋的每个坑都要记在伏笔总表里。
- 主线悬念要清晰，让读者始终知道"接下来看什么"。→ 配 `outline`

### 5. 文笔与手感（决定读感）
- 网文文笔 = 短、快、狠、活。多对话动作，少抒情排比。
- 修饰词要稀疏、峰值集中，过量=油腻+AI 味。→ 配 `deslop`、`xiushi-ci`

### 6. 投稿与平台（决定能不能变现）
- 不同平台要不同的节奏和题材，投稿前按签约标准自检。→ 配 `submission`、`trends`

## 三、新手十大常见错误
1. 开篇大段背景/世界观说明文
2. 金手指太万能、无代价、获取随意
3. 主角被外挂牵着走，没有自己的选择
4. 爽点太慢，铺垫太长（尤其番茄类）
5. 章尾平淡收尾，没有钩子
6. 埋了坑忘了填（伏笔不回收）
7. 反派太弱，主角赢得没含金量
8. 人物前后不一致（OOC）
9. 修饰词堆砌，辞藻华丽但没节奏
10. 不看平台，一套写法投所有站

## 四、给新手的心态建议
- **先完成，再完美**：写完一本烂书 > 改十个开头。
- **日更习惯 > 灵感**：签约后靠的是稳定更新。
- **数据是老师**：追读掉了别玻璃心，用 `review` 找原因。
- **模仿是起点**：先拆爆款学结构，再谈原创。

# 修饰词库（爽文短平快"味儿"词）

> 修饰词是爽文的"味精"——同一句话，加对修饰词就有爽感、有画面、有节奏。
> 短平快爽文尤其爱用：强化程度、外显情绪、加快节奏。本库供 `annotate` / `expand` / `deslop` 调用。
> ⚠️ 修饰词是调味，不是主菜。过量堆砌反而油腻、显 AI 味，原则是"该狠的地方狠，平铺处留白"。

---

## 一、程度强化词（让动作更"猛"）

> 用在动作/变化前，瞬间提速、加力。短平快爽文高频。

`猛地` `骤然` `霎时` `轰然` `生生` `狠狠` `死死` `活活` `硬生生` `直接` `瞬间` `刹那` `陡然` `蓦地` `登时` `一把` `劈手`

- 例：他**猛地**抬头 / **一把**攥住 / **生生**扛下这一掌 / **直接**懵了

## 二、围观反应词（打脸爽点的"放大器"）

> 打脸/震撼时，写旁观者的生理反应，爽感翻倍。这是爽文最爱、新手最常漏的一类。

`倒吸一口凉气` `头皮发麻` `瞳孔骤缩` `浑身一颤` `脸色煞白` `僵在原地` `下巴差点掉地上` `后背发凉` `心头一震` `鸦雀无声` `死一般的寂静` `咽了口唾沫` `腿一软`

- 例：全场**死一般的寂静**，所有人**瞳孔骤缩**，那名长老**踉跄后退**，**咽了口唾沫**。

## 三、神态/语气词（立人设、添阴狠或淡定）

`阴恻恻` `冷冷` `淡淡` `懒洋洋` `似笑非笑` `嗤笑` `挑眉` `眯眼` `勾唇` `嘴角一扯` `不咸不淡` `皮笑肉不笑`

- 例：他**淡淡**扫了一眼 / **似笑非笑**地开口 / **嗤笑**一声："就这？"

## 四、拟声词（加画面和冲击）

`咔嚓` `轰隆` `砰` `噗` `咚` `嗤啦` `哐当` `咯吱` `嗖`

- 例：**咔嚓**一声，骨头应声而断 / **噗**地喷出一口血

## 五、爽点短句/口头词（短平快节奏神器）

> 单独成句、单独成段，制造停顿和冲击。

`就这？` `不过如此。` `找死。` `太慢了。` `结束了。` `轮到我了。` `现在，` `而这，仅仅是开始。`

- 用法：长句铺陈后，突然甩一个三字短句独占一段，节奏感拉满。

## 六、修饰词使用三原则

1. **稀疏使用**：一段最多 2–3 个强修饰词，否则油腻、廉价、AI 味重。
2. **峰值集中**：把最狠的修饰词留给爽点/打脸/高潮的那一两句，平铺段落要克制。
3. **频道有别**：
   - **男频**：强化词 + 围观反应 + 拟声词为主，要"猛、爽、燃"。
   - **女频**：神态/语气词 + 情绪细节为主，要"解气、体面、有张力"，少用拟声轰炸。
