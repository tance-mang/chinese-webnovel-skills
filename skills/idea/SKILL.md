# 选题立意（idea）

帮作者把"想写小说"变成"一个能开写、有市场的具体选题"。

## 何时用
- 用户说"我想写小说但不知道写什么"
- 用户有个模糊想法，需要打磨成爆款选题
- 用户要一句话卖点 / 标签 / 黄金三章方案
- 开新书前的立项

## 工作流

### 第 1 步：定位三连问
先明确（缺哪个就问用户，或给推荐）：
1. **频道**：男频 / 女频？→ 参考 `references/channel-guide.md`
2. **平台**：起点 / 番茄 / 七猫 / 晋江 / Webnovel？→ 参考 `references/platform-profiles.md`
3. **题材偏好**：用户想写什么大类 → 从 `references/genre-library.md` 九大板块选型（悬疑/心理/都市/系统流/玄幻仙侠/女频/历史古代架空/灵异/年代乡土/小众）

> 可带参数直接定位，如 `/webnovel:idea 番茄 男频 系统都市`、`/webnovel:idea 晋江 女频 古言宅斗`。

### 第 2 步：生成 3 个差异化选题
每个选题给出：
- **一句话核心创意**（卖点钩子，能当宣传语）
- **频道 + 类型 + 平台适配**
- **金手指/核心设定**（参考 `references/goldfinger-types.md`）
- **核心爽点链**（参考 `references/trope-library.md`）
- **目标读者画像**
- **同质化风险评估**（这个点子市场上多不多，怎么差异化）

### 第 3 步：黄金三章策略
为用户选中的选题设计：
- **第 1 章**：开篇钩（3 句立危机/反差）+ 金手指上线时机 → 调 `references/hook-library.md`
- **第 2 章**：扩大冲突 + 第一次小爽点
- **第 3 章**：第一个小高潮 + 主线悬念落地（决定收藏率）

### 第 4 步：输出"开书卡"
```
【书名候选】3 个
【一句话卖点】
【频道/类型/平台】
【金手指】
【主角设定】一句话
【核心矛盾】
【爽点链】打脸X → 升级Y → 翻盘Z
【黄金三章】逐章一句话
【差异化】与市面同类的区别
```

## 原则
- 先有"爽点和卖点"，再有"故事"——网文是商品，立意先想清楚卖给谁、爽在哪。
- 不要泛泛而谈，每个选题都要具体到能直接写第一章。
- 主动指出同质化风险，给差异化建议。

## 下一步
选题定了 → 用 `outline` 出大纲，用 `hook` 打磨开篇，用 `goldfinger` 细化金手指。
